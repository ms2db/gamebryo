NiMaterialNodeXMLLibraryParser Command-line Usage

NiMaterialNodeXMLLibraryParser -in input_filename -class class_name [-modifier <storage class modifier>] [-pch <PCH file name to be included>] [-destdir <destination directory for generated files>]

Required Arguments

-in input_filename

Specifies an input XML file. This file must conform to the format expected by the NiMaterialNodeXMLLibraryReader library.

-class class_name

Specifies the class name to be generated. Note that <class_name> must not have any whitespace in it. The generated source files will be named <class_name>.h and <class_name>.cpp.

Optional arguments

-modifier <storage class modifier>

Specifies any modifiers for the class. This is often the DLL import or export predefined symbols. For instance, the standard material XML file defines this value as "NIMAIN_ENTRY", which is preprocessor define that allows the class to be used by other DLLs.

-pch <PCH file name to be included>

Specifies the pre-compiled header to be used by this class. This precompiled header should contain the definition for the modifier if it was specified.

-destdir <destination directory for generated files>

Specifies the output directory for the output source and header file. If this is not specified, the current working directory is used.

Notes

To redirect the output of this program to a text file, use > on the command-line.
