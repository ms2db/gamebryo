The TimeOfDayEditable Model

Fig. 1: The Time Of Day animation file window

This abstract model is used as a mixin. It allows users to create custom entities inheriting from one or more other models as well as this one to be animated by the TimeOfDay. This model only contains two different properties:

Attribute Type Description
TimeOfDayAssetPath String The path to the file name the time of day animation should be saved to / loaded from.
TimeOfDayAnimate List This hidden property should NEVER be modified. It automatically fills with the name of the model's properties that have the TimeOfDayAnimatable trait.

The presence of this model will open a window to select the file path to save data to when the entity is added (Fig.1). This window provides three different options:

  • Do not use time of day: No file will be selected and the entity will be added without being registered to the time of day editor. It will be possible to set the TimeOfDayAssetPath to an already existing file later.
  • Create new file: A new file will be created using the given file path. The 'Open' button will provide the user with a save window. Using an existing file will overwrite its data.
  • Load an existing file: An already existing file can be selected, however using a file that holds properties for other objects will lead to that data being lost.
