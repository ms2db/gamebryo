Animation Tool Samples

The Gamebryo Animation Tool ships with several sample characters that demonstrate facets of the tool. We recommend that you experiment with these samples. Each sample contains source art files so you can re-export Gamebryo assets.

You can find the samples in the following location:

%EMERGENT_PATH%\Samples\Models\AnimationToolExamples

Killbot

Killbot was designed as a next-generation character for video games. He has a large number of bones and has a custom rigged skeleton in 3ds Max. This model uses pixel and vertex shader model 2.0. Certain graphics cards might not be able to render this sample properly. If this occurs, you can load a secondary model that does not use the shader.

Killbot also has several sequence groups pre-defined that allow you to play around with animation blending in different scenarios.

Killbot

CyberMonkey

The CyberMonkey character was modeled in Maya to demonstrate advanced character rigging techniques with an extensive set of hand-to-hand combat maneuvers.

CyberMonkey

Rocky

Rocky is a 3ds Max model that has three animation sequences defined. The "pound" animation shows off a sequence that contains particle emitters and morphed geometry on top of the standard bone hierarchy animation. The animation system supports sequences containing any value that can be animated in Gamebryo.

Rocky

DestroyerBot

The DestroyerBot is a Maya model that represents a complex character with multiple layers of animation. This model features skinning, animated lights, billboards, and advanced particle systems.

The "Walk" and "Run" sequences show morph transitions. The shield and cloaking sequence display chain transitions as well as layered sequences. All movement sequences utilize animation accumulation.

DestroyerBot

Soldier

The Soldier model has examples of many different types of transitions. This character is also a great sample to show off transformation accumulation.

Soldier

BabyLenguin

Baby Lenguin is a character that shows off particle systems and bone animation in several sequences. This sample also has a wide variety of idle sequences.

Several sequences have morph tags indicating the left and right foot down placement with the strings "lfd" and "rfd". These keys can only be created in the source content creation package.

Baby Lenguin

Skeleton

The skeleton is an example of the use of layer roots. In this case, the character has a layer root defined at the lower spine bone. With such a root it is possible to define sequences that affect only the upper portion of the character.

Skeleton

Jack

Jack is an example of the use of additive blending. This KFM has a pre-created sequence group with the LeanLeft, LeanRight, and BigHead sequences all selected as being additive. You can adjust the weights to make Jack additively lean left or right while running or idle. Try comparing the difference between using LeanLeft or LeanRight as an additive sequence versus a normal blend.

Jack
