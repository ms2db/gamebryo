Entity Model Diagram Editor

The Entity Model Diagram Editor is where you visualize important subsections or concepts within your project. You can define the relationships between the entity models and presets within your projects. These relationships define how properties and behaviors are inherited from one entity model to another.

The Diagram Editor consists of a graphical workspace where entity models and presets can easily be added and removed. You can also drag entity models around, create relationships, and modify property values directly in the diagram editor.

The Diagram Editor is similar to the Mix-in Editor in that it gives you a strategic view of the project. However, the Diagram Editor is more flexible, in that it allows you to layout models explicitly. Because of this flexibility, the Diagram Editor is useful for initial experimentation when thinking through mix-in possibilities.

When you load a diagram into the Diagram Editor, a tab is created with the name of the project in which the item exists. The name of the diagram appears in the title bar of the view along with two navigation icons that allow you to switch back and forth between previously loaded diagrams.

Interface Element Description


(Back/Forward)		 Navigation icons that allow you to switch between previously loaded entity models, presets, and diagrams.
(Diagram Editor) A flexible graphical workspace that allows you to visualize important subsections or concepts within your project. You can experiment with different mix-in possibilities between entity models. Entities are "mixed" together to create parent/child relationships that define how properties and behaviors are inherited from one entity model to another.
Note: Only the specific property values defined for each entity model are displayed in each entity model box. This includes any property values that were overridden. Click the expand/collapse icon to display/hide this information.


(Zoom slider)		 Zooms in and out of the diagram.
