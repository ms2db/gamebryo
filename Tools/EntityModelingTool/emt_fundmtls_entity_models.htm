Entity Models

Entity Models are the core items within your project. Each entity within your game is associated with an entity model in the Entity Modeling Tool. Entity models are used to define the set of properties, including their type and default value, as well as the behaviors that are to be associated with the entity in game. An in-game entity is an instance of an entity model.

Note

The Game Framework documentation has more detailed information about Working with models.

Relationship between an entity model and entities

To take full advantage of EMT, entities can be broken down into different pieces using several entity models. Each model represents a different set of properties and behaviors that may be common to several different entities in your game. The properties and behaviors assigned to these different models are then "mixed" together to define the capabilities of each entity. Mixed entity model conceptual diagram

By breaking down an entity into different models, you can easily share common properties and behaviors between models allowing you to create a variety of content in a short period of time.
