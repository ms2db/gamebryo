Entity Model Editor (Presets)

Presets are basically entity models with certain restrictions. The main difference between a preset and an entity model is that presets can't define new properties and behaviors, which means that they must inherit their properties and behaviors from an entity model. You can, however, override and modify the values of each inherited property to help distinguish one preset from another.

When you load a preset into the Entity Model Editor (for presets), only the inherited properties are listed. The properties are sorted by parent entity model. Since no new properties can be added, you can only customize existing property values inherited from each parent. When you modify a property value, the check box beside the property value becomes checked, signifying that the parent's property value has been overridden.

Interface Element Description
Name The name of the preset.
Description Additional information that further describes the preset.
Model The entity model upon which the preset is based. You can easily change the model upon which the preset is based by selecting a new model from the list.
(Properties)
Name The name given to the property inherited from the parent entity model.
Note: The properties are sorted by parent entity model.
Value The value of the corresponding property. Since there are numerous types of properties, the default value can be in the form of a numerical value, a filename, a string, a 3D coordinate in your world, and so on.
Note: When you modify a property value, the check box beside the property value becomes checked, signifying that the parent's property value has been overridden.
