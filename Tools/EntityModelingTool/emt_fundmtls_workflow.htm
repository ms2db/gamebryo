Entity Modeling Tool Workflow

At a very basic level, EMT allows game designers to define the different abstract "models" that make up the different elements within a game world. A set of presets are derived from these models to define more specific variants. Then in World Builder, these presets are instanced into the initial world state.

For example, let's say you are creating a game world with various types of dragons. In EMT, you would create a model that represents all the different instances of dragons that exist in your game world. All the common properties and behaviors for the dragons are defined at this level. From this model, you then create a series of presets. These presets are exact replicas of the dragon model in that they inherit all the properties and behaviors from the main dragon model. At the preset level, you can differentiate each preset from the other by modifying the various property values. For example, you might have a RedDragon and BlackDragon preset. Instances of these presets, for example "RedDragonBoss42", are then added to World Builder where the initial world state is created.
