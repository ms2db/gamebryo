Presets

Presets are basically entity models with certain restrictions. The main difference between a preset and an entity model is that presets can't define new properties and behaviors, which means that they must inherit their properties and behaviors from an entity model. You can, however, override and modify the values of each inherited property to help distinguish one preset from another.

Presets are very useful when you have entities within your game that share the same types of behaviors and properties. You can create several presets for each entity and then link them all to the same parent entity model. For example, let's say all the vehicles in your game share the same types of properties and behaviors. In this case, you could create an entity model for "Vehicle", define the common properties and behaviors, and then create presets for Honda Civic, Toyota Matrix, and Ford Mustang under the "Vehicle" entity model. The specific characteristics of each vehicle can be defined by modifying each preset's property values.
