Adding/Removing Properties from an Entity Model

You can add properties to a model to define its specific attributes. A property is a member variable of a model that has a data type, and an initial/default value. Each property also has specific traits, which control the automated management of the property, such as whether it is persisted, or can be accessed on a client. You can add any number of properties to a model. Properties can also be easily removed when they are no longer required.

To add a property to an entity model

  1. Load an entity model into the Mix-in Editor. (You can also select a model in a diagram to load it into the Entity Model Editor.)

    A list of the model's properties and behaviors is displayed in the Entity Model Editor.

  2. In the Properties section of the Entity Model Editor, click the Add button. (You can also add a new property by right-clicking a row in the Property list and selecting Insert from the context menu.)

  3. A new property is added to the Property list.

  4. In the Name field, type a name that best describes the property.
  5. From the Type list, select the type of property that you want to create. There are numerous types of properties, including String, Float32, and Color.
  6. At the bottom of the Type list, you can also specify the container type for the property, if any. Select one of the following:
    • None to specify that there will be only one property value.
    • List to specify that there will be a list of possible property values.
    • Map to specify that property values will be mapped to specific keys.
  7. In the Default Value field, specify the initial value that you want assigned to the property. Since there are numerous types of properties, the default value can be in the form of a numerical value, a filename, a string, a 3D coordinate in your world, and so on. (If a black Expand arrow is displayed to the left of the Default Value field, it means that additional editing tools exist for the current property values. Click the Expand arrow to display the additional editing tools.)
  8. Click the Expand arrow beside the property name.

    Several new fields are displayed that allow you to control the management of the property.

  9. From the Traits list, select any traits that you want associated with the property. The traits help to control the automated management of the properties, such as whether they are persisted, or can be accessed on a client.
  10. From the Restriction Type list, select the type of restriction you want to place on the property value. Depending on which option you select, additional input fields may be displayed. For example, if you select the "Range" option, two new fields are displayed where you can enter a minimum and maximum value for the range.
  11. In the Description field, enter additional information that further describes the property.

To remove a property from an entity model

  1. Load an entity model into the Mix-in Editor. (You can also select a model in a diagram to load it into the Entity Model Editor.)

    A list of the model's properties and behaviors is displayed in the Entity Model Editor.

  2. From the Property list, select the property that you want to remove. (Use Ctrl+click to select multiple properties.)
  3. Click the Remove button or press the Delete key.

The property is removed from the list and is no longer assigned to the entity model.
