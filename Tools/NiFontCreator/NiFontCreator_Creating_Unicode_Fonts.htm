Creating Unicode Fonts

To launch the NiFontCreator application, double-click NiFontCreator.exe in the following location:

%EMERGENT_PATH%\Source\Tools\DeveloperTools\NiFontCreator\Win32\<VC80 or VC90>

You can use the NiFontCreator application to create Unicode fonts based on a selected font and a font configuration file. The application generates NFF files that contain font information for the NiFont library.

A font configuration file must be created prior to running NiFontCreator tool if a Unicode font is desired. The file contents include a header statement containing the version number of the file (for example, "Version: 1.0"), as well as a list of all of the Unicode characters strings used in the application. In addition, comments (lines beginning with #) may be included in the configuration file. An example of the format of this file may be found in the %EMERGENT_PATH%\Samples\GraphicsTechDemos\UnicodeCharacterSets\Data\Font Folder. The file is named CJK_Sample01.cfg.

Font configuration files must be saved as a Unicode format file. For example, when using Notepad, the encoding should be set to Unicode, and when using Wordpad, the Save As type should be set to Unicode Text Document.

When this configuration file is selected, all of the characters it contains are processed, and a list of unique glyphs needed by the application is generated based on the character strings found in the file. This list is sorted based on the frequency the characters are used to ensure that the most frequently used characters are saved in the base texture for performance.

To create a Unicode font, click the Select Font button. In the Font dialog box, select a Unicode font (such as Arial Unicode MS) and the desired font settings. Next click OK.

The selected settings will be reflected in the grayed-out elements of the main NiFontCreator dialog box.

Next, select an existing font configuration file. An example of such a file may be found at %EMERGENT_PATH%\Samples\GraphicsTechDemos\UnicodeCharacterSets\Data\Font\CJK_Sample01.cfg.

Next, select the AntiAlias checkbox, if desired. If AntiAlias is not checked, the pixels that define the characters in the font image will be fully opaque. If AntiAlias is checked, the edge pixels of the characters will be partially transparent.

Next, select the Add Default Ascii checkbox, if desired. If it is checked, 95 glyphs for Ascii codes 32-126 will also be included, to support ASCII characters in the same Unicode NFF file.

Finally, click the Save NFF button and specify a location for the resulting NFF file that will be created.

The default filename for the NFF file includes the name of the font, letters to indicate the options (B for bold, U for underlined, S for strikethrough, A for antialiased), and digits to indicate the point size for the font. For example, ArialUnicodeMS_BUA_16.nff is a NFF file representing a 16-point Arial Unicode MS font, with anit-aliasing and bold and underlined attributes.

The tool will generate all the unique Unicode glyphs included in the specified font configuration file, and if the Add Default Ascii checkbox is checked, 95 glyphs for Ascii codes 32-126 will also be included, to support ASCII characters in the same NFF file.
