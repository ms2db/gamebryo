Creating Fonts in NiFontCreator

To launch the NiFontCreator application, double-click NiFontCreator.exe in the following location:

%EMERGENT_PATH%\Source\Tools\DeveloperTools\NiFontCreator\Win32\<VC80 or VC90>

To create a font, click the Select Font button. NiFontCreator will display the Windows Font dialog box:

Next, to create an Ascii font, see the section on Creating Ascii Fonts.

To create a Unicode font, see the section on Creating Unicode Fonts.

The Select Font Color button will display the Windows Color dialog box, allowing you to bake a color into the font. At runtime, this color will be multiplied by the color specified for a Ni2DString.

The Use external atlas TGA files checkbox will cause the application to write the pixel data for this font into a separate file, called an atlas; the NFF file will contain font metadata and the name of the atlas. This atlas must not have its name changed and must be kept in the same directory as the NFF file.

Writing an external atlas will also enable the Atlas Padding and Atlas Guide Grid controls in the lower-right-hand corner of the NiFontCreator dialog box.

If Atlas Guide Grid is checked, the atlas will contain two boxes around each glyph (shown above in the sample atlas for ArialUnicodeMS,Bold, Italic, Antialiased, 22 point). The central green box will enclose the core part of that glyph that will be allocated space on the line, while the blue box encloses every pixel that will be drawn when that glyph is rendered, including pixels that may overlap adjacent glyphs.

The four Atlas Padding boxes allow additional space to be added to the four sides of this outer blue box. The sample atlas above has zero atlas padding, which means that the space between the blue and green boxes is only the minimum necessary for any pixels of the character that should (according to its TrueType font definition) overlap the rendering area of adjacent characters. Atlas Padding would expand the blue box without expanding the green.

Although these boxes appear in the atlas, they will not be rendered by Gamebryo because they have an alpha value of zero. Their intent is to guide artists in applying 2D special effects to the text - from filters like bevels or shadows up to arbitrary hand-drawn edits.
