Importing and Exporting Heightmaps

Heightmaps are currently imported and exported on a per-sector basis. In order to import or export a height map, right-click on the terrain for any sector making sure you are using the terrain edit gizmo mode. Select the 'Sector (x,y)' option from the context menu generated and this will open context menu options including export and import height map.

Clicking on 'Import Height Map' will bring up a window where the user is asked to choose an image file to import as a height map. Select a valid image file and then click 'Open' to import. Valid image files for import are image files of type .raw, .bmp, .tga, .targa, .nif, or .DDS. Additionally, files must be of the correct size to fit the sector. For any sector a valid heightmap image is of size (sectorsize + 1) x (sectorsize + 1) pixels. For example, a sector of size 512 x 512 requires importing an image sized 513 x 513 pixel for use as a heightmap.

Clicking on 'Export Height Map' will bring up a window where the user is asked to choose a location and file name and then click 'Save'. Heightmaps are exported as RAW(.raw) image files with 1 channel, 16 Bits Depth, and Byte Order IBM PC. An exported heightmap will be of (sectorsize + 1) x (sectorsize + 1) pixels.
