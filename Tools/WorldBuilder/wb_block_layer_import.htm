Importing a Layer in World Builder

While you work in World Builder, you might want or need to import a layer. For example, you might want to import a layer from one master block file to another master block file. This would allow you to share one layer in two block files. Or, you might need to re-establish a layer that you previously removed.

Notes

If you want to share a layer between two block files, the two block files must reside within the same <Asset Web Root> directory.

When sharing layers between block files, any changes made to the layer will be reflected in both block files. This includes adding/removing entities and adding/removing child layers.

World Builder: Layer option buttons

Note

You cannot import into an unloaded layer. Refer to Loading and Unloading Layers for more information.

To import a layer

  1. Run Toolbench (Start > Emergent > Gamebryo LightSpeed > Tools > Toolbench).
  2. Open your game solution (see Creating/Opening Game Solutions).
  3. In the Game Solution pane, expand the tree to view your game solution (.gsl) contents.
  4. If the Layer Editor is not already open, open it (View > Layer Editor).
  5. Select the layer from the list. If this layer is read-only, make it writable by right-clicking and selecting Revision Control > Add / Edit from the context menu. This will make the file writable and open it for edit in your revision control system.
  6. Click the Import New Layer button.
  7. In the Open dialog, select the layer (.layer) from disk. If it already exists in the block file, you will be prompted to choose another layer.
  8. The new layer appears within the Layer Editor as a child of the parent layer that you selected.
