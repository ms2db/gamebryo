Loading and Unloading Layers in World Builder

If you are working with large numbers of entities and layers in World Builder, you can limit memory usage and clutter in the Layer Editor by unloading some layers.

When a layer is unloaded, its entities as well as its sub-layers and their entities are not available in World Builder, thus freeing up some memory and visual space. Unlike simply changing Layer Visibility, entities are not just invisible, they really are unloaded.

World Builder: Unloading a layer

Note

Not all of the memory used by a layer is freed immediately after unloading it, because some of the data stays in the Undo system until the Block is closed.

To load or unload layers

  1. Run Toolbench (Start > Emergent > Gamebryo LightSpeed > Tools > Toolbench).
  2. Open your game solution (see Creating/Opening Game Solutions).
  3. In the Game Solution pane, expand the tree to view your game solution (.gsl) contents.
  4. In the Game Solution tree, double-click on the Block (.block) you want to work in. Alternatively, right-click on the Block and select Open Root Layer Only to only load entities that are in the root layer. Sub-layers will stay unloaded regardless of their own Load/Unload setting.
  5. If the Layer Editor is not already open, open it (View > Layer Editor).
  6. You can then:
    • Unload a layer by clicking its Load/Unload icon () or by right-clicking and selecting Unload Layer from the context menu. The Load/Unload icon will change and the layer's contents will be unloaded.
    • Load an unloaded layer by clicking its Load/Unload icon () or by right-clicking and selecting Load Layer from the context menu. The Load/Unload icon will change and the layer's contents will be reloaded.
    • Load an unloaded layer but keep sub-layers unloaded (regardless of their own Load/Unload setting) by right-clicking and selecting Load Layer Without Children from the context menu. The Load/Unload icon will change and the layer's entities will be loaded. Sub-layers will appear in the tree but will stay unloaded (they can then be reloaded individually if needed).

Note

The Load/Unload state of a layer is persisted as a user setting. You must save in order for this setting to be persisted, but since it is a user setting it resides in a local, user-specific file so will not affect other users.

Note

When exporting data from Toolbench, an x-block can be exported only if the block or layer it represents is fully loaded in Toolbench. If a layer is currently unloaded, then the x-block that contains it will not be exported and a warning will appear in the Output view.

However, when running from the command-line, local settings (that among other things determine if a layer is loaded or not) are ignored. This means exporting from the command-line will always export everything, regardless of which layers were loaded and unloaded the last time you saved your Block in Toolbench.

Refer to Exporting Blocks/Layers and Toolbench Command-Line for more information.
