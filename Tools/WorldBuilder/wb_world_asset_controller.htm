Working with Asset Controller

Working with World Builder (and game solutions in general) requires a running Asset Controller process in order to operate properly. This requirement exists to facilitate functionality needed by the "Asset" type property seen in both the World Builder and Entity Modeling Tool. When a game solution is opened, the Asset Controller will be commanded to configure itself such that it monitors the asset web corresponding to the currently opened solution. Value assignments for any "Asset" property types are validated using the Asset Controller. Finally, the Asset Controller is used for supporting rapid iteration in World Builder. For example, if a scenegraph changes on disk, it will be automatically reloaded and re-rendered in the World Builder view ports.

When first installed, Toolbench will have been configured such that it launches the Asset Controller process automatically upon starting the game solution plug-in and will kill the process when closing. The actual connection between the tool and the Asset Controller is also automatic.

The default Asset Controller port connection is "13215", however, if a reason is found for requiring a different port configuration, the default port can be modified using the Connections view. Also, occasionally developers find it useful to have the Asset Controller continue to run even after Toolbench shuts down. Toolbench can be configured such that it does not start or stop the Asset Controller process. The steps for performing these configurations are found below.

To change the default port of the Asset Controller

  1. Ensure that the Connections view is visible. If it is not visible, toggle it using the View menu.
  2. Switch to the Connections view and find the Emergent Asset Controller connection. It should resemble the following:
  3. Right-click the Emergent Asset Controller connection and choose Remove Connection.
  4. Next, choose Add Connection. When the Add Connection dialog appears, ensure that the new connection is named Emergent Asset Controller and set the desired port value. In the screen shot below, the port setting has been changed to 13220.

    When finished, click OK.

  5. For changes to take effect, shutdown and restart Toolbench. Note that if the connection named Emergent Asset Controller should ever be removed without specifying a new connection of the same name, then Toolbench will automatically create a default Emergent Asset Controller connection.

To change the Toolbench Asset Controller launch settings

  1. Open the Settings dialog by selecting Utilities > Settings.
  2. In the User tab of the Settings dialog, click Game Solution.
  3. In the Start/Stop Asset Controller field, select either True or False to change the game solution behavior such that the Asset Controller process will not be started / stopped by the tool. The effect will take place the next time Toolbench is started.
  4. To change the actual executable that is launched when the Start/Stop Asset Controller value is true, simply specify the full path in the Asset Controller Executable field. If environmental variables are specified in the path, they will be expanded. Note that it should be rare ever needing to change the Asset Controller executable that is launched.
