Relationship between Entity Modeling Tool (EMT), World Builder, and the Game

One fundamental concept to understand is the relationship that the World Builder has between the Entity Modeling Tool and the game application.

Recall that the Entity Modeling Tool is used to define Models (and Presets) which describe classes of game objects, their relationships, properties and behaviors. EMT can be used to directly affect a game by exporting out the models as Flat models. But even before exporting Flat models, EMT can affect World Builder. World Builder can be used to take Models (pre-exported) directly from EMT and instantiate them as Entity objects into the game world. From World Builder, the world can be exported as a game consumable Xblock file. When the game world is exported as an Xblock file, it affects the game directly which may even be used to alter the world of a live game simulation via Emergent’s rapid iteration technologies. This division between World Builder and the game is made for a variety of reasons, but primarily, it is to guarantee that it is always possible to edit and create levels for your game. If the game and World Builder were intimately linked, it would tie the development of content to any delays of the programming team. One of the key tenets of World Builder's design is that the level design team can keep creating content even if the programming team is blocked.

It should be noted that when placing entities in World Builder, only a very select subset of the game is present. Primarily, this is to assist in visualizing what the result will be in-game at the start of the game simulation. No behaviors on any entity will be executed inside of World Builder. Additionally, no game-time built-in models will be allowed to be created in World Builder. Since no behaviors will be executed in World Builder, if one wishes to see how the behaviors will act, the ability to connect to a live game via rapid iteration should be taken advantage of. Through the Toolbench Script Debugger, behaviors can even be debugged.
