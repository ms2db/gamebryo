World Builder - Layer Editor

You can use the World Builder Layer Editor to create, edit, delete, and manage layers. Layers are especially useful in multi-user editing environments occurring in the same physical space.

  • Entities may only belong to a single layer.
  • Layers can overlap and are completely arbitrary in their meaning.
  • A layer must be viewed by opening the block file that contains it.
  • The same layer file can be used by multiple block files, but this is not a recommended practice.

Within the Layer Editor, the block file is considered a layer. All new entities will be spawned into the block file by default if no other layer is the current (active) layer.

Layer Icons

Icon Details
Represents the active layer where all new entities will be created in.
Represents layers that are writeable on the disk.
Represents layers that are read-only on the disk.

Only one layer can be current and will receive all new entities and new layers. On the right are three icons that represent loaded, visible, and locked states. Layers can be unloaded to reduce resources or clutter while editing. Each layer can be made visible or hidden. If a layer is visible, but some of its entities are hidden, then the icon will be gray. The locked icon indicates whether or not the entities in that layer can be selected and edited. Again, if the layer is not locked, but some of its entities are, then it will be gray. Clicking on any of the icons will toggle the state of the layers.

Layer Commands

Layers can have a number of commands/operations done on them. These commands should be accessible by selecting the layers that the operation should be done on in the Layers Panel and right clicking and picking the operation from the right click menu.

Note

Several of these commands are disabled for layers that are currently unloaded. Refer to Loading and Unloading Layers for more information.
Command Details
Revision Control > Add / Edit This either adds the new file to source control or checks it out if it already exists.
Revision Control > Refresh This updates the displayed status of every revision-controlled file in Toolbench.
Set as Current This sets the working layer for the user. Only one layer at a time can be set as the working layer. Any new entity added to the scene will automatically be placed into this layer.
Add Selection To Layer This adds all of the currently selected entities to the selected layer.
New Empty Layer This creates a new layer with no entities.
New Layer From Selection This creates a new layer containing all currently selected entities.
Rename This action will change the name of the selected layer. It will immediately be written to disk and the old file will be removed.
Remove This action will remove the current layer from the block file's layer list.
Select All This selects all entities in the layer.
Load/Unload Layer The diskette icon next to the layer name will unload or reload the layer's contents. If the diskette is green, the layer is currently loaded and clicking the icon will unload it. If the diskette is red, the layer is currently unloaded and clicking the icon will load it.

When a layer is unloaded, the entities it contains, as well as sub-layers and the entities they contain, are not available in World Builder, thus helping users work on large projects by lessening memory usage and visual clutter.

This state is not part of the runtime state of the scene. However, please note that the export process is affected by this setting: An .xblock cannot be exported for a block or layer that is not fully loaded. When this happens, a message will appear in the Output view.

Refer to Loading and Unloading Layers for more information.
Hide/Show Layer The eye icon next to the layer name will hide and unhide all objects within the layer. If the eye is shown, the layer is visible, and any visible entities will be shown. If some of the entities are not visible then the eye will be shown but grayed out. If the eye is not shown then the layer is hidden and no entities will be visible, even if individual entities are marked as visible. This state is not part of the runtime state of the scene.
Lock/Unlock Layer The padlock icon next to the layer name will lock the selected layer. Layers that are locked are visible in the editor, but the objects inside the layer cannot be selected or edited, even if the file is writable. If the pad lock is unlocked but grayed out then some of the entities in the layer are marked explicitly as frozen.

The user may commonly set this option on layers that are not writable, but it is not on by default since the user may want to copy objects from those layers into another or view properties of entities in those layers.
Layer Settings This action will launch the settings dialog for the current layer.
