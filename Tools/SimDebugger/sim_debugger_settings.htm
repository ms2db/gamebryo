Simulation Debugger Settings

There are several Toolbench settings useful for customizing the behavior of Simulation Debugger. These settings are split between general User settings and Content settings on specific game solutions (.gsl). The game solution settings can be tuned to match the specific needs a game. All settings are accessed via the Toolbench Settings editor in the Utilities menu. The default settings values should be a reasonable starting point for all games.

User Settings

Sim Debugger | Auto Debug on Game Connect

Enabling this setting will allow Toolbench to automatically create a fresh debugging session (.simdebug) whenever a game establishes a connection with Toolbench. This is very handy for frequent users of Simulation Debugger. The generated documents will reside in the folder from which Toolbench was launched.

Sim Debugger | Heat Map - Base Scale

This setting specifies the granularity of the database holding heat map data in world units. Smaller values will yield more precise heat maps but will consume more resources. Heat maps are displayed in the World View.

Sim Debugger | Heat Map - Extents

This setting specifies the maximum world coordinate covered by a heat map. The grid of data used to generate heat maps is always square and centered on the origin. Heat maps are displayed in the World View.

Sim Debugger | Snapshot Camera Z

When connected to a running game, double-clicking the Simulation Debugger World View will display a top-down image of the game state at that location. This setting specifies the Z position of the camera used to render these snapshots.

Game Solution Settings

Sim Debugger | Actor Facing Offset

The Simulation Debugger World View uses icons to represent entities in the world. Actor-based entities include a vector indicating approximate facing. Depending on the export conventions of a particular game's art assets, this facing indicator may be consistently off by some angle. An angle can be specified in this setting to account for these coordinate system conventions.

Sim Debugger | Actor Update Buckets

This setting is a performance optimization for games with many entities active at once. It specifies over how many frames the system should amortize the cost of checking for Actor-based entity position and rotation updates. A higher number will be less taxing on the running game, but could reduce the frequency of Actor updates to Toolbench.

Sim Debugger | Update Threshold - Position

Position and orientation are closely tracked for Actor-based entities. This setting determines how far in 3D world space an Actor must move before triggering a notification for Simulation Debugger.

Sim Debugger | Update Threshold - Rotation

This threshold value specifies how far in degrees an Actor's facing (Yaw) must change before triggering a notification for Simulation Debugger.
