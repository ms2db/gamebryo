Simulation Debugger Playback

The Simulation Debugger records game events and uses them to produce a visualization of the game state. This can be used to track the current state of a connected game, but the same data can be reused to review game history. Furthermore, this review can occur while still connected to the game or much later when loading a .simdebug session from disk.

Playback Controls

The Playback toolbar includes several buttons for controlling the playback of Simulation Debugger events. Any of these buttons can be used to enter playback mode when actively debugging. Playback mode is automatically active when loading a saved .simdebug session.

Icon Name Details
Decrease Playback Speed Reduces the current playback speed by half.
Set Normal Playback Speed Sets playback speed to 1:1 real-time.
Increase Playback Speed Doubles the current playback speed.
Restart Playback Resets playback to the start of the debugging session.
Pause Playback Toggles the playback paused state.
Live Playback Toggles whether recieved events are processed immediately.

The World View also includes a timeline control and several indicators:

  • Game time at session start
  • Game time at end of session
  • Current game time
  • Current playback speed

Clicking on the timeline control will quickly advance the session to that point in recorded time. Use the Live Playback button to exit playback mode and return to real-time debugging of any connected game.

Note that recorded .simdebug sessions are never automatically saved to disk. Sessions should be explicitly saved when desired. Also note that session data in memory will be lost if the connected game resets. The session will be recycled to handle the new event data received from the game.
