Script Debugger: Introduction

Script Debugger is an integrated development environment (IDE) used to author and debug script code. Script Debugger allows quickly changing behavior in a running game without recompiling code and frequently without even restarting the game. Since Script Debugger is embedded in Toolbench, behaviors can be developed as part of an integrated workflow with Entity Modeling Tool and World Builder.

Note

Script Debugger is not a standalone tool! Script editing and debugging functionality is accessed by using Toolbench to create or open Lua behavior scripts. For example, you may open an existing game solution (.gsl), and use the Behaviors View to see a list of behavior scripts. Double-click a script file to open it for edit.
