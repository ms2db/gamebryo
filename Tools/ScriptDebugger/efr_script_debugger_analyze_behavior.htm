Using the Script Debugger to analyze behavior flows

You can use the Script Debugger Call Stack view to view the current context of script execution when you are at a break. This view can help you understand the flow of execution. The currently executing behavior or function is first, with parent calling contexts listed below. Each record in the call stack displays a script filename, line number, and functional context. When you double-click, a call stack record presents the script file listed in the record with the cursor advanced to the relevant line of script.
