Script Debugger command reference

The actions and tools available in the Script Debugger are modeled after Visual Studio and other familiar debugging environments. You can add breakpoints to any open script file by clicking in the left margin of the code editor. Breakpoints are indicated by a red stop icon in the margin and a red highlight on the relevant line of script code. Breakpoint icons will appear dimmed when script debugging mode is not active. When script execution breaks, the current line of execution is indicated by a green highlight on the line.

Script Debugger: Visual indicators in code editor

The following table lists the commands that are available from both the Debug menu and the Debug toolbar.

Command Shortcut Description
Toggle Breakpoint F9 Toggles placement of a breakpoint at the current line of script.
Continue F5 Resumes script execution from a breakpoint.
Break Ctrl + Alt + Break Freezes script execution at the next behavior instruction encountered and prevents additional scripts from executing.
Step Into F11 Debugs into any script function at the current line of execution. If no function is available to debug, this command is equivalent to Step Over.
Step Over F10 Executes the current line of script without tracing deeper into the call stack.
Step Out Shift + F11 Executes the script until returning from the current call stack level. Stepping out of a behavior script resumes normal execution.
Run to Cursor Ctrl + F10 Advances script execution until the line under the cursor is hit. This is essentially a temporary breakpoint set at the cursor.
