Shader Assets in NifViewer

The Xenon NifViewer will copy the contents of the following folder:

    %EMERGENT_PATH%\Media\Shaders\Data

...to the remote directory where the NifViewer is copied (when building in .NET). There is an Xbox360 subfolder that contains the actual shader programs that is copied to the remote directory as well. The art tool chain will also perform these same copy steps prior to launching the viewer.

By default, the PS3 NifViewer expects to find shaders in a Shaders directory relative to the .self executable. Alternatively, you can specify a directory to search for custom shaders by providing a path using the -shaderpath command-line argument. For the NifViewer to load shader programs correctly, all of the .vpo and .fpo files that a NSF/NSB file depends on must already be compiled and located relative to the NSF/NSB file as originally specified in the NSF (typically in the same directory).

When the PS3 NifViewer is launched from the tool chain using the PS3 Viewer plugin or when it is launched from the right-click Windows context menu, the -shaderpath command-line argument is automatically set to the following location:

    %EMERGENT_PATH%\Media\Shaders\Data\PS3

Note

You must ensure that any shader assets that are required by your libraries are contained in the platform-specific shader library path directory prior to deployment to the development machine.
