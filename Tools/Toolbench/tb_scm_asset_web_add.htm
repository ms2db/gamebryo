Adding Your Asset Web to the Revision Control System

After you set up revision control for a game solution, the next step is to add the game solution's asset web to revision control.

To add your asset web to the revision control system

  1. From the Toolbench menu bar, click Revision Control > Add Entire Asset Web

    The Add Asset Web to Revision Control wizard opens.

  2. Click Next to step through the wizard's introduction page.
  3. On the wizard's "Files to be Added to Revision Control" page, verify that the files in your game solution are listed.
  4. Click Finish to add the asset web files to the revision control system.

    The revision control status icon in the lower left corner will switch from green to busy while the files are being added in the background.

    The revision control status icons in the game solution will change from (not in revision control) to (add to revision control)

    The Toolbench Output View will display several info messages indicating that the files are being added to revision control. This may require selecting the other drop-down on the Output View panel and confirming that the Revision Control option is checked.

  5. From the Toolbench menu bar, click File > Save All to ensure that all the files are up-to-date.
  6. Switch to the revision control client (for example, P4V or TortoiseSVN) to submit the added files to revision control. When using Perforce, a changelist will be created whose description matches the changelist setting that you set previously in the revision control settings page.
  7. After the revision control commit operation completes, switch back to Toolbench and press F5 to refresh the SCM status icons. The icons should change from (add to revision control) to (under revision control).
