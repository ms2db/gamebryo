Understanding Toolbench File Types

The game content managed by Toolbench is stored in many different types of files, including projects, entity models, blocks, and diagrams. A unique file extension is associated with each type of content that Toolbench creates and manages. The following table lists each file type in Toolbench along with its associated extension and function.

File Type Extension Function
Game Solution .gsl Stores a list of all of the Game Framework-related content for your game (e.g. Entity Modeling Projects and World Builder Block files).
Entity Modeling Project .emtproj Store a list of all of the models, diagrams, and presets in an entity modeling project.
Entity Model .model The description of an entity model. Contains a list of other models that are mixed into the model, as well as any properties and behaviors that are defined for the model.
Preset .preset A list of property value overrides for a specific model. If you have an Orc model you may have several Orc presets, such as a Warrior Orc, Leader Orc, and Grunt Orc. Each of the presets is a different set of values for the properties that are associated with the Orc model.
Entity Model Diagram .diagram A drawing that shows the relationships between models and presets.
Flat Model .flat The game-ready format of a .model file produced by an EMT project export.
World Builder Block .block A list of entities, their specific property values, and a logical structuring of the entities in a game level
World Builder Layer .layer Layers are used for breaking up a scene into multiple files as opposed to using a single .block file. Layer files contain the entities stored in a layer along with their specific property values (similar to a .block file).
World Builder X-Block .xblock The game-ready format of a .block file produced by a World Builder export.
Lua Script Code .lua A file that contains one or more entity behavior implementations in Lua.
Plain Text .txt A text file.
User Settings File .usersettings Stores user-specific settings for Toolbench content. These generated files are generally not checked into source control.
