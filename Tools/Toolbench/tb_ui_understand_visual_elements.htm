Understanding the Visual Elements in Toolbench

Toolbench uses several icons or visual identifiers to help represent different elements in the interface. The following table describes each of the different visual elements used in Toolbench.

Icon Name Represents
Game Solution Icons
Game Solution A virtual container that holds all of the content within your game. Game content can be broken down into projects, which contain different project items, including entity models, presets, and entity model diagrams.
EMT Project Any group of entity models and other project items that represent a specific part or concept in your game, for example, a level, zone, or special mission.
World Block A series of entity models and presets laid out within a three-dimensional environment that represent a certain area of your world space at time=0.
Reference folder A folder the contains all projects that are referenced by a project or block file
Entity Modeling Icons
Entity Model A data model representing a particular object within your game upon which properties and behaviors can be applied.
Preset A restricted version of an entity model where properties and behaviors are inherited from a parent model. No new properties and behaviors can be added for presets, but you can modify the property values for existing properties.
Entity Model Diagram A graphical representation of the relationships between entity models and presets within a project.
Property Icons
Inherit/Override Property Value A property value that is either inherited or overridden.
Expand/Collapse A command to expand or collapse a list of one or more related properties.
Note: Any version of these icons can be used to represent expand or collapse.
Window Icons
Auto-hide/Display Window A command that keeps a window in view or hides it when not currently selected. When the window is hidden, a tab is displayed at the edge of the parent application window.
Close Window A command to close a floating or docked window.
