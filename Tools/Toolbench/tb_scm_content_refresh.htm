Refreshing Revision Control Status

Toolbench indicates content revision control status by displaying an icon beside the content icon. These status icons are generally correct, but when they are not you can use the F5 key or Revision Control > Refresh menu item to manually refresh the status icons. This operation is Toolbench wide and is not specific to the active content or view.

Revision Control Status in the Game Solution

Revision control status will be updated automatically when Toolbench performs a revision control operation on a given file. But, if the status of a file changes due to an external application (like a revision control client check-in), the status must be manually refreshed using F5 or the Refresh menu to update the icons displayed in Toolbench.

Note

When a refresh is performed, we don't bombard the active revision control provider with hundreds of file status requests. We invalidate the cached version of what we believe the status to be. Then only the status icons that are in view in the user interfaces throughout Toolbench are updated. The rest are updated on demand when they come back into view.
