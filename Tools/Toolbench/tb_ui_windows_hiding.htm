Hiding Windows When Not In Use

To save screen real estate, especially when building large Entity Modeling Tool diagrams, you can hide docked windows when they are not in use by collapsing them to a tab that appears adjacent to the hidden window.

Hiding a window is different than closing the window because a hidden window can be quickly shown by moving the mouse cursor over the tab that the window collapsed to. Closed windows must be re-opened by using the View menu.

Only windows with the Auto-hide icon in their title bar can be hidden. Press the Auto-hide icon to collapse the window into a tab along its edge.

To hide a window

  1. In the title bar of a window, click the Auto-hide icon. A tab with the window's name appears adjacent to the window.
  2. Click in any other window to hide the current window.

To dock a hidden window

  1. To display the window again, mouse over the window's tab.
  2. To remove the auto-hide option, click the Auto-hide icon a second time. The window is docked back into the parent application window. The Auto-hide icon will now point down indicating that the window is tacked, or docked, into place.
