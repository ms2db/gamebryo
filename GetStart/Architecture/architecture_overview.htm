Gamebryo LightSpeed Architecture Overview

LightSpeed is divided into several major subsystems:

  • The Foundation subsystem abstracts platform-specific SDK libraries and offers utilities and helpers that could apply to almost any program. All of the other subsystems rely on Foundation.
  • The Core Runtime subsystem consists of code for rendering, animation, effects, scene management, input services, and many other important visual and user interface technologies for game clients.
  • The Game Framework subsystem contains simulation-related functionality, including entities, scheduling, behaviors, and script execution.
  • The Online subsystem contains multiplayer-related functionality, including entity replication, multiplayer rapid iteration, and other network-related services.
  • Game Modules represent independent features of various types of games that can be extended or replaced to meet your specific needs. Associated services provide specific runtime entity properties, behaviors, and builtins.
    • The Standard Model Library provides these capabilities for interacting with the Game Framework.
    • LightSpeed also provides modules that serve as integration points to other middleware products (for example, PhysX).
    • LightSpeed includes optional game modules that provide additional system services. Some of these modules depend on just the Game Framework, while others depend on both the Game Framework and the Core Runtime. Examples of these modules are the InputService, the LightService, and the VisualTrackerService.
  • Gamebryo delivers several tools that enable designers and artists to expedite game develop, moving quickly from concept and design to first-playable and beyond. Tools include:
    • Toolbench
    • DCC exporters (3ds Max, Maya, and XSI)
    • Several other tools for programmers, designers, and artists
