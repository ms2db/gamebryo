About the Asset Infrastructure

Gamebryo LightSpeed's Asset Infrastructure consists of two primary sub-systems. The Asset Locating Service provides location-independent access to game assets of all types. The Asset Factory Manager controls the loading of assets into an application.

Asset Locating Services

The Asset Locating Services provide a consistent framework for naming, locating, and organizing game content assets. The asset location subsystem can be used to select locale- or platform-specific assets given a logical asset ID that identifies that set of related assets. It is also the basis of the discovery, delivery, and notification of asset changes during rapid iteration.

The metadata for assets is gathered automatically. It is kept up to date by the system and cached in an optimized database. If desired, you have the option of configuration-managing the source metadata files to better support concurrent content development on large teams. For more information about how to integrate revision control, see Adding Your Asset Web to the Revision Control System.

The Asset Runtime Service recognizes various Gamebryo asset types, including Entity models, scene graphs, and animations. The system is extensible via asset tagging and asset loaders.

In addition to asset organization and metadata, the Asset Runtime Service enables rapid iteration of known asset types. You can enable rapid iteration in your application by instantiating and adding the Rapid Iteration Service to your application. When your application is attached to an asset controller, any asset changes are automatically detected and forwarded to your application. Handling asset change notification messages falls on the appropriate asset loader(s).

Asset Loading Services

The Asset Factory Manager provides a central service for loading assets at runtime. Asynchronous background loading is supported, enabling the application to provide interactive UI updates while assets are loading. Factories are provided to load world files, Entities, scripts, scene graphs, and more. The system is extensible to new asset classes, either loaded as generic data or handled through a custom factory.
