Source Directory Orientation

The Gamebryo codebase contains a large amount of code, and it can be difficult at first to understand how the engine is organized on disk. Before the directory structure of the engine can be explored, it is important to understand how Gamebryo is organized functionally.

Gamebryo's functionality is broken down into a number of subsystems, of which the primary subsystems are Foundation, Core Runtime, Game Framework and Online.

Many high-level directories in the engine, such as Build, Samples, and Source, are further broken down by subsystem.

Foundation Directory

The Foundation directory contains the source code for the Foundation subsystem.

Directory Contents
efd Utilities, Universal types, Messaging and Services used by all other libraries.
efdLogService Remote (networked) logging and remote log control.
efdNetwork Networked Messaging used by tools.
efdPhysX Base PhysX library used by all other PhysX libraries.

GameFramework Directory

The GameFramework directory contains the source code for the Game Framework subsystem.

Directory Contents
ecr System Services that make use of the Core Runtime library.
ecrInput System Services that use Core Runtime Input.
ecrPhysX System Services that use Core Runtime PhysX.
egf Main Game Framework library; contains Scheduler and Entity.
egfLua Adds Lua scripting ability to egf.
egfPhysX Exposes PhysX simulation capabilities to the Game Framework. Capabilities include the ability to apply a force to objects in the PhysX simulation, for instance.
egfPython Adds Python scripting ability to egf.
egmAnimation Exposes the Animation system to the Game Framework to allow manipulation of animation playback and transitions from the Entity system.
egmPhysXRagdoll Specialized system service to interface PhysX ragdoll models with the Game Framework and the PhysX simulation controlled by egfPhysX.
egmPhysXTerrain Exposes PhysX-enabled Terrain to the Game Framework.
egmPhysXWhitebox Support for whiteboxing backed by collision physics, in which simple geometry is used as a stand-in for final level art.
egmTerrain Exposes the Terrain system to the Game Framework. Capabilities include runtime determination of terrain metadata based on geometric picking, for instance.
egmToolServices Contains system services used by Toolbench that could be useful in building additional tool applications.
egmVisualizers Specialized system service used by World Builder tool to display graphical depictions of entity property values.
egmVisualTracker Exposes the Visual Tracker subsystem to the Game Framework. This enables the display of real-time game performance metrics in a screen overlay.
egmWhitebox Support for whiteboxing, in which simple geometry is used as a stand-in for final level art.

Online Directory

The Online directory contains the source code for the Online subsystem.

Directory Contents
eon The main library for online system services. Provides entity replication.
eonLua Adds Lua script built-in APIs related to Online functionality.
eonPython Adds Python script built-in APIs related to Online functionality.
