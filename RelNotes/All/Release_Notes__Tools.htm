Additional Tools Release Notes

Added Features, Changed Behavior, and Fixed Bugs

Known Issues and Limitations

Note

For added features, changed behaviors, fixed bugs, and known issues and limitations related to the DCC plug-ins, Toolbench, the Entity Modeling Tool, World Builder, and other tools, see Artist and Designer Tools Release Notes.

Added Features, Changed Behavior, and Fixed Bugs

Gamebryo LightSpeed 3.2

  • The DDE Header Generator no longer writes machine-specific paths in the generated comments. Fixes were also applied to the Shipping build config and error handling routines.
  • The AssetController command AssetWeb.ListAssets has been fixed. This command was broken in version 3.1.
  • The ground plane object in PhysXNifViewer now has the skin width scaled by the PhysX scene scale, which significantly improves the behavior with very small or large PhysX objects.

Gamebryo LightSpeed 3.1.1

  • Fixed a bug that resulting in a crash when debugging Lua behaviors marked as 'immediate'. Immediate behaviors do not support runtime debugging in Toolbench. Previously, attempts to set breakpoints in such behaviors would crash the application on resume. These breakpoints are now ignored by the debugger. A message is written to the Toolbench output view indicating the breakpoint was ignored.

Gamebryo LightSpeed 3.1

  • The UpdateArtistTools script now supports Maya 2010, 3ds Max 2010 and XSI 7.5.
  • PhysXNifViewer has been modified to correctly display characters with accumulation. As a result, these characters will now have correct transforms.

Gamebryo LightSpeed 3.0

  • Added the Remote Viewer tool.
  • The UpdateArtistTools script now supports Maya 2009, XSI, Remote Viewer, and Asset Controller.
  • Added the Toolbench tools suite:
    • Toolbench Release Notes
    • Entity Modeling Tool Release Notes
    • World Builder Release Notes
    • Script Debugger Release Notes
  • Added NiThumbnailExtractor, which extracts pre-generated thumbnails from NIF and KFM files for viewing in tools or the Windows shell.
  • Added ImageGenerator, which generates images from the command-line from input NIF files.
  • Added support for Wii to Tool Plug-in Batch Tool.

Gamebryo 2.6.0

  • NiFontCreator was modified to fix incorrect texture coordinates stored in the NFF header for fonts that used padding in external TGA atlases.
  • PhysXNifViewer now loads Gamebryo shader libraries.

Gamebryo 2.5.0

  • ToolPluginBatch and ToolPluginTestbed now incorporate the new NiMeshProfileProcessor and NiMeshProfileXMLParser libraries and support the Mesh Profile System.
  • Added several new options to SceneGraphPrinter:
    • -meshverbose
      show mesh data including values within data-streams
    • -modifiers
      shows modifiers
    • -notimecontrollers
      requests that no timecontrollers be shown
    • -skipnodes
      filters out nodes from being displayed
    • -out [output_filename]
      specifies that the output should be dumped to a file having the name output_filename.
  • Added a new tool MeshProfileCompiler that helps the process for writing mesh profiles.

Gamebryo 2.3.0.1

  • NifViewer has been modified to fix a problem that arose when the "up" axis was switched and the camera was looking along the new axis.

Gamebryo 2.2.2

  • The ability to display 2-byte Unicode characters (UTC-2) through the Gamebryo NiFont library has been added for Gamebryo 2.2.2. In addition, the NiFontCreator tool has added the capability to create Unicode fonts based on a selected font and a font configuration file containing the Unicode strings used by an application.

Gamebryo 2.2.1

  • Increased the information output by Scene Graph Printer for extra data. The name, type and contents of Extra Data is now available.
  • Added the PhysX Scene Graph Printer for textual dumps of a NIF file with PhysX content.

Gamebryo 2.2.0

  • Created the PhysX Nif Viewer tool for previewing Gamebryo-PhysX content.

Gamebryo 2.1.0

  • The Tool Plugin Batch Tool now writes all tool plugin information that would have been displayed in modal dialogs to the file GamebryoOutputLog.xml instead in order to prevent batch processing from stopping prematurely.
  • The Tool Plugin Batch Tool and Tool Plugin Testbed applications now build their applications to their local directories rather than to an Application directory.
  • Fixed a bug in the UpdateArtistTools.bat script that placed the built 3dx max Plugin in the incorrect directory.

Known Issues and Limitations

All Releases

  • Asset Viewer has a known issue where it will crash if an item from "Scene Graph List" is drag-dropped into the main viewport.
  • The usability of Collada2NSF in a PS3 shader authoring pipeline is limited, and the tool is under review for removal in a future release. Customers who do wish to use it should be advised that the transposed Cg matrix format issue mentioned in Features and Limitations of the PS3 Renderer means that shaders authored in FX Composer are viewable in either FX Composer or Gamebryo for PS3, but not both.
  • The NiApplication "-refrast" command-line option does not work for any tools.
  • Gamebryo intentionally generates degenerate triangles in tristrips on some platforms because this approach is much faster than restarting a strip on these platforms. As a result, in wireframe mode on some platforms/cards (e.g. GeForce series), there may be many additional (seemingly random) lines in the rendered output that represent the degenerate triangles. This behavior results because the driver is generating lines for every edge, even for back-facing and degenerate triangles.
