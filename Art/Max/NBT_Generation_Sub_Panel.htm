NBT Generation Sub-Panel

The NBT (Normal, Bi-normal, Tangent) Generation Sub-panel is meant for use with pixel and vertex shaders doing dot-3 bump mapping or some other shader effect that requires them. By default, custom shaders themselves specify whether or not to generate NBTs and what Normal mapping method should be used when creating the art files. The additional information generated is the NBT or normal, bitangent, and tangent information in the geometry. This information is vital for dot-3 bump mapping to work correctly in a vertex shader.

NBT Generation sub-panel

Generate NBTs "On"

This check box enables the generation of NBT arrays even if they aren't required by the current material settings. NBT arrays will automatically be generated when a custom shader is applied that requests them. Additionally, NBT arrays will be generated when a map is assigned to the Normal or Parallax map slots. The Generate NBTs option should be turned on in cases where the material settings do not indicate that NBT methods are necessary but an effect that requires NBTs is going to be applied at run time.

NBT Method

This list displays all of the known normal map generation methods. Select the method matching how you generated the normal map texture. Currently, only the "Gamebryo", "ATI" and "MAX" methods are supported.
