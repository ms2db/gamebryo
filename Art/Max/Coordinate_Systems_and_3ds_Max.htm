Coordinate Systems and 3ds Max

Units

Gamebryo exports all assets using the 1 Max Unit to 1 Gamebryo unit conversion factor. Changing the display unit will have no effect on the exporter. Similarly, changing the system unit will have no effect because Max stores all internal values in Max units. If you model a landmass as a 1x1 grid using 1 Max system unit equals 1 mile and then modeled a 1x1 mosquito at 1 Max system unit equals 1 centimeter, both files would be the same size. This is because the Max internal units do not change. However, if both files were modeled with the same system unit and different display units, the sizes would be correct relative to each other. Therefore all Max assets should be modeled using the same system units.

Coordinate_System

Max cameras and Gamebryo cameras differ in how their direction vectors map to the world-space axes. The following table illustrates the mapping from Max axes to Gamebryo axes with respect to the camera. These conversions are performed automatically as rotations on the Gamebryo cameras upon export.

Coordinate systems for geometry and node objects are the same for Max and Gamebryo.

Camera Direction Max Coordinate Axes Gamebryo Coordinate Axes
Up + Z + Y
Right + X + Z
Forward + Y + X
