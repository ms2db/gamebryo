Bitmap Textures

Supported Texture Map File Types

The following image file types are natively supported in Gamebryo:

  • NIF
  • TGA
  • BMP
  • PNG
  • DDS - nVidia supplies an Adobe Photoshop plugin for importing and exporting .dds textures. See http://developer.nvidia.com/object/nv_texture_tools.html. DDS cube maps are supported.
  • SGI
  • HDR

Other image types, including JPG are not officially supported except for exports with internal textures or textures as external NIFs.

Gamebryo supports many of the parameters available in Bitmap textures.

Coordinates Panel

The Mapping type "Explicit Map Channel" is the only supported type. The Map Channel itself must map to a valid set of UV coordinates. If the UV set is not valid, this map will not be exported. UV Offset and Tiling are supported and can be animated independently. Mirror is not supported. Tile is supported. Only the W angle is supported and can be animated. The UV radio button is the preferred texture coordinate space, however, VW and WU are supported to a limited degree. Blur and Blur offset are not supported.

Noise Panel

The Noise Panel is unsupported.

Bitmap Parameters Panel

Most of the mapping types are supported in one of two ways, direct image loading from disk or sampling the Max version of the texture. The formats TGA, BMP, DDS, PNG, HDR, and SGI are directly supported. All others are sampled.

The Filtering mode determines whether or not mip-mapping is performed. Pyramidal filtering does TRILERP. Summed Area does BILERP. None performs no mip-mapping but does Nearest-Neighbor filtering. For more information on precisely what this means, go to the Texture Pixel Interpolation and Filtering section. For other texture filtering modes, such as anisotropic filtering, a Texture Export Attribute must be added to the texture using the appropriate Gamebryo toolbar button. Mono and RGB channel Output are unsupported.

The exporter supports Cropping/Placement options for UV Cropping but not Placement.

If the Image Alpha radio button is selected in the Alpha Source settings, the exporter will enable alpha blending based on whether or not the texture actually contains any transparent pixels. For example, if the texture is stored in RGBA format, but does not contain any transparent pixels, alpha blending will be disabled. The None option always disables alpha blending and the RGB Intensity setting is not supported.

Time Panel

For animated images like IFL files or AVI files, this panel is fully supported.

Output Panel

This panel is unsupported.
