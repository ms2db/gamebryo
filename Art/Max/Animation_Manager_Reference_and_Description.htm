Animation Manager Reference and Description

Actors Panel

This panel gives information about the actor root for this animation. This actor root should be an object at the top of the hierarchy containing all of the animations that you want to be extracted into kf files. For biped characters, this is usually the Bip01 or Root bone.

Actors panel

Use this field... To...
Refresh All Update the entire animation utility plug-in with the contents of the Max scene. This step is required if you rename objects or manually create and destroy sequences by editing the notetrack.
Known Actor Roots Manage more than one character in a scene. Since this is rarely the case for game development, you will usually only have one entry in this panel. The dropdown list indicates all of the known actors in the scene. Selecting from this list will set the current Actor Root for this plug-in. It was included in case you have a character that rides a horse, dragon, etc or when you need to actors to be in sync.
Add Select a node that you want to use as an actor root. After you select a node, the root will be included in the list of known actor roots.
Remove Reset the selected Actor Root to a simple node.
Accumulation Root View the name of the current Accumulation Root of the actor (usually the Actor Root). You can use this field if you are accumulating animations. This is where all accumulated translations, scales, and rotations are applied. An example of this idea would be a walking forward animation in which the character actually translates in space. Setting the Bip01 to be the Accumulation Root will allow the character to continue to translate forward without the skip back to the translation specified by the first key when the animation loops.

When exported, the Accumulation Root is divided into two nodes. One node stores the animation that needs to be accumulated, while the other stores the animation that does not get accumulated. Because this divide is made, it is important that your accumulation root does not use a non-standard animation controller such as a path controller. Additionally, the accumulation root should not be a bone influencing a skinned mesh. Since the exported nodes animations are split between two nodes, you may see serious skinning artifacts.
Pick To select a node in the Actor root's hierarchy that you want to serve as the accumulation root.
Default To define the active Actor Root as the Accumulation Root.

Active Sequence Panel

This panel is the meat of the Gamebryo Animation Utility. This panel is where you set the individual animation sequences of your actors.

Active Sequence panel

Use this field... To...
Sequences Select the sequence from the list of available sequences.
Active Sequence Name Type the name of the active animation sequence. Active sequence names cannot contain spaces. If you include spaces in the name, they will be converted to underscores (_).
Loop Sequence Toggle whether the animation sequence will loop or clamp.
Start Frame Specify the sequence's start frame.
End Frame Specify the sequence's end frame.
Start Now Set the sequence start time to the current time on the animation slider of 3ds Max.
End Now Set the sequence end time to the current time on the animation slider of 3ds Max.
Previous Select the previous sequence for this Actor on this Layer.
Next Select the next sequence for this Actor on this Layer.
Make Time Range Fit Match the current animation time range with the start and end time of this sequence.
Tip: You can jump between animations very easily by using the Next and Previous buttons in combination with the Make Time Fit button. Previewing multiple animations is easy with this addition.
If you are using morphs, the first frame of the time slider indicates the state of the base mesh, so changing the current animation time range can cause undesirable results upon export unless you set your time back to include that first frame. This capability is meant as a preview tool only.
Append New Sequence Create a new sequence immediately after the current sequence if there is more than one animation frame between the next sequence and the current sequence.
Prepend New Sequence Create a new sequence immediately before the current sequence if there is more than one animation frame between the previous sequence and the current sequence.
Delete Current Sequence Delete the current animation sequence from the file.
Note: The actual keyframes remain, but the information that identifies the keyframes as a sequence is removed.

Active Layer Panel

This panel gives information about the various Layer Roots of the current Actor Root. Layer Roots are meant to allow flexibility in mixing animations. For instance, you could mix and match a walk animation and an arm wave animation using layer roots. One Layer Root would control the arm and another Layer Root would control the hips.

Active Layer panel

Use this field... To...
Known Layer Roots Select the known layer root that you want to make active. This dropdown list shows all of the known layer roots for this animation.
Add Select a node in the hierarchy of the current Actor Root that you want to be a new Layer Root. The new Layer Root is added to the Known Layer Roots list.
Remove Removes the known layer root that is currently selected.

Sequence Accumulation Panel

This panel details the accumulation flags for the current animation sequence. This panel will only apply for sequences using the current Accumulation Root as the Layer Root.

Sequence Accumulation panel

Use this field... To...
Accumulate Rotations Specify whether or not the sequence will accumulate rotations. If selected, you can also specify the axes on which the rotation will accumulate.
Actor Faces Specify the axis that the actor is facing in its accumulations.
Accumulate Translations Specify whether or not the translations are accumulated. If selected, you can also specify which axes are accumulated.

Sequence Tags Panel

This panel gives information about morph tags and plain text tags.

Sequence Tags panel

Use this field... To...
Morph Tags
Morph Tags Selects from one of the known morph tags.
Key Text Allows editing of the text part for the current morph tag. Note that spaces are not permitted and will be translated to underscores (_).
Tag Time Specify the time for when the current morph tag occurs.
Add Morph Tag Create a new morph tag at the current time in the time slider.
Remove Morph Tag Remove the morph tag at the current time.
Extra Info Tags
Key Text Specify the text of the current extra info tag. If you try to create an info tag that starts with the strings "start", "morph: ", or "end" it will be automatically replaced so that the info tag does not interfere with other text keys that the Animation Manager uses.
Tag Time Specify the time of the current extra info tag.
Add Info Tag Create a new Info tag at the current time in the time slider.
Remove Info Tag Remove the Info tag at the current time.
