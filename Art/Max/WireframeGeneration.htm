Generation of Wireframe ABV Representations

Gamebryo allows you to create wireframe representations for ABVs, their proxy boxes, and for general selection purposes. These wireframe representations can be generated on export by using the User Defined naming conventions described in the next section and by editing the export "Process Script" such that it includes the CreateABV plug-in. The appropriate wireframe creation options must also be checked. See the CreateABV documentation for more information.

Naming Scheme

To create wireframes that represent ABVs, their proxies, and selectables, out of the Gamebryo 3ds max Plug-in, the User Defined data for a node must be tagged appropriately. The convention is:

NDL_Wireframe = <WIREFRAME_TYPE> <COLOR> <SCALE> <NAME>

WIREFRAME_TYPE

The WIREFRAME_TYPE field can be ABVProxy, ABV, or ABVSelection. ABVProxy indicates that the wireframes should be recursively created to represent proxy boxes that are found. ABV indicates that the wireframes should be created recursively to represent ABVs encountered. ABVSelection indicates to create a wireframe for each child of the given node as a selection wireframe.

COLOR

The COLOR field is used to specify the color of the wireframe. Color is represented by three float values (delimited by spaces) to represent the red, green, and blue component values respectively. The range of values is 0.0 to 1.0.

SCALE

The SCALE field acts as a float multiplier for the wireframe size that is to be created. Typically, setting the scale to be slightly larger than 1.0 (e.g., 1.01) is sufficient for a suitable wireframe representation that fits around the actual object.

NAME

The NAME field should be a string placed inside quotes and is the name used for the wireframe object.

Examples for Generation of Wireframes

Example 1

The following example creates wireframes for all ABV Proxies under the node's hierarchy. The wireframes created will be pure red, slightly larger (1.01 times) than the actually proxy, and named "Proxy ABV".

NDL_Wireframe = ABVProxy 1.0 0.0 0.0 1.01 "Proxy ABV"

Example 2

This next example will create wireframes representing the actual ABVs for all objects under the hierarchy that have ABVs. These wireframes will be pure blue, will be slightly larger (1.02 times) than the ABV, and they each will have the name "ABV Representation".

NDL_Wireframe = ABV 0.0 1.0 0.0 1.02 "ABV Representation"

Example 3

In this last example, wireframes will be created for each child of the node. These wireframe representations are referred to as selectables. The color of the wireframes will be black, they will be scaled to be larger (1.1 times) than the bounds, and they will each have the name "Select Box".

NDL_Wireframe = ABVSelection 0.0 0.0 0.0 1.1 "Select Box"
