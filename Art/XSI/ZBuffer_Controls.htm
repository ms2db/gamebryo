Controlling z-buffer usage in the XSI Plug-in

You can control how objects are rendered to the z-buffer at runtime on a per-object basis. The exporter defines a custom property set named GamebryoZBuffer for assigning z-buffer metadata to objects.

You can assign the property set to the selected object(s) in either of the following ways:

  • Use the CreateGamebryoZBuffer custom command
  • Click the Create Gamebryo Z-Buffer button on the exporter toolbar.

Three properties—Enable Testing, Enable Writing, and Test Function—govern how the object will utilize the z-buffer at runtime.

  • Enable Testing—A boolean that determine if the z-buffer will be tested at all when the object is rendered
  • Enable Writing—A boolean that determines if the z-buffer is written to when the object is rendered
  • Test Function—A drop-list that shows what comparison function will be used when testing new object z-values against the existing values in the z-buffer

The most common values (and the values that are used at runtime if the GamebryoZBuffer property set is not defined for an object) are:

  • Enable Testing "True"
  • Enable Writing "True"
  • Test Function "Less"

Note

"Less" in the Test Function context means that the fragment of the object that is being rendered is closer to the camera than an existing fragment in the z-buffer.
