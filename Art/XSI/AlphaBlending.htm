Controlling alpha blending in the XSI Plug-in

You can control how alpha blending is applied to an object in your exported NIF. The exporter defines a custom property set named GamebryoAlphaBlending for assigning alpha blending metadata to objects in the scene.

You can assign the GamebryoAlphaBlending property set to the selected object(s) in either of the following ways:

  • Use the CreateGamebryoAlphaBlending custom command
  • Click the Create Gamebryo Alpha Blending button on the exporter toolbar.

The Use Alpha Blending property determines if alpha blending is enabled. The Alpha Source Blend and Alpha Destination Blend properties specify how blending is performed between the background pixel and the object's pixel. The Alpha Testing property allows you to specify how alpha testing is accomplished. Alpha testing determines if a pixel is drawn based on an alpha test. If the alpha test is true, the pixel is drawn; otherwise it is not. For example, an Alpha Testing value of "Always" means the alpha test always passes and the pixel is drawn. Other options are available that compare the alpha value of the pixel to the Alpha Test Value property (for example, doing a "less than" or "greater than" comparison), and if that comparison is true, the pixel is drawn.

The Use Alpha Sorting property determines if this object will be alpha sorted. If this is not enabled and alpha blending is used, it is possible for the blending order to be incorrect and thus produce an incorrect final image.
