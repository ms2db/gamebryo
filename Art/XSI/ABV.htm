Defining alternate bounding volumes (ABVs) for collision

ABVs for the Gamebryo collision system are created by the Gamebryo Create ABV Tool Plug-in Pipeline plug-in when the plug-in is invoked by the processing script executed on export. The plug-in creates ABVs by keying off of the names of proxy mesh objects in the scene that are added as children of the primary geometry meshes in your scene.

The child meshes that cause ABV creation must be named according to a particular scheme in order for the Create ABV plug-in to detect them.

The following rules govern how ABV child meshes should be named:

  • The name of the child mesh must begin with "NDLCD ". The name should not include quotes and note that there is a space after the final character.
  • After "NDLCD " there can be two letters that indicate the collision volume shape and propagation technique. The first letter selects the shape:
    • S for sphere
    • C for capsule
    • B for box
    • U for union
    • H for half-space
    • T for triangles
    • O for object
    • W for bounds
    • N for "no test"
  • The second letter selects the propagation technique:
    • A for always propagate
    • N to never propagate
    • S to propagate on success
    • F for propagate on failure

For example, a node named "NDLCD BN" would use a box ABV and never propagate the collision.

Note that, as in the screenshot above, the ABV mesh proxy must be a direct child of the normal mesh in XSI. Once you have created the ABV node in the XSI scene, you may want to turn off its visibility to keep it out of the way.

Also note that XSI will replace the space in the node name with an underscore character. This does not pose a problem as the exporter converts the underscore to a space before the Gamebryo Tool Plug-in pipeline is executed so the Create ABV plug-in will function as normal.
