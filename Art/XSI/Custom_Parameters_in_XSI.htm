Using custom parameters in XSI

You can add custom parameters to custom parameter sets. You can then attach the custom parameter sets to objects in your scene. When you export the scene, these custom parameters are attached to objects as NiExtraData objects. The parameters are then associated with the Gamebryo scene graph nodes that represent these objects.

Each custom parameter type creates a corresponding NiExtraData type. Supported parameters and their associated NiExtraData types include the following:

  • Boolean—NiBooleanExtraData
  • Text—NiStringExtraData
  • Integer—NiIntegerExtraData
  • Small Integer Number—NiIntegerExtraData
  • Floating Point Number—NiFloatExtraData

You can use this facility to add application-specific metadata to objects that will be interpreted by your application at runtime. Some Tool Plug-ins in the Gamebryo Tool Plug-in Pipeline have their runtime behavior triggered by the presence of certain NiExtraData on objects at export time. For instance, the Gamebryo Scene Graph Optimization Plug-in searches all NiStringExtraData objects (regardless of their name) attached to a scene graph node for the string "NiOptimizeKeep" somewhere in the extra data's value. If the plug-in finds this string, it will not optimize away the scene graph node that has this extra data attached.
