Introduction to the Gamebryo Tool Plug-in Framework

It is often useful to customize your export out of Max, Maya, or XSI. Perhaps you want to convert your level into portalled geometry. Perhaps you need to create custom objects. Perhaps you want to save into a different file format. All of these options are possible by taking advantage of the Gamebryo Tool Plug-in Framework.

Just as the Gamebryo Max Plug-in, the Gamebryo Maya Plug-in, and the Gamebryo XSI Plug-in are plug-ins to their respective parent programs, Gamebryo Tool Plug-ins can be used to manipulate the Gamebryo asset processing pipeline. Gamebryo Tool Plug-ins can do operations like optimizing the scene graph, stripifying geometry, warn about possible performance issues, or launch external programs.

Gamebryo supports a completely configurable asset processing pipeline, illustrated in the diagram below. Note that any number of post-processing plug-ins can be added to the system before writing the file out to disk or viewing it. If necessary, multiple export plug-ins can also be used to write multiple files at once.
