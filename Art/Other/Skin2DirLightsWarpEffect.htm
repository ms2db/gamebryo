Skin2DirLightsWarpEffect

<Skin2DirLightsWarpEffect.NSF>

Requirements

  • NSF Shader Library
  • Vertex Shader 1.1 and Pixel Shader 1.4

Description

This shader was used on the character in Gamebryo's GDC 2003 demo. The shader implements skinning, lighting, normal mapping, and a warp effect. The skinning algorithm is the one from the ASMSkinningLighting shader with two directional lights added. The lighting values are calculated at the pixel level using the normal map. The warp effect works by looking up the U axis with a per-pixel normal and view vector dot product. The V axis of that lookup is the attribute, warpAlpha.

When using this particular shader, the geometry must be skinned. Use the standard Dot3BumpMap shader for static meshes.

Art Requirements

  • Skinned model
  • Base map (Using first UV set)
  • Normal, binormal, and tangent arrays (automatically calculated during export)

Custom Attributes

  • WarpAlpha—Effectively the V coordinate of the WarpEffects texture lookup.
  • WarpEffects—The texture for the warp effect. WarpAlpha will be used as V and the normal dotted with the view vector will be used for U for texture coordinates.
  • NormalMap—The tangent-space normal map that encodes the normals into texture. (Using first UV set)

Global Variables

There are no global variables for this shader.

Object Variables

  • Light1—A generic light whose diffuse color and direction are used to light the model.
  • Light2—A generic light whose diffuse color and direction are used to light the model.

Sample Art Files

This file is a good example of taking a high-resolution mesh and transferring the lighting detail via a Dot-3 Normal map onto a similarly structured but lower tessellated mesh.

This sample is also animated. When using this particular Dot-3 Shader, a Skin or Physique modifier must be applied to your mesh.
