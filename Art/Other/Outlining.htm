Outlining

<Outlining.NSF>

Requirements

  • NSF Shader Library
  • Vertex Shader 1.1 and Pixel Shader 1.1

Description

The Outlining shader implements a simple toon outlining algorithm using the normal and the view vector to determine which surfaces have normals orthogonal to the camera and hence are probably silhouette edges. These surfaces are colored with the outline color set in the shader. Hence, this is not a true outline technique since the outline actually creeps into the object geometry. Because of the algorithm used, the outline thickness will not be constant over the object and will change depending upon viewing angle. The attribute, outlineThickness, is used to change the size of the outline. When set to 1.0, the entire object will be rendered black. When set to 0.0, there will be almost no visible outlining.

Art Requirements

  • Optional ambient lights

Custom Attributes

  • outlineThickness—Used to scale the thickness of the outline.
  • outlineColor—The outline color. Note: the Alpha spinner that appears in the custom attributes rollout controls the alpha value for this color, but has no effect for this shader.

Global Variables

There are no global variables in this shader.

Object Variables

  • Light1—A generic light whose diffuse color and direction are used to light the model.
