BaseBumpWithSpatialGlow

<BaseBumpWithSpatialGlow.NSF>

Requirements

  • NSF Shader Library
  • Vertex Shader 1.1 and Pixel Shader 1.4

Description

This effect from Gamebryo's GDC 2003 demo renders an object with three textures and two lights. The vertex colors and two lights modulate the Base map. Lighting is calculated per pixel using the normal map. The glow map is applied to the object but it is attenuated by a world position that conceptually represents a light source for the glow map. In the art packages, this glow map light source defaults to the origin of 0,0,0. The glow map is applied at double intensity so that it will saturate slightly.

Art Requirements

  • Base map (Using first UV set)
  • Glow map (Using first UV set)
  • Two lights of any kind.
  • Normal, binormal, and tangent arrays (automatically calculated during export)

Custom Attributes

  • GlowAttenuation - The attenuation constant of the glow parameter over linear distance. The quadratic and linear attenuation components of the glow map are controlled by the x and y components of the 'GlowAttenuation' attribute, while the w component of that attribute represents the size of the hot spot in the middle of the glow.
  • normalMap- The tangent-space normal map. (Using first UV set)

Global Variables

  • GlowLightPosition - The world-space position of the glow light.

Object Variables

  • Light1 - A generic light whose diffuse color and direction are used to light the model.
  • Light2 - A generic light whose diffuse color and direction are used to light the model.
