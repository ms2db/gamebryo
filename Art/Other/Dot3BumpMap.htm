Dot3BumpMap

<Dot3BumpMap.NSF>

Requirements

  • NSF Shader Library
  • Vertex Shader 1.1 and Pixel Shader 1.1

Description

This shader implements a simple dot3 bump mapping effect. It uses two dynamic lights in the shader, as well as an ambient light component. Normal maps for the effect should contain scaled and biased normals in tangent space.

Art Requirements

  • Base map (Using first UV set)
  • Two lights of any kind
  • Optional additional ambient lights
  • Normal, binormal, and tangent arrays (automatically calculated during export)

Custom Attributes

  • normalMap - The tangent-space normal map for this shader. (Using first UV set)

Global Variables

There are no global variables in this shader.

Object Variables

  • Light1 - A generic light whose diffuse color and direction are used to light the model.
  • Light2 - A generic light whose diffuse color and direction are used to light the model.

Programmer Notes

The code is commented to show how to add additional lights to the effect although there would be a limit considering the number of instructions available in the pixel shader.

Dot3BumpMap was coded to vertex shader version 1.1 and pixel shader version 1.1 to provide maximum compatibility with cards. For a pixel shader 1.4 version of dot3 bump mapping, look at BaseBumpWithSpatialGlow, as that example supports more features like vertex colors and a glow map.
