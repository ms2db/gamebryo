Multi-Material Shapes and NiMesh Object Count

A convenient way of texturing an object in Maya is to use different materials on the different faces of a shape. However, Gamebryo only supports one material per NiMesh object because each material may require unique rendering settings. When the Gamebryo Maya Plug-in encounters a multi-material shape it must split the shape into multiple NiMesh objects (one for each material). If used indiscriminately, multi-material shapes can significantly increase the NiMesh object count. Multi-material shapes do not need to be avoided, but they should be used with thought.

When creating characters and discrete objects, the concept of a single texture 'Skin' for the subject is recommended. Try to use texture atlas maps (many textures packed into a single image) for the object and use UV coordinates to pick particular sub-textures out of this atlas. Because all of the object will have the same maps and shaders, it is less likely to be split for export.
