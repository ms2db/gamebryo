Maya NiEmitter Node Reference

One or more emitters can be associated with a particle system. Emitters control direction and initial attributes of emitted particles.

Note

NiEmitter nodes must be associated with a NiParticleSystem node in order to be exported and represented in the Gamebryo scene.

Base Parameters

NiEmitter Base Parameters user interface

Attribute Keyable Description
Volume Type No Possible values for this attribute are:
  • Point - Particles emit from a single point.
  • Sphere - Particles emit randomly within the volume of a sphere.
  • Box - Particles emit randomly within the volume of a box.
  • Cylinder - Particles emit randomly within the volume of a cylinder.
  • Torus - Particles emit randomly within the volume of a torus.
Direction Type No Possible values for this attribute are:

  • Directional - Emit from a cone defined by the Spread and Direction attributes.
  • Omni - Emit in all directions.
  • Hemi - Emit from a hemisphere.
Spread No Controls the angle in degrees of the Directional emitter. A setting of zero emits particles in a straight line along the Direction. For very wide spreads (~170 degrees), consider using the Hemi direction type to keep the on screen widget from becoming to large.
Direction (Degrees) No Controls the direction of the Directional and Hemi emitters. Enter positive or negative degrees to rotate the emitter around the X, Y, or Z axis.

Rate Parameters

NiEmitter Rate Parameters user interface

Attribute Keyable Description
Rate Yes The number of particles emitted per second by this emitter node.
Reset On Loop No This setting causes the particle system to kill all particles and start over when the emitter animation resets. For example, when creating smoke particle system from a fire you would not enable this checkbox.
Cycle Type No Possible values for this attribute are:

  • Loop - As time moves forward automatically restart animation keys at the beginning.
  • Reverse - Ping pong animation keys values forever.
  • Clamp - Clamp values at the last animation key value as time moves forward.
Cycle Time No This attribute controls how long an emitter will emit particles. Normally, keying the Rate attribute should be the preferred approach for controlling emission over time. The Cycle Time attribute is provided for additional control when keys are not used. Since the emitter's Cycle Type defaults to Loop, the effects of this setting may not be obvious in the Gamebryo Remote Viewer. Try setting Cycle Type to Clamp to see how Cycle Time can affect particle emission. Possible values are:

  • Last Key - The duration of particle emission will be based on the Rate attribute.
    • If Rate is not keyed, the emitter will emit particles until the Animation End Time specified on the Maya timeline control.
    • If Rate is keyed, but the last key value is not zero, the emitter will emit particles until the Animation End Time specified on the Maya timeline control. For example, if the Cycle Type is set to clamp, the Rate keys are set to 0 at the first frame and 50 on the 60th frame, and the Maya Animation End Time is set to 90 (with Maya timeline preferences set to 30 frames/sec), then the emitter will ramp up to 50 particles/second and stop emitting after 3.0 seconds (90th frame / 30 frames/sec) before stopping.
    • If Rate is keyed and the last key is zero, the emission cycle will end on the last key time. For example, if the Cycle Type is set to clamp and the last Rate key is set to zero on the 60th frame (Maya timeline preferences are set to 30 frames/sec), then the emitter will emit for 2.0 seconds.
  • Fixed - Allows the artist to specify the number of seconds that the emitter will emit particles. Selecting this option will enable the End Time attribute .
End Time No This attribute controls the duration that an emitter will emit particles. For example, if the Cycle Type is set to clamp, Cycle Time is set to fixed, and End Time is set to 7.0, then the emitter will emit particles for 7 seconds and then stop.

Initial Particle Parameters

Note

The term variance is Gamebryo’s way of describing adding randomness to an attribute. The variance represents a range of evenly distributed random values that will be added to the base value. For example, if the Particle Size is 7.0 and the Particle Size Variance is also 2.0, then the Particle Size at birth will be a random number between 5.0 and 9.0.

NiEmitter Initial Particle Parameters user interface

Attribute Keyable Description
Life Span Live Forever No If checked, particles will never die of old age.
Life Span Yes Number of seconds that a particle lives.
Life Span Variance No The life span variation for new particles. The resulting life span values assigned to new particles will be evenly distributed over the range [Life Span +/- Life Span Variance].

A variance of 0.0 indicates no randomness in life span.
Particle Size Yes The size assigned to new particles at emission. Particle Size can also be affected by an animated particle size over the life of the particle on the NiParticleSystem node.

The final particle size is calculated as follows:

Final Size = [Particle Shape Particle Size] x [Emitter Particle Size] (assuming zero variance)
Particle Size Variance No The size variation for new particles. The resulting size values assigned to new particles will be evenly distributed over the range [Particle Size +/- Particle Size Variance].

If Particle Size Variance is larger than Particle Size, it is possible that the size will be set to a negative value. In this case, the particle will be flipped, and the bounding volume may be computed incorrectly. Thus, it is recommended that Particle Size Variance be less than or equal to Particle Size.
Speed Yes The speed assigned to new particles at emission.
Speed Variance No The speed variation for new particles. The resulting speed values assigned to new particles will be evenly distributed over the range [Speed +/- Speed Variance].

A variance of 0.0 indicates no randomness in particle speed.
Speed Flip Ratio No The proportion of particles that have their velocity inverted from the value defined by other parameters. This attribute is a percentage expressed as a decimal and must be in the range [0.0, 1.0].

For example, if Speed Flip Ratio is 0.25 (25%), then a quarter of the particles will have an inverted velocity at birth.

Rotation Parameters

NiEmitter Rotation Parameters user interface

Attribute Keyable Description
Random Rotation Direction No This attribute controls whether or not the particle's spin direction is random. The rotation speed sign is randomly set (positive or negative) when this attribute is checked.
Rotation Speed Yes The number of degrees each particle will rotate every second of the particle's lifespan.

For example, a particle with a Life Span of 2.0 seconds and a Rotation Speed of 45.0 degrees/second will rotate 90.0 degrees during its lifetime (assuming zero variance).
Rotation Speed Variance Yes The rotation speed variation in degrees per second for new particles. The resulting rotation speed values assigned to new particles will be evenly distributed over the range [Rotation Speed +/- Rotation Speed Variance].

For example, if Rotation Speed is 90.0 degrees/second and Rotation Speed Variance is 45.0 degrees/second, new particles will have a random rotation speed that ranges between 45.0 and 135.0 degrees/second.
Note: This speed is constant for the life of the particle. Only the initial speed is random.
Rotation Angle Yes This attribute represents the rotation (in degrees) of a particle at birth. It does not make the particle spin, this attribute represents the initial angle of the particle.

To make a particle spin during, set the Rotation Speed and/or animate the angle by creating keys on the Rotation Angle attribute of the NiParticleSystem node.

Rotation Angle = [Particle Shape Rotation Angle] + [Emitter Rotation Angle] + [Rotation Speed] * time during particle’s life (between 0.0 and 1.0)

This is assuming zero variance.
Rotation Angle Variance Yes The resulting rotation angle values assigned to new particles will be evenly distributed over the range [Rotation Angle +/- Rotation Angle Variance].
Mesh Particle Random Rotation Axis No Whether or not a random rotation axis will be assigned to new particles. As the name implies, this attribute only applies to mesh particles.
Mesh Particle Rotation Axis No The rotation axis assigned to new particles. This value is only used if Mesh Particle Random Rotation Axis is false. As the name implies, this attribute only applies to mesh particles.

Surface Emitter Parameters

These parameters are only displayed for surface emitters.

NiEmitter Surface Emitter Parameters user interface

Attribute Keyable Description
Initial Velocity Type No Determines how tDetermines how the initial velocity direction for a particle is computed.
  • Normals - Use the normals to determine the velocity direction.
  • Direction - Use the direction specified in the Direction parameter as the velocity direction.
Emission Type No Determines how tDetermines how the mesh data will be used to compute the emission position for a particle. Possible values for this attribute are:
  • From Vertices - Emit from a random vertex in the mesh.
  • Face Center - Emit from the center of a random triangle in the mesh.
  • Edge Center - Emit from the center of a random edge of a random triangle in the mesh.
  • Face Surface - Emit from a random position inside a random triangle in the mesh.
  • Edge Surface - Emit from a random position along a random edge of a random triangle in the mesh.
Mesh No A convenient way to navigate and view the list of meshes that emit particles from their surfaces. These connections can also be viewed in the Maya Hypergraph.
