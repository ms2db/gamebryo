Four Texture Concerns

There are four issues you must keep in mind when adding textures to a Maya scene:  the texture's dimensions, its aspect ratio, the effect of filtering, and its size in VRAM.  VRAM is the memory that is actually on the graphics card and can be accessed quickly by the rendering hardware.  The framebuffer (the part of VRAM used to draw and display the screen) and the Z-buffer both occupy some portion of VRAM and, thus, reduce the amount of memory available for textures.

Cropping/resizing for the Gamebryo Maya Plug-in

The first texturing issue is that Gamebryo limits a texture to having each of its dimensions be a power of two.  The Gamebryo Maya Plug-in will give an error message in the Output Window if any of the texture's dimensions are not a power of two, and the texture will be resized to fit a power of two.

Aspect Ratios for the Gamebryo Maya Plug-in

Second, textures in Gamebryo do not need to be square, but extreme aspect ratios will fragment VRAM.  This behavior results because odd-shaped textures are not as easy to pack together as regularly shaped ones.  In general, you should only use aspect ratios of less than 8:1. Additionally, you should settle on a base set of texture sizes.  Textures with the same dimensions are easily interchanged in VRAM—allowing for quicker VRAM management schemes.  Having a lot of textures with widely varying sizes will defeat this replacement-based texture management.

Filtering for the Gamebryo Maya Plug-in

The third texturing issue is that filtering (also known as "mipmapping" or "trilinear interpolation" in Gamebryo) increases the original VRAM size of a texture 1.33 times.  When mipmapping is enabled, all textures must be stored along with their filter levels, where each level is a quarter the size of the prior level.  Mipmapping will improve texture aliasing, but it comes at the cost of some space in VRAM.  In other words, mipmapping improves visual quality in the scene and should be used, in general, depending on the platform. A default global filter can be set at export and overridden on individual NiMultiShader Materials.  For a detailed discussion of Filtering in the Gamebryo Maya Plug-in, please see the section on Gamebryo Maya Plug-in_Export_Filtering_Options.*MyImmerse_Export_Filtering_Options

Formats for the Gamebryo Maya Plug-in

Finally, for a given texture, the amount of VRAM it occupies is determined by its size and format. This information is important because ultimately it determines how many textures can be used in a scene.  You can use palletized BMP files, compressed files like .DDS, or uncompressed .TGA files.

It is possible to have more textures than will fit in VRAM.  In this case, the graphics card will fetch the texture from system memory whenever it is needed for rendering.  This process is much slower than getting a texture from VRAM and should be completely avoided on PCI graphics cards.  It is not necessary to completely avoid overflowing VRAM on AGP graphics cards, but performance will still degrade, as more textures have to be retrieved from system memory.  Other than avoiding it entirely (by fitting all textures into VRAM), there is little you can do to improve the fetching of textures from system memory.
