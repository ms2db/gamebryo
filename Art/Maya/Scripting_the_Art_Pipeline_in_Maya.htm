Scripting the Art Pipeline in Maya

It is often useful to customize your export out of Max or Maya. Perhaps you want to convert your level into portalled geometry. Perhaps you need to create custom objects. Perhaps you want to save into a different file format. All of these options are possible.

You use the plug-in's Script Selection Dialog to choose a process script to execute each time you export the scene.

Script Types

Scripts are divided into four separate categories to allow reusability:

  • Import Scripts are composed entirely of process plug-ins. These plug-ins should not be used with the Gamebryo Maya Plug-in.
  • Process Scripts are composed entirely of process plug-ins. These plug-ins are always run whenever you export or view the file.
  • Viewer Scripts are composed of one or more viewer plug-ins. Viewer scripts are generated when the Gamebryo Maya Plug-in loads.
  • Export Scripts are composed of one or more export scripts.  These kinds of scripts are not used by the Gamebryo Maya Plug-in.

As noted above, the Gamebryo Maya Plug-in only uses two different types of scripts: Process scripts and Viewer scripts.

Script Directories

The Gamebryo Maya Plug-in loads the script template files from the following directory:

\Maya\bin\plug-ins\Gamebryo Scripts

Plug-in Directories

The Gamebryo Maya plug-in loads all of the plug-ins from the directory %EMERGENT_PATH%\SDK\Win32\DLL\ToolPlugins. By default, the EMERGENT_PATH variable is set by the installer.

Default Scripts

The following scripts are provided by default in the Gamebryo Maya Plug-in installation.

  • Default Export—This basic script is all that is needed for almost all scenes to export correctly with optimization.
  • Default Character Export—This script is very similar to the Default Export script but with the addition of the Controller Extractor for exporting animation. All animations will be exported to a single KF file.
  • DefaultMultiExport—This script contains the MultiNIFExportPlugin, which will write out multiple NIF files if MultiNIF export attributes have been added to the scene.
  • Default Multi-File Base Character Export—This script is the same as the Default Character Export and can be used to create the NIF and KFM file when a character's animations are stored in multiple Maya files.
  • Default Multi-File Character Export—This script does not create a NIF or KFM file; just the KF file associated with the scene's animations.
  • Default PhysX Export—The PhysX extractor is needed to export scenes that contain PhysX content.
  • Default PhysX Ragdoll Export—This script contains the necessary plug-ins for PhysX ragdoll characters to be exported.
