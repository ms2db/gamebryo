Asset Conversion with the NiGeometryConverter

In order to avoid requiring customers to re-export immediately their Gamebryo 2.3 and earlier NIF assets with the Gamebryo 2.5 DCC plugins, automatic asset backwards-compatibility is provided. It is highly recommended that customers utilize the NiGeometry backwards-compatibility feature of Gamebryo 2.5 during transitional development only. The NiGeometryConverter class is not a desirable option for a shipping title because of the memory use overhead inherent in the conversion process and the layout of the output geometry. NiGeometryConverter does not interleave data. For maximum performance, customers are encouraged to re-export assets from the DCC tools or to use the ToolPluginBatch utility to process NIF files for Gamebryo 2.5.

Instances of these classes in NIF files are automatically converted at load time by the NiGeometryConverter class, the NiGeomMorpherConverter class, and/or the NiPSConverter class.

Instances of the following classes are converted to instances of NiMesh:

  • NiAdditionalGeometryData
  • NiGeometry
    • NiLines
    • NiTriBasedGeom
      • NiTriShape
        • NiScreenElements
      • NiTriStrips
    • NiParticles
  • NiGeometryData
    • NiLinesData
    • NiParticlesData
    • NiTriBasedGeomData
      • NiTriShapeData
        • NiScreenElementsData
        • NiTriShapeDynamicData
      • NiTriStripsData

Instances of the following classes are converted to instances of NiSkinningMeshModifier:

  • NiSkinData
  • NiSkinInstance
  • NiSkinPartition

Instances of the following classes are no longer supported by the runtime engine. They are ignored or removed, not converted by the NiGeometryConverter class:

  • NiScreenTexture
