Foundation Bridge Layer

In Gamebryo LightSpeed 3.0, many classes and methods in the NiSystem and NiMain libraries have been moved to a new library called Foundation (namespace efd). The classes that were moved are bridged. This means that very few code changes are required to convert to 3.0. However, you will need to make sure that project file settings are modified to link against the efd library.

Most classes are bridged via the preprocessor to ensure no additional overhead. However, a few classes are subclassed (for example, to provide NiStream streaming support). You may find that these classes are slightly slower in debug builds. In optimized builds, there will be no additional overhead for these classes.

Note

In the tables below, "non-trivial" indicates that a bridging class exists (as opposed to being bridged by the preprocessor).
