Checks if the specified value represents a true value.

If the value is equal to the string "true" (ignoring case) or the integer "1" then the value is considered true. All other values are considered false. If the value is not found at all then the defaultValue is returned.

virtual ISection* AddSection(
const efd::utf8string& strName,
IConfigSource* pSource) = 0

Adds a new section as a child of the current section.

If a dot separated section hierarchy is specified, it will add the section(s) as a child of the current section. If a section exists, it will use that section; otherwise it will add the new section specified.

A section name may not contain the characters specified by IConfigManager::kIllegalSectionChars.

Parameter Description
strName Name of the section to retrieve. Dot separated section hierarchy names are accepted.
pSource Pointer to the source that added the section.
Return Value ISection* : Pointer to the section specified(regardless of whether it was added or an existing section was found), or NULL if the section names specified are illegal.

virtual bool AddValue(
ISection* pRoot,
const efd::utf8string& strName,
const efd::utf8string& strValue,
int iPriority,
IConfigSource* pSource) = 0

Adds a new value as a child of the current section.

If a dot separated section hierarchy is specified, it will add the required section(s) as a child of the current section and then as the name/value pair to the last section just as if AddSection() were called first.

A value name may not contain the characters specified by IConfigManager::kIllegalNameChars.

Parameter Description
pRoot Root section, so that it can find variables.
strName Name of the section to retrieve. Dot separated section hierarchy names are accepted.
strValue Value string to associate with the name specified.
iPriority Priority of the name/value pair.
pSource Pointer to the source that added the name/value pair.
Return Value bool True if the value was added; false if the name already exists with a higher priority.

virtual bool AddLink(
ISection* pRoot,
const efd::utf8string& strName,
const efd::utf8string& strLink,
const ISectionEntry* pILink,
int iPriority,
IConfigSource* pSource) = 0

Adds a new value link as a child of the current section.

A link simply points the given name to the value of another entry in the configuration. It creates a true link, not a copy, so that if the original information changes, the link will still point to the updated information. If a dot separated section hierarchy is specified, it will add the required section(s) as a child of the current section and then add the link to the last section the same as if AddSection() were called first.

A link name may not contain the characters specified by IConfigManager::kIllegalNameChars.

Parameter Description
pRoot Root section, so that it can find variables.
strName Name of the section to retrieve. Dot separated section hierarchy names are accepted.
strLink Name of the entry to which to point.
pILink Entry to which to point.
iPriority Priority of the name/value pair.
pSource Pointer to the source that added the name/value pair.
Return Value bool True if the value was added; false if the name already exists with a higher priority.
