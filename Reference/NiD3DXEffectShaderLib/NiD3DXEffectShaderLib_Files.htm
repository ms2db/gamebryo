NiD3DXEffectShaderLib Files

Description

NiD3DXEffectShaderLib.cpp and NiD3DXEffectShaderLib.h provide the interface functions for static lib based builds of the shader library. It defines a single function for creating the library, which the application must call during ClassCreationCallback. Refer to the documentation on Exposing your ShaderLibrary to the tool-chain for more details. The interface also provides functions for setting flags used to compile the FX files.

Supported Library Functions

bool NiD3DXEffectShaderLib_LoadShaderLibrary(NiRenderer* pkRenderer, int iDirectoryCount, char* apcDirectories[], bool bRecurseSubFolders, NiShaderLibrary** ppkLibrary)

This function, similar to the other shader library loader functions, creates an NiD3DXEffectShaderLibrary instance, and searches the directories passed into the apcDirectories array for valid D3D Effect files. It is assumed that such files will have the extension ".FX" or ".FXO". If the pkRenderer parameter is valid, it will use that renderer to load the D3D Effect files; otherwise, it can parse the file without a valid renderer, but it will not be able to generate an NiD3DXEffectShader until a renderer is created.

unsigned int NiD3DXEffectShaderLib_GetD3DXEffectCreationFlags()

void NiD3DXEffectShaderLib_SetD3DXEffectCreationFlags(unsigned int uiFlags)

These functions get and set the flags that will be used to create the D3D Effect object from the files. These flags are described in the Microsoft DirectX9 documentation under "D3DXSHADER", and are used to allow for debugging of vertex and pixel shader programs.

These functions continue to exist, but an easier method to access flags are the NiD3DShaderProgramFactory::GetUniversalShaderCreationFlags and NiD3DShaderProgramFactory::GetUniversalShaderCreationFlags, described in the NiD3DShaderProgramFactory documentation.

void NiD3DXEffectShaderLib_EnableFXLSupport(bool bEnable)

Enable or disable support for parsing .FXL files in the D3DX Effects shader library. By default, this feature is set to true on all platforms other than Xbox 360.

void NiD3DXEffectShaderLib_SetApplicationInstance(HINSTANCE hInstance)

This function is deprecated, and no longer necessary. Calling the function will have no effect whatsoever.
