Window Creation

The window is created with a client region of the size specified by the calls to GetWidth and GetHeight.  The NiApplication-derived class implements both of these functions. A typical Gamebryo application may specify a client region of size 640x480. The full window size depends on width, height, window style, and whether or not the window has a menu.

GetMenuID is used to determine if a menu is required.

The window style is obtained by a call to GetWindowStyle. Currently, the style under Windows is a static member of NiApplication and defaults to WS_OVERLAPPEDWINDOW | WS_CLIPCHILDREN

The title bar of the window has the text specified in the call to GetWindowCaption, which is implemented by the NiApplication-derived class.

Both the application instance handle and the window handle are passed back to the NiApplication object via calls to SetInstanceHandle and SetWindowHandle.

The NiApplication-derived object for constructing additional Windows objects, such as dialog boxes, uses these functions.
