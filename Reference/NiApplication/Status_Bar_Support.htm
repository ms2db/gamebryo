Status Bar Support

The NiApplication-derived class may specify that the window should have a status bar. The mechanism is via a call to GetNumStatusPanes, which returns the number of desired panes in the status bar. If the returned value is zero, then no status bar will be built. At this point, if the number of panes is larger than one, the panes are equally sized. The status bar handle is passed back to the NiApplication-derived class, so that it may write text to that bar, as desired.
