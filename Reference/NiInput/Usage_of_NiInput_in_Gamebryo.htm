Usage of NiInput in Gamebryo

NiInput is used throughout the tutorials, demos, and games. NiApplication contains a smart pointer to an NiInputSystem instance. Each platform-specific instance of NiApplication contains a function CreateInputSystem which creates the instance of the input system the applications utilize.

The NiApplication class also defines two additional functions related to input. The first is UpdateInput, which calls NiInputSystem::UpdateActionMap if the system is operating in Action Mapping Mode or NiInputSystem::UpdateAllDevices if the system is operating in Immediate Mode. This function simply processes the input system and is called once per frame.

The second function, ProcessInput, is where all the processing occurs. In all the samples (except for InputDemo) this simply involves polling the input devices for events the application is interested in.

Please refer to the Learning Gamebryo section for more information on how NiInput is used in Gamebryo in more detail.
