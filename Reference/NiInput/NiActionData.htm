NiActionData

: public NiMemObject

<NiActionData.h>

Description

NiActionData is used to allow the system to report actions that have occurred during processing of action mapped devices.

Constructors

The system creates a pool of action data instances. An application should never create one - therefore the application should view the constructor as 'internal use only'.

Supported Member Functions

const unsigned int GetAppData() const

Returns the user-supplied app data value for the action that happened.

const unsigned int GetDeviceID() const

Returns the device ID associated with the action. This is set to the device on which the action occurred on, allowing applications to identify which device generated the action.

const void GetDeviceID(unsigned int& uiPort, unsigned int& uiSlot) const

Returns the port and slot combination for the device on which the action occurred on.

const unsigned int GetDataValue() const

Returns the data value of the control that generated the action. For example, and analog control will set this to the analog value of the button when the action occurred. If the action was set to RETURN_BOOLEAN, this value will be 0 or 1.

const void GetContext() const

Returns the app-supplied context data associated with the action.

const unsigned int GetTimeStamp() const

Returns the time stamp of when the action occurred. [NOTE: This is not used in the current release.]

const unsigned int GetSequence() const

Returns the sequence number of the generated action. Reported actions get assigned an incremental number to allow for identifying the order actions occurred in.
