Inheritance graph for etk::RCSResult

Description

This class is used to return status and error information from RCS methods.

The caller passes a RCSResult object into the method, which is populated by the based on the results of the RCS operation.

Public Types

etk::RCSResult::Reason

The reason an RCS operation failed.

Name Description
RCS_NO_ERROR The operation succeeded. No error occurred.
RCS_UNRECOVERABLE_ERROR A serious error has occurred that cannot be retried by RCSFeedback::HandleError() method.
RCS_ERROR A RCS operation has failed, but a retry is possible is a custom RCSFeedback object has been assigned to the provider.
RCS_NOT_UNDER_REVISION_CONTROL The file is not in the depot or repository.
RCS_USER_IGNORE The user has chosen to ignore the RCS operation.
RCS_USER_CANCEL The user has cancelled the RCS operation by invoking RCSProvider::Cancel().

void SetHideCommand(bool hideCommand)

Sets flag indicating that the RCS command should be hidden from the user by GetDisplayMessage().

This is rarely used when the RCS command fails, but may confuse the user since it may not map directly to the high-level action being performed or in cases where the command succeeds (like a RCSProvider::Status() method), but the status indicates that something failed.

Parameter Description
hideCommand True indicates that the command should not be included in the message returned by GetDisplayMessage().

Parameter Description

efd::utf8string GetDisplayMessage()

This method combines the action, command, and response associated with a RCS operation into a single user-displayable string.

The command portion will be excluded if the the SetHideCommand() method has been called to set hide to true.
