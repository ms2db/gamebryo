Inheritance graph for etk::RCSFactory

Description

This singleton object is used to create a revision control system provider object from available provider plug-in DLLs.

When the RCSFactory singleton is initialized, it will attempt to load plug-ins that are used to access RCS through a common RCSProvider interface. The available providers can be queried after initialization.

Public Static Methods

static bool Initialize(
RCSResult* pResult = 0,
const char* pPluginDirectory = 0)

Create the singleton and query for RCS providers by attempting to load plug-in DLLs.

Parameter Description
pResult Optional result object passed by the caller to hold any error messages generated during initialization
pPluginDirectory Optional path to a directory to find the providers DLLs. If no directory is supplied the following directory will be used EMERGENT_PATH%\sdk\Win32\DLL.
Return Value True if the singleton was successfully created and there is at least one provider plug-in available. Check pResult for additional details when an error occurs.

static void Shutdown()

Unload all provider DLLs and destroy the singleton.

Make sure all provider objects previously created by the Create() method are deleted prior to calling this method since it will unload the DLL associated with the implementation of these objects.

static bool GetAvailableProviders(efd::list<efd::utf8string>& rcsNames)

Query the display names for the available revision control system providers.

For example: "Perforce" or "Subversion"

The RCS names returned by this method only indicate the provider DLLs are available. It does not guarantee the RCS client software required to access the system is properly installed. This check is not performed until the Create() method is called.

Parameter Description
rcsNames An list of RCS name strings
Return Value True is there is at least one provider available.

static RCSProvider* Create(
const efd::utf8string& rcsName,
RCSResult* pResult = 0)

Instantiate a provider object that can be used to access a revision control system through the common RCSProvider interface.

Parameter Description
rcsName Name of RCS system obtained from GetAvailableProviders() method call.
pResult Optional result object passed by the caller to hold any error messages generated while trying to instantiate a RCS provider object
Return Value This method will return NULL if the underlying RCS client is not properly installed. For example, if the Subversion DLLs are not in the PATH, Windows error message boxes will be displayed and this method will return NULL.
