NSBRenderStates

: public NiMemObject

<NSBRenderStates.h>

Description

NSBRenderStates is a static class, providing platform-independent mappings for render states. Functions are provided for converting these values to the platform-dependent counterparts during loading. This capability is required due to certain platforms having different values for the same enumeration.

Constructors

As this class is a static class, i.e., all member functions are static, no constructor is provided.

Static Member Functions

static bool ConvertNSBRenderStateValue(NSBRenderStateEnum eNSBState, unsigned int uiNSBValue, unsigned int& uiValue)

Used to convert platform-independent render state values to platform-specific values. For the given platform-independent render state, eNSBState, and the platform-independent uiNSBValue, retrieve the platform-specific value in uiValue. Returns true if successful, false if the render state was not found or the value could not be converted.

static NSBRenderStateEnum LookupRenderState(const char* pcRenderState)

Looks up the platform-independent render state for the given pcRenderState string. Primarily used by the parsers to form NSB render state group entries.

static bool LookupRenderStateValue(NSBRenderStateEnum eRenderState, const char* pcValue, unsigned int& uiValue)

For the given platform-independent render state, eRenderState, convert the given value string, pcValue, to the platform-independent value. Primarily used by the parsers to form NSB render state group entries.

Notes

As stated above, this class defines a platform-independent method for defining render states and their associated values to allow for the shader system to be used on the various renderers. It is important to note that conversion is not done during run-time, just upon loading the binary shader.

NSBRenderStateEnum Enumeration

The render state enumeration provides a mapping of render states that is platform-independent. This arrangement is necessary due to some render states being available on only a subset of the platforms, as well as the fact that some platforms have different enumerated values for the same render state.

There render states are too numerous to list in this document, but they are named by simply replacing 'D3DRS_' with 'NSB_RS_' in front of the name of the D3D render state. For example, D3DRS_ZFUNC becomes NSB_RS_ZFUNC. For a complete list of them, please refer to the NSBRenderStates.h source file.

NSB_RS_INVALID provides an indicator for invalid render states this setting is used to identify invalid entries in the list.
