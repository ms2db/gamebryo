NSBStageAndSamplerStates

: public NiMemObject

<NSBStageAndSamplerStates.h>

Description

NSBStageAndSamplerStates is a static class, providing platform-independent mappings for texture stage and sampler states. Functions are provided for converting these values to the platform-dependent counterparts during loading. This capability is required due to certain platforms having different values for the same enumeration.

Constructors

As this class is a static class, i.e., all member functions are static, no constructor is provided.

Static Member Functions

static NSBTextureStageState LookupTextureStageState(const char* pcTextureStageState)

Looks up the platform-independent texture stage state for the given pcTextureStageState string. Primarily used by the parsers to form NSB texture stage state group entries.

static unsigned int LookupTextureStageStateValue(NSBTextureStageState eStageState, const char* pcValue)

For the given platform-independent texture stage state, eStageState, convert the given value string, pcValue, to the platform-independent value. Primarily used by the parsers to form NSB texture stage state group entries.

static NSBTextureSamplerState LookupTextureSamplerState(const char* pcTextureSamplerState)

Looks up the platform-independent sampler state for the given pcTextureStageState string. Primarily used by the parsers to form NSB sampler state group entries.

static unsigned int LookupTextureSamplerStateValue(NSBTextureSamplerState eSamplerState, const char* pcValue)

For the given platform-independent sampler state, eSamplerState, convert the given value string, pcValue, to the platform-independent value. Primarily used by the parsers to form NSB sampler state group entries.

Notes

As stated above, this class defines a platform-independent method for defining texture stage and sampler states and their associated values to allow for the shader system to be used on the various renderers. It is important to note that conversion is not done during run-time, just upon loading the binary shader.
