Creating UI Elements

NiSample uses the NiUserInterface library to provide a consistent set of UI controls for all demos.

Several member variables are provided for an application to consistently lay out UI elements:

  • m_fUIElementHeight - Specifies the default height of an UI element.
  • m_fUIElementWidth - Specifies the default width of an UI element.
  • m_fUIGroupHeaderHeight - Specifies the default height of a window's title bar.
  • m_kUIElementGroupOffset - Specifies the default offset for adding UI elements to a UI group.

Several member functions are provided to allow a developer to override how the UI is created:

(Please see the class reference for more details on these methods)

  • CreateUISystem - Initializes the UI system and navigation system.
  • CreateUIElements - Called to customize what UI elements are created. This is overloaded by most samples.
  • CompleteUISystem - Called to generate all rendering resources for the UI system.
  • RenderUIElements - Called to recursively render the UI elements.
  • AddDefaultUIElements - Adds the default elements used by all NiSample derived classes. Should be called from the CreateUIElements method.
  • AddNavigationHelpUIElements - Adds the help table for each navigation controller registered.
  • AddLogEntry - Adds a line to the output log.
  • ChangeLogEntry - Edits a line in the output log.
  • ReinitializeLogText - Recreates the label that defines the output log.
