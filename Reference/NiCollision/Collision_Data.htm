Collision Data

During the narrow phase of the Gamebryo collision detection system, collision detection is performed on NiAVObject objects, if the objects have collision data. This collision data will almost always be of the type NiCollisionData.

Using NiCollisionData, a user may specify the method of collision, how the collision tree is to propagate the collision processing, the collision callback handler, and the associated callback data.

Whenever the callback is called, as a result of a detected collision, a pointer to the callback data is passed back to the application so that the application may store information about intersections that it desires. Typically an application will use callback data storage for information such as references to the objects involved in the collision, time of the collision, the location of intersection, and the collision normals. The callback data is then available to be processed later in the application.

See NiCollisionData for more information.
