More Ideas for 2D Collision

Here are further refinements to the general idea of 2D collision detection:

  • A second pick on the actual terrain may be done to obtain the instantaneous height above the terrain for the character. However, this pick need not be performed every time the character moves if the application knows the terrain is relatively flat in a particular area (e.g., interiors).
  • The terrain geometry should be a separate portion of the scene graph, for performance efficiency.
  • If the terrain polygon count is high, then the terrain may be partitioned into different "pickable" sets, with run-time culling for picking operations requiring terrain.
  • For an indoor level with fairly regular geometric objects, it is possible to set up your own collision detection system as a 2D+1D system. If the character must move from one room to another by passing through a doorway, the height of the capsule is compared to the height of the doorway, hence the 1D portion of the system.
