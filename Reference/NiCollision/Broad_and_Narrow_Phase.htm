Broad and Narrow Phase Collision Detection

For collision detection, a "narrow phase" signifies the process of determining whether or not two given objects collide, by performing the appropriate collision intersection tests. On the other hand, a "broad phase" component of a collision detection system deals with whether or not two given objects should be tested against each other at all. For example, if two objects are static and unmovable, then there is no reason to perform collision detection between those two objects.

The Gamebryo collision detection system provides numerous solutions for both phases. For example, the collision detection system provides numerous "broad phase" capabilities, such as collision groups, propagation modes, scene graph culling, ABVs and other collision proxies, and special processing for colliders vs. collidees. Examples of "narrow phase" capabilities include much of the basic low-level math capabilities, such as squared-distance calculations, intersection calculations, and so on.
