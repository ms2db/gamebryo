Minimize the Number of Collidees

Minimizing the number of collidees is another important consideration for optimal performance, particularly for collidees that never move relative to each other. If you create a single spatially hierarchical tree of collidees to add to the NiCollisionGroup with culling ABVs defined at each of the grouping nodes, collision testing is highly optimized, since a group of collidees may be culled in a single test.
