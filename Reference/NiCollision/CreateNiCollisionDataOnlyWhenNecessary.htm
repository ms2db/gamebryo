Create NiCollisionData Only When Necessary

When an object is added to a NiCollisionGroup as a collidee or collider, the bCreateCollisionData parameter should be set to false wherever it makes sense to do so. This step can be done when loading NIF files.

Since collision data will be created at every level in the hierarchy where the bCreateCollisionData is set to true, memory and processing time will be wasted needlessly in situations where collision detection is not required on every object.
