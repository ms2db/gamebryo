NiTexture

: public NiObjectNet

<NiTexture.h>

Description

NiTexture is an abstract base class that represents texture image data. Any subclass of this class may be used as a part of a texturing operation. This class does not define how the texture data is stored, updated or created. This behavior is left up to the subclasses to define.

Constructors

There are no supported constructors, as this class is an abstract base class and cannot be instantiated.

Static Member Functions

static bool GetMipmapByDefault()

static void SetMipmapByDefault(bool bMipmap)

Set and get the global mipmapping flag. When NiTexture subclasses are created with format preferences that specify MIP_DEFAULT as the preferred mipmapping mode, this flag is consulted. If it is true, then the texture will be mipmapped. If it is false, the texture will not be mipmapped. In this way, the mipmapping behavior of the entire set of textures as a group may be determined by a single, global flag. Note that if a texture sets a mipmap flag of YES or NO, then the global flag is overridden by the local preference. Note also that even though the default value for this flag is false, NiApplication sets this value to true.

Supported Member Functions

virtual unsigned int GetWidth() const = 0

virtual unsigned int GetHeight() const = 0

These functions virtual overrides in subclasses return the width and height of the texture, or 0 if the value cannot be computed at the time of the call. See the NiTexture subclasses for details.

const FormatPrefs& GetFormatPreferences() const

Returns the format preferences that were passed in to the creation function. These preferences are a request to the renderer to use a format the most closely matches the settings, and may have been ignored. Note that there is no related "Set" function, as this value is bound at creation time (when the texture is preloaded to the renderers), and as such cannot be changed once the object is created. See the section below for a discussion of this class.
