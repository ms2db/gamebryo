NiSyncArgs

: public NiMemObject

<NiSyncArgs.h>

Description

NiSyncArgs objects are used to hold data about parallel execution synchronization points. The only member variables are m_uiSubmitPoint and m_uiCompletePoint, both of type NiSyncArgs::SyncPoint (defined as type NiUInt16), and both public.

The following pre-defined sync points are available in the enumeration NiSyncArgs::SyncPoints:

  • SYNC_ANY = 0x8000: Represents any synchronization point. Typically, this is interpreted as "don't care" by functions that base their behavior on sync points. For example, the NiMeshModifier::CompleteTasks function will complete all outstanding modifier tasks if the m_uiCompletePoint of its NiSyncArgs argument is SYNC_ANY.
  • SYNC_UPDATE = 0x8010: Synchronize when an object is updated.
  • SYNC_POST_UPDATE = 0x8020: Synchronize when all of the objects in a scene graph have been updated.
  • SYNC_VISIBLE = 0x8030: Synchronize when an object is determined to be potentially visible.
  • SYNC_RENDER = 0x8040: Synchronize when an object is rendered.
  • SYNC_PHYSICS_SIMULATE = 0x8050: Synchronize when a physics simulation step begins.
  • SYNC_PHYSICS_COMPLETED = 0x8060: Synchronize when a physics simulation step has completed.
  • SYNC_REFLECTIONS = 0x8070: Syncronize after all data necessary to calculate reflections is ready.

When defining new SyncPoints for your application, do not use values above 0x7FFF - they are reserved for Gamebryo internal use.
