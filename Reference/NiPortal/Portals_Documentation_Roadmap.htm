Portals Documentation Roadmap

The "Portal Basics" chapter describes the use of the Gamebryo portals or "room-to-room visibility system". The NiPortal library of the Gamebryo game engine is the implementation of the portal visibility system. The remaining sections in this chapter describe the portal visibility system at a high level including what such a system is designed to do, what it cannot do, what the components are, and how they are assembled into an efficient system. These sections are designed mainly for developers, but artists should read them as well, to gain an understanding of how the objects defined by the portals system interact.

The chapter "Portal Scene Strategies and Performance" describes several optimization techniques and tricks that an application developer may use to increase the performance of games using portal-based scene graphs.

The chapter "Portals Class Reference" is the reference manual, describing all of the classes defined by the portals module.

Naturally, 3D modeling packages will benefit from tools that assist the artist in creating portal-enabled levels. Gamebryo has a tool called the Portal Converter whose functionality is exposed by plug-ins for 3ds max and Maya. The tool plug-ins allow artists to create these levels and walk through them (to see how effectively the portals cull the dataset) without having to leave their modeling tool. The converter documentation also discusses dataset decomposition (into the component parts required for portals) from a more artistic point of view. The documentation is written for artists familiar with 3ds max, but developers should also consider reading it to understand common methods of breaking datasets into portal-enabled levels. Although Maya exposes functionality for the same converter, the user interface is slightly different. The details for this modeling package are located in Portals with Maya.
