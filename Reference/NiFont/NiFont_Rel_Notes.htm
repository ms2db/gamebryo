Gamebryo 2.6.0

  • The NIFont class added getters to permit run-time discovery of the amount of atlas pad used in an external TGA atlas.

Gamebryo 2.5.0

  • NiOldString2D has been removed from the engine.
  • Ni2DString now uses NiMeshScreenElements instead of the (deprecated) NiScreenElements.
  • The library now uses the NiFile::GetFile method for loading assets.

Gamebryo 2.3.0

  • Added the ability to display 2-byte Unicode characters (UTC-2) through the Gamebryo NiFont library. Other strings in Gamebryo, such as file paths, object names, etc. will remain in ASCII.
  • New class Ni2DString creates 2D NiScreenElements geometry that will display a text string using a Unicode or ASCII font defined in an NiFont instance. Additionally, Ni2DString enables you to dynamically scale glyphs to any reasonable point size. By contrast, the deprecated class NiString2D (now named NiOldString2D) creates 2D NiScreenTexture geometry that will display a text string using an ASCII font defined in an NiFont object, but which may not be scaled. (If you prefer to continue to use the NiOldString2D class (formerly named NiString2D) in your existing application, all you need to do is rename NiString2D references within the application to NiOldString2D).
  • Improved memory allocation behavior within the font library.
  • Added additional helper methods to find the text extents for part of a string.
  • Removed unused NiFont color variable.
  • NiFont::GetCharUVPair returns ASCII char 127 for characters less than ASCII 32.

Known Issues and Limitations

  • NiFontString::SplitString can write outside the bounds of an array if any line contains more than 260 characters.
