NiQuaternionEvaluator

: public NiKeyBasedEvaluator

<NiQuaternionEvaluator.h>

Description

NiQuaternionEvaluator uses NiRotKey objects to animate an NiQuaternion value over time. This NiKeyBasedEvaluator only supports one key channel. A posed (i.e. constant) channel is stored using the key array. As a result, such posed evaluators are handled more efficiently as an NiConstQuaternionEvaluator.

Evaluators are primarily intended for sequence-based animation. Refer to the Animation Sequences section in the Introduction to Animation for more details.

Supported Member Functions

NiRotData* GetQuaternionData() const

void SetQuaternionData(NiRotData* pkRotData)

These functions get and set the NiRotData for this NiQuaternionEvaluator.

NiRotKey* GetKeys(unsigned int& uiNumKeys, NiRotKey::KeyType& eType, unsigned char& ucSize) const

This function returns the keys currently held by this evaluator. The number of keys, key type, and key stride are returned in uiNumKeys, eType,and ucSize.

void ReplaceKeys(NiRotKey* pkKeys, unsigned int uiNumKeys, NiRotKey::KeyType eType)

This function specifies the key data to store. Note that NiQuaternionEvaluator does not copy the NiRotKeys -- it assumes ownership of the keys and deletes any previous keys.

void SetKeys(NiRotKey* pkKeys, unsigned int uiNumKeys, NiRotKey::KeyType eType)

This function specifies the key data to store. Note that the NiQuaternionEvaluator does not copy the NiRotKeys -- it simply overwrites the pointers to the old keys. This action can easily cause memory leaks, so calls to SetKeys should generally be avoided in favor of ReplaceKeys.
