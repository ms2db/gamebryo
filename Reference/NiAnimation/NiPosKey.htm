NiPosKey

: public NiAnimationKey

Description

NiPosKey is the abstract base class from which all other position keys are derived. It provides the basic data access routines and interpolation code. This class should be considered pure virtual and never instanced directly.

Constructors

Because NiPosKey is an abstract base class, it has no public constructors.

Static Member Functions

static NiPoint3 GenInterp(float fTime, NiPosKey* pkKeys, KeyType eType, unsigned int uiNumKeys, unsigned int& uiLastIdx, unsigned char ucSize)

GenInterp will interpolate the array of NiPosKeys pkKeys at time fTime and return the result. The number of keys in the array is given by uiNumKeys, and the interpolation method expected by the keys is given by eType. The index value of the last interpolation is given in uiLastIdx; this acts as a hint to the function on where to restart the interpolation process, and can significantly increase performance. The index value of the current interpolation is returned in uiLastIdx for use during the next interpolation. The size of each individual key in the key array is specified by the ucSize variable.

static void GenCubicCoefs(float fTime, NiPosKey* pkKeys, KeyType eType, unsigned int uiNumKeys, unsigned int& uiLastIdx, unsigned char ucSize, float& fTime0, float& fTime1, NiPoint3& kValue0, NiPoint3& kOutTangent0, NiPoint3& kA0, NiPoint3& kB0)

GenCubicCoefs will return the cubic coefficients for the animation curve between the keys in the array of pkKeys that bound the specified fTime. The number of keys in the array is given by uiNumKeys, and the interpolation method expected by the keys is given by eType. The index value of the last curve segment that was used is given in uiLastIdx; this acts as a hint to the function on where to restart the bounding key search, and can significantly increase performance. The index value of the current curve segment is returned in uiLastIdx for use during the next search. The size of each individual key in the key array is specified by the ucSize variable. The values returned in fTime0 and fTime1 correspond to the times of the bounding keys. The cubic coefficients for the matching curve segment are returned via kValue0, kOutTangent0, kA0, and kB0. This cubic equation is of the form kValue0 + (kOutTangent0 + (kA0 + kB0 * fT) * fT) * fT where fT is the normalized time along the curve segment. This function is used internally for sequence animation and normally should not be called separately by an application.
