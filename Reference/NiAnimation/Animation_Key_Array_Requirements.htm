Animation Key Array Requirements

The animation system utilizes the GenInterp function (see NiFloatKey, NiRotKey, NiPosKey, NiColorKey, etc.) to sample the key array data at the specified time. This function requires that:

  1. The time passed into GenInterp be within the range of the key times.
  2. Key arrays have at least one key.
  3. Key times must be strictly increasing. In particular, no two keys may have the same time.

The NiAnimation system expects keys at the start and end times of a controller. For example, say you have an NiTransformController with a BeginKeyTime of 0.0f, and an EndKeyTime of 3.0f. For each type of key (i.e. rotation, position, and scale) in the attached NiInterpolator, if there are any keys of that type, there must be a key at time 0.0f and a key at time 3.0f. The exporters and streaming system automatically guarantee that the controllers have keys at the start and end times. However, care should be taken when calling NiTimeController::SetBeginKeyTime and SetEndKeyTime, or when modifying key data.
