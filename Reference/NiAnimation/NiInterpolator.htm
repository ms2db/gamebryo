NiInterpolator

: public NiObject

<NiInterpolator.h>

Description

NiInterpolator is an abstract base class for all objects that convert a time and an object into one of the supported data formats. The supported data formats are bool, float, NiQuaternion, NiPoint3, NiColorA, and NiQuatTransform.

Interpolators perform a similar role as evaluators; however, evaluators are more efficient for sequence animation. Interpolators are automatically converted into equivalent evaluators when sequences are generated with the Gamebryo art export tools.

Constructors

This is an abstract base class and as such has no public constructors.

These functions return the value of an NiInterpolator after it has been updated. If this value is invalid, the function returns false. In the base implementation, all values are invalid and return false. fTime specifies the sample time in the local timeline of the interpolator as opposed to the application's global animation time.

virtual bool IsBoolValueSupported() const

virtual bool IsFloatValueSupported() const

virtual bool IsQuaternionValueSupported() const

virtual bool IsPoint3ValueSupported() const

virtual bool IsColorAValueSupported() const

virtual bool IsTransformValueSupported() const

These functions return whether or not the NiInterpolator supports one of the data formats. NiInterpolators may support one or more data formats. In the base implementation, all values are unsupported and return false.

virtual void Collapse()

This function will try to reduce the memory footprint of the NiInterpolator by invoking operations such as animation key reduction. The exact operations performed are dependent on the derived class implementation.

void ForceNextUpdate()

Causes the next call to Update to force re-computation of the interpolated value. Some interpolators cache the last time they were updated, and simply return their cached value when updated with the same time more than one time in a row. Calling ForceNextUpdate will override this behavior and force the full update to occur. This is useful if you have changed animation data or animation keys and are updating the interpolator with the same time as its last update to ensure that the value returned from Update is accurate.
