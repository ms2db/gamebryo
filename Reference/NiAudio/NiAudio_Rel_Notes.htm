Gamebryo 2.5

  • Upgraded to Miles 7.2a.
  • GetPlayPosition() was reporting its position in milliseconds instead of the audio data offset for non-streamed audio sources.
  • Play() was not using the Miles Audio resume call required to start audio playback from the middle of a sound source for non-streamed audio sources.
  • Updated the NiAudioSource and NiAudioListener interfaces to provide overloads for UpdateDownwardPass(), UpdatedSelectedDownwardPass(), and UpdatedRigidDownwardPass(). Additionally the method UpdateAudioData() has been added. All audio data should now be updated inside of UpdateAudioData(). This addresses an issue where Update() had to be explicitly called on each NiAudioSource & NiAudioListener object. Calling Update() on a root of a scene graph containing NiAudioSource and/or NiAudioListener objects will now properly update the audio data inside those objects. The NiMilesAudioSource and NiMilesAudioListnener classes have updated to match these changes.

Gamebryo 2.2.1

  • Upgraded to Miles 7.0s.
  • Removed class NiProviderInfo from NiMilesAudio as it no longer necessary to enumerate possible providers.
  • Added Miles support for Xbox 360.

Gamebryo 2.0.0

  • Upgraded to the current version of Miles 6.6h.
  • Changed the function name NiAudioSystem::Click to NiAudioSystem::Update, as the concept of "clicking" no longer exists in GB2.0
