Controller Extractor

Type: EXPORT

Version: 2.6

<ControllerExtractor.dll>

Description

This plug-in extracts NiInterpolator objects attached to NiInterpControllers in the scene graph, converts them to equivalent NiEvaluator objects using NiInterpolatorConverter, and saves the evaluators in NiSequenceData objects that reside in KF files. Animation sequences must be specified in the art package for this plug-in to execute successfully. See the 3ds max and Maya plug-in documentation for more information on specifying animation sequences.

The plug-in relies heavily on the functionality provided by the NiControllerExtractor library.

See the Known Issues and Limitations for additional information.

Options

Controller Extractor Options

Sequences Per KF File

There are two ways of extracting sequences: all the sequences can be extracted to a single KF file or each sequence can be extracted to its own KF file. If each sequence is extracted to its own KF file, the KF File Naming Convention options are available. Otherwise, they are grayed out and the KF file has the same name as the NIF file.

KF File Naming Convention

If each sequence is being extracted to its own KF file, these options determine the naming convention for each of those KF files. A preview of the KF file name is shown below these options. The exported file name is the name of the NIF file for this character. The character name is the name of the actor root node as specified in the art package. The layer name will only be written out for layered sequences. There is also the option to use an upper or lower case file extension.

Example File Name

This box shows a preview of the exported KF file name.

Compression

Sometimes, compressing animation keys in the sequence can result in a larger file size, especially if the animation keys are not easily fitted to a curve. This option will disable compression if the compressed sequence will be larger.

KFM File

You have the option of creating and saving out a KFM file for the character that includes information about the extracted NIF and KF files and sequences. Transitions are set up between every sequence as they would be in the Gamebryo Animation Tool. Note that when a new KFM file is created for the character, all the animation set-up work done in Animation Tool is lost, and must be completely re-done. In general, a KFM file should only be created once for a character.

KFM files support only one character. If you are exporting from a file that contains more than one actor root, you have the option here of appending the character name to the KFM file name so that exporting KFM files for additional characters later will not overwrite the KFM file for this character.

Checking the "Create KFM File" check box will enable the "Default Transition Settings" section.

Default Transition Settings

When generating a KFM file upon export, transitions are set up between every sequence. If morph tags are present for two animations, a default synchronized transition is used. Otherwise, a default non-synchronized transition is used. The default transition type and duration for these synchronized and non-synchronized transitions can be set here.

Excluded Controllers

The Controller Extractor plug-in provides the option to exclude certain types of controllers from being extracted and included in sequences.

Checking the "NiFlipController" check box will prevent any NiFlipController objects from being included in any sequences. NiFlipController objects are used when performing flipbook texture animation using IFL files. Often, the looping frequency of an IFL file is short and is intended to loop indefinitely and not be synchronized to any particular sequence. However, if its looping frequency is less than the particular sequence, clamping may occur upon export. To avoid this, NiFlipController objects can be excluded from sequences, causing them to always be active no matter which sequences are active.

Checking the "NiPSResetOnLoopCtlr" check box causes any NiPSResetOnLoopCtlr objects to be removed from particle systems. These controllers cause particle systems in sequences to loop independently of the sequence looping values, which may be undesirable. This option is checked by default and is effective when the KFM Viewer tool plug-in is used to preview animations. When looping of particle systems with reset on each loop is required, set the looping period of the particle system to match the sequence length for the sequence in which the particle system is used, and do not check the "NiPSResetOnLoopCtlr" box.

A list of custom controller types can also be supplied that will be excluded during sequence extraction. Checking the "Custom controller types" check box will enable the text box, in which can be entered a semicolon-delimited list of controller names that should be excluded. An example string that excludes NiAlphaController and NiTextureTransformController objects would look like this: "NiAlphaController;NiTextureTransformController;".
