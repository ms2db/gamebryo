Source Vertex Color Ranges in DX9

Applications that supply vertex colors as part of a model must take care to ensure that the vertex colors are in the range [0.0, 1.0], as the NiDX9Renderer does not clamp these values. Values outside of this range will tend to "wrap", and will not "saturate" the way some applications might expect. This is a performance optimization, as applications that generate colors that are out of range can simply clamp in the application code. When using D3D lighting, lighting values outside of the above range may occur this is fine. The restriction applies only to vertex colors specified in engine-level geometry objects.
