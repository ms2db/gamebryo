Hardware Texture Stage Usage in DX9

The legacy texture pipeline in the DX9 renderer detects the number of available texture stages per pass from the hardware and attempts to use them all to maximize the amount of work done per pass. While this will usually result in an efficient use of texture stages, there are circumstances where a combination of texture effects will take up more passes than are necessary. In cases such as these, the application can perform its own optimizations by using an NiShader to override the legacy pipeline. Details on this can be found in the NiShader documentation.
