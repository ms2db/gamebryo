NiD3DDefines.h

This is included by all shader system files. It includes the platform-specific defines that are required for the system to function. To allow for platform-independent operation, several values are typedef'd to allow for shader system files to be shared across multiple renderers. This includes the following:

The first set of typedefs define the mapping of D3D types to a common name:

// D3D Mappings

typedef LPDIRECT3DRESOURCE9 D3DResourcePtr;

typedef LPDIRECT3DDEVICE9 D3DDevicePtr;

typedef LPDIRECT3DSURFACE9 D3DSurfacePtr;

typedef LPDIRECT3DBASETEXTURE9 D3DBaseTexturePtr;

typedef LPDIRECT3DTEXTURE9 D3DTexturePtr;

typedef LPDIRECT3DVERTEXBUFFER9 D3DVertexBufferPtr;

typedef LPDIRECT3DINDEXBUFFER9 D3DIndexBufferPtr;

The next set of typedefs define the mapping of Gamebryo types to a common name:

// Gamebryo Mappings

typedef NiDX9Renderer NiD3DRenderer;

typedef NiDX9RenderState NiD3DRenderState;

typedef NiDX9VertexBufferManager NiD3DVBManager;

typedef NiDX9PaletteData NiD3DPaletteData;

typedef NiDX9TextureData NiD3DTextureData;

typedef NiDX9SourceTextureData NiD3DSourceTextureData;

typedef NiDX9MaterialManager NiD3DMaterialManager;

Vertex and pixel shader handles, as well as declaration pointers are typedef'd after these:

// Shader 'handle'

typedef LPDIRECT3DVERTEXDECLARATION9 NiD3DVertexDeclaration;

typedef LPDIRECT3DVERTEXSHADER9 NiD3DVertexShaderHandle;

typedef LPDIRECT3DPIXELSHADER9 NiD3DPixelShaderHandle;

The last section of this file defined D3DVECTOR4 for platforms on which it is not defined. An aligned matrix type, D3DXALIGNEDMATRIX, is also defined to allow for D3DX math routines to function at optimal levels.
