Changing the Display Parameters of an NiDX9Renderer

Applications can change the display parameters, including screen size and depth, using the function Recreate.

RecreateStatus Recreate(unsigned int uiWidth, unsigned int uiHeight, unsigned int uiUseFlags, NiWindowRef kWndDevice = 0, FrameBufferFormat eFBFormat = FBFMT_UNKNOWN, DepthStencilFormat eDSFormat = DSFMT_UNKNOWN, PresentationInterval ePresentationInterval = PRESENT_INTERVAL_ONE, SwapEffect eSwapEffect = SWAPEFFECT_DEFAULT, FramebufferMode eFBMode = FBMODE_DEFAULT, unsigned int uiBackBufferCount = 1, unsigned int uiRefreshRate = REFRESHRATE_DEFAULT)

Recreate takes many of the same parameters as the Create function and uses them to adjust the display settings before making the DX9 call Reset. The NiDX9Renderer will automatically take care of destroying and recreating all the resources that this call requires. If the reset was not successful, the function attempts to restore the display parameters to their last known successful values. Recreate returns an enumeration that can be one of three values:  RECREATESTATUS_FAILED, which indicates that Reset has failed and the renderer was unable to restore the previous device settings; RECREATESTATUS_RESTORED, which indicates that the Reset call failed but the renderer did manage to restore the previous device settings; and RECREATESTATUS_OK, which indicates that Reset was successful.

If the application only intends to resize the window, it can use a simpler version of the Recreate function.

RecreateStatus Recreate(unsigned int uiWidth, unsigned int uiHeight)

This version will use the same display settings as are currently active, except for the height and width values.

Finally, there is a version of Recreate that takes no parameters, and is used to simply reset the device.

bool Recreate()

Most applications would not need to use this call, since it would be most useful for resetting a lost device, which the NiDX9Renderer handles automatically.

In any case, after a successful call to Recreate, the textures and geometry data that need to be reloaded to the device are handled automatically. Note that if the fullscreen resolution has been increased, there may not be sufficient VRAM to create NiRenderedTextures of the same size and number as with the original renderer.
