NiVersion

<NiVersion.h>

Description

This is a header file that defines a set of potentially useful macros related to the current Gamebryo version, NIF version, and build date.

Macros

GAMEBRYO_SDK_VERSION

This macro combines the major, minor, and patch version numbers to create an unsigned integer. It will increase monotonically as the SDK version numbers grow. (Note that the build version number can be larger than an 8-bit value, so it is not included in the GAMEBRYO_SDK_VERSION.)

GAMEBRYO_SDK_VERSION_STRING

This is a string version of the above GAMEBRYO_SDK_VERSION, but with the build version number included. It resolves to something similar to "3.0.0.0", representing "major-version.minor-version.patch-version.build-version".

NIF_MAJOR_VERSION

NIF_MINOR_VERSION

NIF_PATCH_VERSION

NIF_INTERNAL_VERSION

These four values can be used to get the current NIF version. The following statement can be used to combine them into a single unsigned int, suitable for comparing with other versions:

unsigned int uiVersion = NiStream::GetVersion(NIF_MAJOR_VERSION, NIF_MINOR_VERSION,
        NIF_PATCH_VERSION, NIF_INTERNAL_VERSION);

GAMEBRYO_NIF_VERSION_STRING

Gamebryo's NIF version is not necessarily the same as the SDK version. This is the version that all files streamed out with NiStream will have. The NIF version string is in the format "x.y.z.w", where x, y, z, and w are all unsigned integers. If it is necessary to compare NIF versions, the utility function NiStream::GetVersionFromString will turn a NIF version string into an unsigned int.

GAMEBRYO_BUILD_DATE

Similar to GAMEBRYO_SDK_VERSION, this macro resolves to an unsigned int that represents that date. This value will increase monotonically as the date moves forward in time.

GAMEBRYO_BUILD_DATE_STRING

This macro resolves to a string version of the GAMEBRYO_BUILD_DATE. It uses "m-d-YYYY" formatting.

NI_DLL_SUFFIX

DLLs and their associated libraries in Gamebryo are named in the form (LibraryName)(GamebryoVersion)(Compiler)(Debug/Release).(lib/DLL). For example, NiAnimation32VC80D.lib would be the library for the Debug DLL version of NiAnimation built on Gamebryo 3.2 using Visual Studio 2005. The NI_DLL_SUFFIX macro resolves to the suffix of these library and DLL names. In the above example, it would resolve to "32VC80D". This is particularly useful if you are using #pragma to link in DLL libraries in code.
