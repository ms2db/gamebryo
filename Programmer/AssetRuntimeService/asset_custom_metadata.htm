Adding New Asset Types to Metadata

Finding the appropriate file system path for loading an asset at runtime is done through the Asset Locator Service (efd::AssetLocatorService). Any new asset loaders that you add will also need to be supported by the Asset Locator Service, which means you must configure the Asset Controller to discover these assets and create metadata for them.

Adding a new asset type to the Asset Controller is very easy. You just need to add a unique file name extension and asset MIME type value to a configuration file in <Asset Web Root Directory>/asset-web-config/. The configuration system looks for any files in this folder with "[MimeTagger]" sections and merges them to create a list of all asset types that are discoverable by the system. Just add your extension to an "ini" file in this folder and create a type using the format of "media-type"/"asset-type". Use the existing type definitions in <Asset Web Root Directory>/asset-web-config/MimeTagger.ini as a guide and make sure that your MIME type is unique.

An example of an "CustomAssets.ini" file that would add a file with ".myextension" can be found below:

[MimeTagger]
myExtension=myMediaType/myExtension-asset

[MimeTagger.RunTime]
=myExtension

If your new asset type will be assigned as an Asset Property value for entities when using Toolbench, then you should also enable logical asset id generation for these assets. The configuration system looks for any files in <Asset Web Root Directory>/asset-web-config/ folder with "[LogicalIdTagger.Assets]" sections and merges them to create a list of all asset types that use logical id's.  Use the configuration found in <Asset Web Root Directory>/asset-web-config/LogicalIdTagger.ini as a guide.

An example of an "CustomAssets.ini" file that would add a file with ".myextension" and register it as a logical id file type can be found below:

[MimeTagger]
myExtension=myMediaType/myExtension-asset

[MimeTagger.RunTime]
=myExtension

[LogicalIdTagger.Assets]
=myExtension
