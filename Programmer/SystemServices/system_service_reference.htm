System service reference

System Service Description
Foundation Services
AssetConfigService Manages the current state of the Asset Locator and allows configuration changes to be applied via .ini file, command-line, or command messages.
EnumManager Is used to load data driven enumeration files that can map between string names and numerical values.
AssetLocatorService Translates logical asset names into physical asset locations for loading.
AssetServer Provides the functionality for servicing asset lookup and fetch requests from a remote client.
ConfigManager Provides applications access to a hierarchical configuration of name/value pairs drawn from multiple sources.
MessageService Handles sending messages and registering callbacks for message delivery.
ReloadManager Central manager for forwarding asset change notification events to interested parties.
PlatformService PS3, Win32 Encapsulates platform specific methods and message handlers.
LogService Responsible for handling log configuration calls, messages, tracking log destinations, and log files.
Foundation Network Services
NetService Connects to and communicates with the ChannelManagerService to handle all the network communications.
BridgeService Bridges two different types of NetLibs potentially talking different protocols or transports.
ChannelManagerService Handles messages from connected NetServices and routes the messages to all other connected NetServices by subscriptions to categories.
NameResolutionService Resolves names to IP addresses and other information.
Game Framework Services
NotificationService Monitors changes to entities and notifies other entities and services about those changes.
EntityLoaderService Listens to and responds to Entity Load and Unload requests.
EntityManager A cache of all active entities that belong to the local scheduler.
FlatModelManager Responsible for loading and parsing Flat Model files.
RapidIterationSystemService Multiplexer to manage messages required during rapid iteration development scenarios.
Scheduler Manages 'owned' entities and feeds cycles to the behavior models of the Entities that need it.
Game Framework PhysX Services
SceneService Handles adding, removing, modifying and simulating PhysX scenes.
TriggerService Used to register trigger events of interest and send notifications when they occur.
Core Runtime Services
CameraService Handles initialization and update of NiCamera objects using the properties in a Camera model attached to an entity.
LightService Manages light sources and the lists of entities which they affect.
PickService Issues pick requests on the scene graphs contained in the SceneGraphService.
RenderService Manages renderable objects and the Core Runtime renderer.
SceneGraphService Manages all the Core Runtime scene graphs that are loaded in the system.
ShaderService Manages the shaders loaded in the systems.
InputService Handles user input by managing action maps, events, actions, processing input each tick and sending out messages when events occur.
Core Runtime PhysX Services
PropService Handles adding, removing, modifying and simulating PhysX content saved in NIF files.
Game Module Services
AnimationService Provides an interface and a set of script built-ins for working with the Core Runtime animation system.
TerrainService Provides an interface and a set of script built-ins for working with the Core Runtime terrain system.
PhysXTerrainService Provides an interface and a set of script built-ins for working with the PhysX features of the Core Runtime terrain system.
RagdollService Provides an interface and a set of script built-ins for working with PhysX rag-dolls.
Game Module Visualizer Services
PropertyVisualizationService Manages visualizers for entities that specify visualizers in their mix-in extra data.
ProxyService Handles the registration and management of 2D proxy textures and 3D proxy geometry.
Game Module Tool Services
VisualTrackerService Provides a graphical display for tracking performance over time or other statistics.
GizmoService Manages the gizmos used to position, orient, and otherwise graphically modify objects.
GridService Responsible for constructing and managing grid meshes used to help visualize scale in scenes.
InteractionService Handles user interaction for the tool services.
SelectionService Provides methods for and manages selection of objects.
ToolCameraService Manages the camera for Tool Services.
ToolSceneGraphService Manages the scene graph for Tool Services.
ToolSnapService Maintains a list of snap points and snaps objects to those points as requested.
Other Tool Services
RemoteLogControl Used to allow remote processes to change the log levels, destinations, etc. of the Logger.
AssetMonitorService Monitors the status of Assets stored in an asset web
IconAppService Service to handle platform specific aspects of system window decorations.
