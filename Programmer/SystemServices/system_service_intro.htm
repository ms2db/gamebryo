System Services: Introduction

System services in combination with the service manager make up the subsystem that is responsible for sharing compute cycles across various logical areas of the Game Framework. System services implement a well known interface that the service manager uses to move each service through the different stages of the service's life cycle.

Many of the fundamental Game Framework subsystems are implemented as system services. Subsystems like the scheduler, log service, and message service are all implemented as system services and share cycles via the service manager.

Each system service is responsible for behaving asynchronously. An ill behaving system service can starve the remaining services if it is not designed to accommodate this asynchronous approach.
