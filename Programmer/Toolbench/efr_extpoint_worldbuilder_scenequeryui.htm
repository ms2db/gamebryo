SceneQueryUI (Extension-Point)

Emergent.Toolbench.WorldBuilder.UI.SceneQueryUI

The SceneQueryUI extension point allows you to introduce new user interfaces to the SceneQueryDialog that should be asociated with a specific SceneQuery. There are two custom interfaces that can be provided to improve interaction and feedback with a SceneQuery. A header interface will be displayed at all times when a query is active while a content interface will only be displayed when the users expands the previously defined query.

<!-- New scene query UI -->
<extension point ="Emergent.Toolbench.Worldbuilder.UI.SceneQueryUI">
    <DataQueryUI ContentUserControl="CustomNameSpace.NewSceneQuery.NewQueryUIContent"
        HeaderUserControl="CustomNameSpace.NewQueryUIHeader"
        SceneQuery="New Query">
    </DataQueryUI>
</extension>

scenequeryui

Attribute Type Description
ContentUserControl String The unique identifier of a UserControl derived class to be used as the Query's content. This is optional however not defining it will display a "No parameters..." label
HeaderUserControl String The unique identifier of a UserControl derived class to be used as the Query's header. This is optional however not defining it will display a "No parameters..." label
SceneQuery String The name of a previously created SceneQuery the UI should be attached to. This must be defined.

The plug-in Emergent.Toolbench.WorldBuilder.Terrain.UI contains several standard queries to use as reference points for creating your own. Those queries provide the user with functionality allowing them to retrieve terrain specific data such as point information, material information or meta data information.
