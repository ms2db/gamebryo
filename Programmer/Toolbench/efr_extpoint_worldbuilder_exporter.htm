Exporter (Extension-Point)

Emergent.Toolbench.WorldBuilder.Exporter

The Exporter extension point allows you to customize how your block and layer files are exported to runtime formats.

<extension point="Emergent.Toolbench.WorldBuilder.Exporter">
  <exporter name="XmlBlockExporter"
              class="Emergent.Toolbench.WorldBuilder.XmlBlockExporter" />
</extension>

exporter

Attribute Type Description
name String The unique identifier for this exporter. This name will be the key that is used when matching the user setting for "World Builder\Run-time Exporter".
class String The class name of the object to be created. This object handles converting from the BlockDocument to the final runtime file formats.

Specifying Custom World Builder Exporters

Users may specify the type of exporter used when exporting by adjusting the "World Builder\Run-time Exporter" setting in the settings dialog for the block document. The value of this setting must match the name specified in the extension for the custom exporter.

Creating Custom World Builder Exporters

It is often valuable to construct alternative exported formats for blocks and layers in a game. Some example custom exporters might be: binary formats, write a separate file for each platform, write a separate file per layer, etc. The default exporter, XmlBlockExporter can be found in "Source\Tools\Toolbench\Plugins\Emergent.Toolbench.WorldBuilder\Exporters\XmlBlockExporter.cs". If you are using the same general xml format as xblock files, a convenience class has been provided at "Source\Tools\Toolbench\Plugins\Emergent.Toolbench.WorldBuilder\Exporters\XmlEntityExporter.cs", which takes a hashtable of entities by their id and converts to a streaming XML element. It is possible to take these XML elements and embed them in other XML files.
