Adding a Custom Import Library

The following topic will assist developers in adding a custom import library that can be easily imported into a Game Solution. This topic is primarily targeted towards partner developers or internal teams of a studio that have a need to share entity modeling projects with multiple active games under development.

Basic Steps

  • Create a custom GSLIB file that references your import library and any asset web configuration files needed and place it in the root of your asset web.
  • Install a master copy of your asset web into the user's directory. This can be done via network share or installer.
  • Install a custom plug-in into the Toolbench directory of the user's machine that points to your asset web. See below for an example.
  • In your documentation, prompt the user to use the Integration Wizard and have them choose to install your custom import library.
  • Users should now add your library as a reference to any block or entity modeling projects they desire.

Example Import Library Only Plugin

Install to "%EMERGENT_PATH%\SDK\Win32\Bin\VC90\Shipping\Plugins\" with a folder named MyCustomPlugin. The following would be the contents of the "plugin.xml" file at the root of this folder.

This plug-in tells Toolbench to look for .GSLIB files in "%CUSTOM_PATH%\MyCustomRoot". Note that the environment variable,"CUSTOM_PATH", is automatically expanded.

<?xml version="1.0" encoding="utf-8" ?>
<plugin
  xmlns="http://www.emergent.net/schemas/plugin-base-schema.xsd"
  name="Custom Test Plugin"
  id="CustomTestPlugin.Test"
  version="1.0.0.0"
  provider="CustomProvider">

    <extension point="Emergent.Toolbench.GameSolution.ImportLocation">
        <location path="%CUSTOM_PATH%\MyCustomRoot"/>
    </extension>

  <requires>
    <import plugin="Emergent.Toolbench.GameSolution"/>
  </requires>
    
</plugin>
