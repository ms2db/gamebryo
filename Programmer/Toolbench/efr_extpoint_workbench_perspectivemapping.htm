PerspectiveMapping (Extension-Point)

Emergent.Toolbench.Workbench.PerspectiveMapping

The perspective mapping allows you associate actionsets which are composed of menu and toolbar items and views with a particular perspective.

<extension point="Emergent.Toolbench.Workbench.PerspectiveMapping">
  <mapping perspectiveId="Emergent.ImagePlugin.ImageEditingPerspective">
    <view id="Emergent.ImagePlugin.ImageView"/>
    <view id="Emergent.ImagePlugin.ImagePropertiesView"/>
    <actionset id="Emergent.ImagePlugin.ActionSet"/>
  </mapping>
</extension>

mapping

Identifier Type Description
perspectiveId String The unique identifier for the perspective that you want to associate the views and actionsets with.

view

Identifier Type Description
id String The unique identifier used to refer to a previously defined View.
states String [optional] If the states attribute is set on a view, then only if one of the states is active will that view become visible, if the perspective it is mapped to is currently visible. You can set multiple states as triggers for a view by using delimiters to separate the states. (,;| ) are all valid delimiters for states.

actionset

Identifier Type Description
id String The unique identifier used to refer to a previously defined ActionSet.
states String [optional] If the states attribute is set on an actionset, then only if one of the states is active will that actionset become visible, if the perspective it is mapped to is currently visible. You can set multiple states as triggers for a view by using delimiters to separate the states. (,;| ) are all valid delimiters for states.
