ActionProcessor (Extension-Point)

Emergent.Toolbench.Actions.ActionProcessor

To intercept actions handled by the action system, users will need to create an extension for the action processor extension point. With it you can record any action performed by the system. This is very important for having things like a macro recorder or an undo/redo system. Note: Having an action processor will cause your plug-in to be loaded when the action system loads, which for all intents and purposes is when Toolbench loads.

<extension point="Emergent.Toolbench.Actions.ActionProcessor">
  <processor class="Emergent.Toolbench.Actions.UndoRedoActionProcessor"/>
  ...
</extension>

processor

Attribute Type Description
class String The full class name of the action processor. Action processors must implement the Emergent.Toolbench.Actions.IActionProcessor interface.
