Extension Point

To expose new functionality that your plug-in can implement for other plug-ins you will want to create an extension-point. An example of an extension point would be Workbench's ActionSet extension point. This extension point allows other plug-ins to expose menu items, toolbar items, and actions for Workbench to display to the user.

<extension-point id="Emergent.Toolbench.Workbench.ActionSet"
                 schema="Schemas\ExtPoint-ActionSet.xsd"/>

extension-point

Attribute Type Description
id String The unique identifier for the extension point. Plug-ins that implement this extension point in the form of an extension will have to use this identifier to identify which extension point they are implementing.
schema String [optional] The schema attribute is the relative path from your plug-ins directory to an xml schema file for this extension point.
While optional the schema is a highly encouraged attribute to add to your extension point. It gives Toolbench a way to validate an extension before your plug-in parses the data, which will help you debug problems faster and decreases the amount of error handling code your plug-ins extension point code will need to contain.
