Requires

The requires element allows Toolbench to load any plug-ins you are dependent upon before loading your plug-in. There are a few reasons why this isn't done by scanning the imported assemblies at load time for your plug-in assembly. Your plug-in might be a data only plug-in because no plug-in in Toolbench is required to have a corresponding assembly in which case it wouldn't have an assembly to scan. Similarly your plug-in may hold a dependency on a data only plug-in which would not appear in the import list.

To ensure you don't forget to import an assembly you depend on, when running Toolbench in debug mode, it will recursively trace your assembly's imported assemblies to see if you are importing an assembly for a plug-in that you have forgotten to add to your manifest. Not all dependent plug-ins need be referenced in the manifest directly, so long as a path to each dependency can be traced via some imported plug-in.

<requires>
    <import plugin="Emergent.Toolbench.TestRunner" version="1.0.0.0"/>
    ..
</requires>

import

An import tag can occur within a requires tag and is meant to indicate a plug-in dependency on another plug-in.

Attribute Type Description
plugin String The unique identifier for the plug-in you depend on.
version String [optional] The version of that plug-in you depend on.
