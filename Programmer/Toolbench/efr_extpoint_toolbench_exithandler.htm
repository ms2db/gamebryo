ExitHandler (Extension-Point)

Emergent.Toolbench.ExitHandler

After the Runnable application returns from the Run() method call, any plug-ins that are currently loaded that extend this extension point will have their exit handlers called. This is very important for plug-ins that don't necessarily just work in a UI environment and therefore may not need to shutdown when Workbench is closing, but may need to shutdown when Toolbench itself is closing.

Note: After a runnable extension has returned from its Run() call there is no way to cancel the shutdown of Toolbench.

<extension point="Emergent.Toolbench.ExitHandler">
  <class id="Emergent.Toolbench.Settings.ExitHandlerAction"/>
</extension>

class

Identifier Type Description
id String The id is the full class name of the exit handler class. The exit handler class must implement the Emergent.Toolbench.IExitToolbenchHandler interface.
