SettingsSet (Extension-Point)

Emergent.Toolbench.Settings.SettingsSet

The settings set extension point allows you to register settings with Toolbench so that users can configure their plug-ins without loading them.

<extension point="Emergent.Toolbench.Settings.SettingsSet">
  <settings>
      <setting id="Emergent.SamplePanels.SampleFloatSetting"
               scope="PerUser"
               property-type="Float"
               visible="true">
        <value>256.256</value>
        <display-name>My Sample Float Setting</display-name>
        <description>A custom float setting.</description>
      </setting>
      ...
  </settings>
</extension>

setting

Attribute Type Description
id String The unique identifier for the setting entry.
path String A path for the setting.
scope enum The scope of the setting determines how it will be saved and scoped for the users.
  • PerUser - Settings are saved on a per user basis.
  • PerContent - Settings are saved on a per file basis.
  • PerUserPerContent - Settings are specific to a document and user.
property-type String The property type is the name of the type for this setting. These types must be one defined in a Emergent.Toolbench.PropertyType extension.
content-type String [optional] The type of content supporting a PerContent or PerUserPerContent setting.
visible Boolean [optional] [default: true] The visible property controls if the setting should be visible to the user in a settings editor.
encrypt Boolean [optional] [default: false] The encrypt property controls whether the setting value is encrypted on save.
