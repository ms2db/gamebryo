Perspectives (Extension-Point)

Emergent.Toolbench.Workbench.PerspectiveMapping

The perspectives extension point allows you to create new perspectives. A perspective is a collection of views, menus and toolbar items that become visible based on their perspective type. For example a perspective with the ActiveContent type, and a content-type-id of ImageDocument will become visible when the active content type is an image document.

<extension point="Emergent.Toolbench.Workbench.Perspectives">
  <perspective name="Image Editing"
               id="Emergent.ImagePlugin.ImageEditingPerspective"
               type="ActiveContent"
               content-type-id="Emergent.ImagePlugin.ImageDocument"/>
</extension>

perspective

Attribute Type Description
id String The id is a unique identifier used to refer to this perspective.
name String The name of the perspective.
type enum The perspective type is used to determine when the perspective should be visible.
  • Global - Always visible to the user.
  • LiveContent - Only visible when a type of content is opened, but not necessarily focused.
  • ActiveContent - Only visible when a type of content has focus.
  • ActiveView - Only visible when a particular view type has focus.
icon String [optional] The path to the icon associated with this perspective. The path should be relative to the directory of your plug-in.
content-type-id string [optional] If your perspective type is either LiveContent or ActiveContent, the content-type-id must be provided so the correct content can be associated with the perspective.
view-id String [optional] If your perspective type is ActiveView, the view-id must be provided so the correct view can be associated with the perspective.
