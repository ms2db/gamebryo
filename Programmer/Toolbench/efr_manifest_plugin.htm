Plug-in Manifest

To create any Toolbench plug-in, you will have to write a plug-in manifest. A plug-in manifest is an XML document that describes your plug-in and all of its exposed capabilities through extension and extension points. Some plug-in manifests can become fairly large depending upon how much capability they want to expose.

The root node of a plug-in manifest is the plugin element. It provides the general description of the plug-in.

<?xml version="1.0" encoding="utf-8" ?>
<plugin 
  xmlns="http://www.emergent.net/schemas/plugin-base-schema.xsd"  
  name="Workbench" 
  id="Emergent.Toolbench.UI.Workbench"
  version="1.0.0.0"
  provider="Emergent Game Technologies"
  class="Emergent.Toolbench.UI.Workbench.WorkbenchPlugin">

  <extension-point/>
  <extension/>
  <requires/>
  <localization/>
  <runtime/>
</plugin>

plugin

Identifier Type Description
name String The display name for your plug-in.
name-key String The resource key to lookup the name of the plug-in if you want the name of your plug-in to be localizable*. Which would be advisable if you want the correct name to appear in things like the plug-in browser. *Localization and resources will be explained later in the document.
id String The unique identifier for your plug-in.
version String optional, default: "1.0.0.0"
The version that this manifest corresponds to for your plug-in. The version pattern for a plug-in is currently: [1-9].[0-9].[0-9].[0-9]
provider String The plug-in provider or author.
class String The full class name of the class to load when the plug-in is loaded. Several plug-ins use this as a way of starting services and to do other initialization things. The class must inherit from the Emergent.Toolbench.PluginBase class.
