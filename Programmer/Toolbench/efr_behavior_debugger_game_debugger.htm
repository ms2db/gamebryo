Game Framework Behavior Debugger

Behavior Debugger modules are designed to integrate with the language-specific behavior schedulers in the Emergent Game Framework. However, these debuggers are also easily severable and, in fact, typically compiled out of Shipping build configurations.

The behavior scheduler is responsible for creating the debugger module, which must perform the minimal initialization task of subscribing to ScriptDebuggerCommand messages via the MessageService. Further initialization can be delayed until a BEGIN_SESSION command is received from the Toolbench Script Debugger, requesting interactive behavior debugging.

When behavior execution is frozen for interactive debugging, the scheduler is paused and time does not advance, preventing queued behaviors from piling up. Services are otherwise updated normally. When the behavior debugger module is notified from the Toolbench connection that execution can proceed, the scheduler is resumed.
