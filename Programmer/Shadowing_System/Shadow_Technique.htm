Shadow Technique

A shadow technique represents a unique combination of methods for reading from and writing to a shadow map. There are many different shadow mapping techniques that exist, and undoubtedly many will emerge. An NiShadowTechnique object encapsulates the necessary information of a technique into a single object and allows application developers to create their own shader fragments to write to and read from a shadow map. Encapsulating the information in this way eases the process of extending the shadowing system.

When the NiShadowManager object is first initialized, all shadow techniques employed by the system will be initialized. Initialization of a shadow technique will register the technique to one of four possible slots. These slots directly correspond to the 2 bits used in the NiStandardMaterialDescriptor that define which shadow technique is used when generating shaders. The shadow manager may be aware of more than four shadow techniques, but there can only be a maximum of four active shadow techniques used by an application.

The shadowing system assumes that all shadow techniques in use are compatible with each other and that all shadow map render targets requested by the shadow click generator will hold all the required data. These assumptions are important when considering what shadow techniques to register to the application. If the techniques are incompatible, rendering results are undefined.
