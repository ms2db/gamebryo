Shadow Click Generator

The shadow click generator is used by the shadowing system to create all the NiShadowRenderClick objects necessary to render the shadow maps for all active NiShadowGenerator objects. The shadow click generator creates as many or as few shadow render click objects as needed. Its main responsibilities include:

  • Determining how to setup shadow render clicks for all the different possible light types that a shadow generator object may reference.
  • Assigning shadow map callbacks for a shadow generator.
  • Requesting NiShadowMap objects from the NiShadowManager object.
  • Assigning shadow maps to the appropriate shadow generator.

A shadow click generator encapsulates an algorithm that the shadowing system uses to determine which shadow maps are assigned to which geometries for each shadow generator. Applications should subclass the shadow click generator to define the shadow map generation method that best suits the needs of the application.

When the shadow manager generates a set of shadow render clicks, the active NiShadowClickGenerator object is used. This shadow click generator produces all the shadow render clicks needed to update the shadow maps. This process can be as simple or complex as desired. In general the main principles will remain the same across click generators:

  1. Iterate through all the active shadow generators.
  2. Determine which geometry objects assigned to the current shadow generator will be casting shadows in the current scene and which geometry objects will be receiving shadows.
  3. Decide what available shadow maps to assign to the shadow generator and assign the shadow map callbacks to use for the shadow generator.
  4. Request more shadow maps, if needed, or possibly throw away unneeded shadow maps.
  5. Provide the shadowing system with the shadow render clicks needed to render the current scene.
