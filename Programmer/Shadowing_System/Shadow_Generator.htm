Shadow Generator

Shadow generator objects cause shadows to be rendered. A shadow generator can be viewed as an optional modifier attached to an NiDynamicEffect object that causes the effect to cast shadows. Each shadow generator corresponds to a single NiDynamicEffect object. Gamebryo currently supports shadow generators with any non-ambient light type (directional, point, and spot).

By default all the NiNode objects affected by an NiDynamicEffect object will both cast and receive shadows from the dynamic effect's associated shadow generator. The set of objects that cast and receive shadows may be further refined by adding NiNode objects to the shadow generator's unaffected caster list and unaffected receiver list. If a node or its children are located in a shadow generator's unaffected caster list then they will not cast shadows for that shadow generator. Similarly, the unaffected receiver list defines objects that will not receive shadows from that shadow generator.

Shadow generators define the NiShadowTechnique object used when reading from or writing to a shadow map for that shadow generator. This behavior gives an application control over which shadowing technique to use per light. A geometry object may be a receiver for shadow generators with different techniques. However, only a single shadow technique will be used by the geometry when reading from shadow maps. The shadow technique used when reading is chosen based on the priority of the different shadow techniques. The geometry uses the shadow technique with the highest priority. Priority is defined by the slot to which the shadow techniques are assigned. Higher slot indices corresponds to higher priorities.

The shadowing system assigns and retrieves shadow maps from a shadow generator. Since there are many different shadowing algorithms available that require a single light source to multiple shadow maps assigned to it, the algorithms for assigning and retrieving shadow maps from a shadow generator are defined via callbacks. These callbacks are assigned to a shadow generator by the shadow click generator, allowing the shadow click generator to define the way shadow maps are handled based on the algorithms used by that generator.
