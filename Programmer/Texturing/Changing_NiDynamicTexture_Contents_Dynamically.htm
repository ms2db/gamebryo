Changing NiDynamicTexture Contents Dynamically

Updating the contents of an NiDynamicTexture is straightforward. The basic idea of an NiDynamicTexture is that it is at once both a "write-only" destination texture and a source texture which may be applied to objects in the main scene.

For a given frame, the basic principle of operation is for the application to lock the dynamic texture (signaling to the renderer that texels are being edited by the application) using NiDynamicTexture::Lock. This step returns a pointer to a buffer to which the application can write. The application can then perform a write-only edit operation on the texels of the dynamic texture. It is expected that every texel will get written by the application every time a lock operation is performed by the application, and that the application will write the texel data in the renderer-specific pixel format that the dynamic texture expects. When the application is done, it unlocks the dynamic texture (signaling to the renderer that the application is finished updating the a texture for a given frame) with NiDynamicTexture::Unlock.
