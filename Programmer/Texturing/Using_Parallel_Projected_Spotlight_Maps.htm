Using Parallel Projected Spotlight Maps

Parallel Projected Spotlight Example

Effect

A parallel spotlight map creates an effect of an infinitely distant light source shining through a hole, scrim, photographic slide, window, etc. The light rays are all parallel—the effect does not change size with distance from the source.

Filter

Any filter is possible, but BILERP or a BILERP-ed mipmap mode is recommended to avoid blockiness.

Clamping

Spotlights should use CLAMP_S_CLAMP_T mode. Only one copy of the spotlight image should appear in the scene. If a wrapped mode is used, multiple copies of the texture image will be tiled along the projected surfaces.

Effect mode

PROJECTED_LIGHT

Texture coordinate mode

WORLD_PARALLEL

Image

The image used may be any image that the application wishes to project into the scene, with one important restriction. The edge pixels of the image (left, right, top, bottom) must all be black. If the image itself is stretched to the edge of the texture, then clamped texture mode will cause the pixels at the edges to be "smeared" out past the boundaries of the desired projection volume. The black edges ensure that any objects outside of the projection volume of the texture that receive the projected texture will not be illuminated by the effect.
