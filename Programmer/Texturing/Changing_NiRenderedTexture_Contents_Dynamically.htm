Changing NiRenderedTexture Contents Dynamically

Updating the contents of an NiRenderedTexture is quite straightforward, as this type of texture was specifically designed to be updated on a per-frame basis. The basic idea of an NiRenderedTexture is that it is at once both a texture and a renderer's framebuffer. It is by treating the NiRenderedTexture as a framebuffer that an application can update the contents.

In order to switch a renderer to render to an NiRenderedTexture, first create an NiRenderTargetGroup that refers to the rendered texture. Then begin rendering to that render target group by calling NiRenderer::BeginUsingRenderTargetGroup with the NiRenderTargetGroup object as the argument.

When the application is finished rendering to a texture for a given frame, it signals the renderer by calling NiRenderer::EndUsingRenderTargetGroup. The application can then begin rendering to a different render target group, or the default one.
