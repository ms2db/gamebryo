Coordinate Specification

Since static multitextures require their texture coordinates to be defined on each object to which the effect will be applied, there are several options applications have for defining secondary texture coordinates:

Use the same coordinate set as the Base Map (commonly the set at index zero). This is not a generally useful option, since, if all multitextures could use the same texture coordinates as their bases, the textures should probably have been combined into a single texture.

Attach additional sets of texture coordinates to the object(s) directly using NiMesh interfaces for adding streams in the application itself. See the next section for a discussion of changing texture coordinates on the fly.

Define multiple sets of texture coordinates on the object(s) in the modeling package. Consult the Gamebryo export manual for the desired exporter prior to experimenting with this option, to determine what level of support the exporter has for multiple texture coordinates.

The number of sets of texture coordinates on an object is unlimited in theory, but note that objects should share and limit texture coordinate sets where possible, as additional sets of texture coordinates use memory and can cause problems if the number in simultaneous use exceed limits inherent in GPU hardware. Specifically, do not keep unused sets of texture coordinates on objects.
