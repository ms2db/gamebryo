NiSkyDome Customization

The NiSkyDome object is a sub class of NiSky and allows the quick generation of a sky dome geometry object suitable for rendering the NiSkyMaterial on. The following set of variables demonstrates the configurability of the sky dome and allows the user to place more vertices where extra detail is needed.

For the syntax on to adjust these values, please refer to the NiSkyDome class documentation.

Property Name Default Value Notes
Radius 50000.0f This value represents how far away the camera is from the horizon. A large number is recommended to represent infinity. The default sky material destroys the depth information of the dome so that far clipping and depth culling are not an issue.
Detail Multiplier 1.0f This value directly adjusts the number of vertices that are used in the generation of the sky dome geometry. Higher numbers increase detail, lower numbers decrease detail. A value of 1 is a general compromise between quality and speed and will use roughly 400 verts.
Detail Axis Bias 0.6f This value adjusts the distribution of vertices on the latitudes and longitudes. A value closer to 0 causes the number of longitudinal divisions to be increased and the pitch axis to be decreased. A value closer to 1 is the opposite, and a value of 0.5 is an even distribution of sub-divisions between the two.
Detail Horizon Bias 0.9f This value adjusts the distribution of the division of the latitudes between a linear distribution (a value of 0) and a power of 3 distribution concentrated at the horizon (value of 1). Higher detail at the horizon is preferred during vertex mode atmospheric scattering sunsets and when the camera never looks directly up.

The following set of wireframe screenshots highlight the effect of changes to the detail axis bias variable. In these examples it can be seen how the axis bias shifts the concentration of vertices between longitudinal detail, and latitude detail. Notice that a value of 0 results in all the detail going to the longitudinal divisions, whilst a value of 1 results in most of the detail going to latitudinal divisions.

This set of wireframe screenshots show the effect of changes to the detail horizon bias variable. Notice that values closer to 0 exhibit a linear latitudinal division, whilst a value closer to 1 results in an exponential shift in detail from horizon to zenith. This variable is very useful for increasing the detail of the sky shader sunsets in vertex calculation mode, as shown by the second row of screenshots.
