Entity property limitations

Properties can only use types known to the Game Framework

One of the limitations on entity properties is that all properties must be defined in terms of a data type known to the Game Framework. You can add support for your own data types but this requires additional programming steps. You cannot add properties of unknown types or types resolved at runtime.

See Adding entity properties for details on how you can add property support to your data types

Properties must be streamable.

All properties must be streamable. What this means is every property value must have the ability to be inserted and extracted from efd::ParameterList instances. This ensures properties can be marshaled over the wire to remote simulators and saved to persistent storage.

ReadOnly properties cannot be specialized per entity or changed at runtime.

Any property with the 'ReadOnly' trait is specified by the model and never set or changed per entity. These properties are edited in the Entity Modeling Tool but not available for edit in World Builder.

ServerOnly properties are not visible to clients.

Any property with the 'ServerOnly' trait set is not visible to any clients. These properties can only be accessed from a server process.

Reflected entity properties cannot be modified unless the 'Mutable' trait is set.

All reflected entity properties cannot be directly modified by remote simulators unless the 'Mutable' trait is set. This prevents remote simulators from altering data owned by another simulator.
