Tuning entity property performance

This section describes some strategies for optimizing your use of entity properties.

Cache property IDs for frequently accessed properties.

All of the available property accessors are overloaded to take either the property name or a property ID. Accessors that take the property name are convenient but take more time than their property ID counterparts. To increase property lookup performance, cache the property IDs for properties you access frequently.

Property IDs are assigned by the entity modeling tool (EMT) when it generates flat models. You cannot rely on EMT using the same ID each time the property is exported but you can cache property IDs at runtime. To do this, look up the property ID using the property name and store the resulting value. Then update you calls to property accessor methods to use overloads that take a property ID instead of a string.

Avoid unnecessary property accessor calls.

Property lookup calls can be expensive, especially when made from script. Try to avoid unnecessary property accessor calls. For example, if the property is marked 'ReadOnly' look it up once and cache the resulting value for future use.
