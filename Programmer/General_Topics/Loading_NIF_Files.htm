Loading NIF Files

The class supporting NIF loading is NiStream. Please see Object Systems for examples of loading files in one pass and incrementally loading files over time. Also, most Gamebryo sample applications load several NIF files, and are good examples of how to load NIFs and parse their contents.

The object creation functions necessary to load the classes in a particular Gamebryo library are automatically registered when the top-level header file for that library is included in an application. For example, if you want to load a NIF file containing NiAnimation objects, you must include NiAnimation.h in your application.

Background Loading

While most of the Gamebryo samples load their NIF files at application startup, this approach is not the only way that Gamebryo can load NIF files.

Gamebryo also enables NIFs to be loaded "in the background", using otherwise empty cycles during an application's rendering loop. This technique may be used to "silently" load scenes while the game is playing. In doing so, many games may hide or remove the "load hits" that could occur when new objects must be added to a scene during gameplay.

The BackgroundLoad sample demonstrates the background NIF loading system in Gamebryo.
