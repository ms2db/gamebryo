Categories overview

In Gamebryo LightSpeed, categories are used in both local and network messaging to limit delivery of a message to those handlers that are interested in receiving the message. Message producers send (publish) messages to a conceptual channel that is identified by a specific category. Interested consumers subscribe to the conceptual channels that they are interested in by using those categories. The messaging infrastructure ensures that only subscribed handlers (consumers) receive messages that are sent to a subscribed category. For example, at login time, a server process creates a special category for each client. Both client and server use this category to ensure that server-to-client messages go to the correct client, and that client-to-server messages arrive at the correct handlers in the server process. Categories provide both bandwidth and processing preservation and are abstract enough that a studio can use any application-specific policy to tune performance. This includes both remote and local-only processing.

Categories can be utilized by entities even in single player games. A category can be used to limit the number of entities that receive a behavior invocation request such as a grenade damage.
