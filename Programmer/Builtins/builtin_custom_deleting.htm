Deleting Custom Built-in API Functions

Built-in API functions are simply C++ functions. You can delete them and recompile your code to remove them.

SWIG .i files do not have deep dependency checking. If you have removed or altered functions by directly editing a SWIG .i file, you may need to force your script wrapper to recompile.

When you delete a custom built-in API function, you need to pay special attention to existing scripts that might be using the built-in. Because scripts are not compiled, they may appear to continue to work even after the built-in API function has been removed. When you remove a built-in API function, you should search through all of your scripts for places where that function might be used.
