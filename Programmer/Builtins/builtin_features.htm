Built-in API Features

  • Call built-in API functions from scripted or C++ behaviors
  • Function arguments support most Foundation and Game Framework data types
  • Function arguments can accept efd::list<T> and efd::map<efd::utf8string, T> for all supported types
  • Multiple values can be returned to using output parameters
  • Input-output parameters are also supported; values returned to script in this way act just like output parameters
  • Function overloading is supported for some but not all types. For more information, see Built-in API limitations.
  • New libraries of built-in API functions can be added
  • In DLL builds, you can dynamically load additional built-in API modules
  • Additional user-defined data types can be easily added by following existing examples in egfLua.
