Networking Limitations

In this release of Gamebryo LightSpeed, there are a few limitations related to networking.

  • Message coalescing—The network libraries (NetLibs) that currently ship with Gamebryo LightSpeed do not perform any kind of message coalescing. Every sent message typically results in at least one packet being sent over the wire.
  • Application-specific message handling—The NetLibs that currently ship with LightSpeed don't do anything that requires application-specific knowledge of message. If you require message reordering, dropping of low priority messages, or changing contents of messages, you will need to create a custom NetLib.
  • Message routing—The Network system has no concept of message routing. All processes must be connected directly to the subscriber/recipient or to same Channel Manager.
  • Qualities of service—You can define multiple qualities of service, each providing unique communication quality. However, because you can only use one NetLib in a process, the NetLib that is being used must be capable of providing all of the qualities of service that you define.
  • Channel Manager service—In your online game, there can only be one Channel Manager service and it must provide all of the qualities of service that you define.
  • Message order consistency—Ordered Categories only provide order from a single producer to a single consumer. Multiple consumers may receive messages in different orders if there are multiple producers on the same Category. If you require consistent message ordering across multiple consumers on a Category that has multiple producers, you should use the QOS_TOTAL_ORDER quality of service. QOS_TOTAL_ORDER ensures that all consumers, even consumers local to the sending process, will receive messages in the same order.
