Networking Overview

Gamebryo LightSpeed's networking system extends the messaging system by enabling seamless intra and inter-process message delivery to subscribers on both local and remote machines. The network system is responsible for delivering messages to all subscribers. When a consumer subscribes to a specific message Category, the networking system routes these messages to the consumer wherever it may be.

Like many networking systems, the network system in Gamebryo LightSpeed consists of several layers. Starting from the top down:

  • Message service—a network-aware service that you use to send a message. It determines whether or not it is connected to any other processes. If it is, it forwards the message over the wire as needed.
  • Channel Manager Service and Network Service—The former provides a listening endpoint; the latter provides the connecting endpoint. Each connection has its own unique connection ID, which is visible to both endpoints.
  • Network library (NetLib)—abstracts the network layer for the host OS. There are multiple NetLibs, each potentially capable of handling multiple qualities of service.
  • Transport—the layer that the NetLib uses to handle the work of actually passing data to the OS network layer. There is a separate transport implementation for each transport type (for example, TCP and UDP).

Messages may have various delivery characteristics that determine the order in which they are delivered and the reliability with which they are delivered. Traditional IP networking recognizes two distinct models:

  • TCP—for guaranteed in-order delivery
  • UDP— for situations that do not demand reliability

Altogether, message ordering and reliability (as well as other qualities, such as whether or not communications are encrypted) encompass Qualities of Service in the LightSpeed networking system. If you require a Quality of Service that is not one of the defaults provided, you can implement a new Quality of Service, or you can integrate an existing network library by creating a new NetLib. For more information, see Quality of Service and Creating a NetLib.
