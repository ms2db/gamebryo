Using messages

You can use messages for asynchronous or synchronous notifications. SendImmediate delivers messages in a blocking nested call. Other send methods asynchronously queue the messages for handling in the next OnTick of the MessageService. Message senders (producers) do not need to know about the subscribers (consumers) to a message. Similarly subscribers do not need to know about the message senders. This approach provides a scalable loose coupling between major systems in the application. Messages can be used to receive notifications from existing services without changing that existing service or can be used to create a new custom notification or callback system intended to be consumed by other as yet unknown services.
