Using the message service for multithreading

MessageService has not been thoroughly tested in a multithreaded environment and should therefore not be used from multiple threads. Access to any MessageService functions from any thread other than the main thread (the thread that ServiceManager::Run or ServiceManager::RunOnce is called from) is not currently supported.

The design of the message system allows for sending from background threads directly into the MessageService. But all subscriptions and handlers would only operate on the main thread. Sending to a background thread containing a second instance of the message service is not currently supported but is the intended system design.
