NiTerrain Metadata System

Meta-data is used to associate arbitrary data with terrain materials. For instance, a boot-on-metal sound file name could be associated with a metal terrain material. Every time the player walks (or collides) with the terrain, you can check for meta-data at this point through the interactor. If the player is walking on a metal material, the meta-data returned will contain the boot-on-metal sound information. Currently, meta-data can be attached to a terrain material using the tools in Toolbench or alternatively by manually editing the XML in the material package definition. For information on how meta data is handled when multiple materials are applied to a single point, see Querying Metadata.

Storage

Each material holding meta-data stores the information in an NiMetaData object. This object is used to store both the field and value of each meta-data entry on the material. Meta-data entries may be either an integer, a float, or a string. Further types are also defined for blended floats and integers which allow these values to be blended together when querying the data from a particular point on the terrain.

Setting Metadata on a Material

NiSurface objects contain a NiMetaData object for metadata to be stored. Setting metadata on a material is quite simple, first, one must get the metadata object from the material using the SetMetaData() and GetMetaData() functions available on the NiSurface interface, then the user may set the appropriate information on the metadata object directly.

The Toolbench world builder terrain plug-in provides a user interface to this data that allows artists to easily change and update metadata on a material.

Applying Metadata to a NiMetaData object

Most objects that can have metadata applied to them carry a NiMetaData object that stores a list of metadata entries and their values. Several types of meta data are supported: integer, float, and string. Blended versions of integer and float are also available. Blended versions are automatically blended when their values are queried.

Methods also exist within the object to remove all the metadata from an object and to load/save the data using a NiDOMTool instance (usually an XML file).

Get and set functions exist for all value types and the metadata keys may be queried. For further information on the ways to manipulate NiMetaData objects, please consult the class reference.

Querying Metadata

For information on how to query the meta-data that has been painted upon a terrain object, please consult the documentation on querying metadata in the common use cases section of this documentation.
