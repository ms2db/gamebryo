Build Prerequisites for Gamebryo LightSpeed for Win32

Gamebryo LightSpeed for Windows PCs has been compiled with Microsoft's Visual Studio 2005 and Visual Studio 2008. See Visual Studio Support for details. After installation, LightSpeed is set to use the versions of its tools that were built with Visual Studio 2005. If you desire to run the Visual Studio 2008 versions, you will need to change the installed shortcuts to point to the proper version.

Note

On non-development machines (machines without Visual C++ installed), an up-to-date copy of the MSVC8.0 runtime DLLs (for applications and libraries built Visual Studio 2005) or the MSVC9.0 runtime DLLs (for applications and libraries built Visual Studio 2008) must be present. The LightSpeed installer will give the option to install these prerequisites.

DirectX SDK

LightSpeed for PC requires the DirectX 9.0c runtime libraries. To compile libraries and applications that use DirectX 9.0c, the same version of the DirectX 9.0 SDK (DX SDK) must be used to compile all libraries. This means that if you link to the libraries that shipped with LightSpeed, you must use the same version of the DX SDK that was used to compile LightSpeed libraries. This SDK is available from the following location:

http://msdn.microsoft.com/directx.

Note

The LightSpeed installer runs the DirectX 9.0c runtime installer for the DX SDK version that was used to build the engine. This ensures that the correct versions of the D3DX9 library are present. The default DirectX 9.0c runtime installer available from the standard Microsoft DirectX web page or from Windows Update does not include the necessary DLLs. Refer to the DirectX Support documentation for more information.

Cg Toolkit

There is some code that makes use of NVIDIA's Cg Toolkit 1.5. While it is not necessary to install this Toolkit, it is required for any code that will make use of Cg. The Cg Toolkit 1.5 is available from the following location:

http://developer.nvidia.com/object/cg_toolkit.html.

Note

If you have installed Maya, Maya may install its own version of the Cg Toolkit to the system path by adding the bin directory from the Maya install location. This version is older than the above 1.5 version and should not be used with Gamebryo. Maya's bin directory should be removed from the path if this is the case.

Bink and Miles

LightSpeed uses RAD Game Tools' Bink SDK for video decompression and Miles SDK for spatialized audio. Gamebryo was built and tested using Bink version 1.9n, and Miles version 7.2g.

Although the redistributable portion of the Bink and Miles SDKs are included with LightSpeed, this fact does not in any way grant a Bink or Miles license to LightSpeed customers or end-users. The Bink and Miles SDKs must be requested from RAD Game Tools at http://www.radgametools.com/. Contact sales@radgametools.com) to request an evaluation SDK.

NVIDIA PhysX

Emergent is working closely with NVIDIA to ensure that our customers have a complete physics solution. A full-featured integration between the NVIDIA PhysX SDK and Gamebryo is part of LightSpeed. PhysX itself must be licensed separately from NVIDIA.

By default, the LightSpeed installer includes a prompt to install the PhysX system software. This software enables you to run PhysX applications. To compile PhysX applications, the PhysX SDK must also be installed. Most PhysX SDKs are shipped with the LightSpeed installer under the PhysX directory, which is located in the root of the install media. In addition to the SDKs, Max and Maya PhysX plug-ins (required for exporting PhysX content from those tools) are also located in that directory.

This version of Gamebryo LightSpeed has been tested with NVIDIA PhysX 2.8.3.
