Using behaviors

Behaviors are used to implement logic for Entities in a game. They are the actions that an Entity may be asked to perform, handlers for events performed in response to an action, methods to update Entity properties or whatever else the game may require. But before a Behavior is used, it must be defined in a model that will be mixed together to form Entities. After it is defined the logic may be written in C++ or various supported scripting languages.

Behaviors may be invoked by plain application C++ or by Behaviors using system methods. You can also cause them to be invoked by sending a system message. Calls to a Behavior may be blocking or non-blocking and may receive a callback when the Behavior is finished. To control how and where a Behavior may be invoked specific traits are applied to each Behavior in Toolbench.
