Introduction to NiPhysXParticle

The NiPhysXParticle library is modeled after Gamebryo's NiParticle library. Essentially all the NiParticle API is available for NiPhysXParticle and works in the same way. This documentation focuses on the differences between Gamebryo particles and PhysX particles and the additional things that must be done to use PhysX particles. Before reading this, please read the documentation for Gamebryo's NiParticle library to familiarize yourself with particle systems in Gamebryo.

The NiPhysXPSParticleSystem and NiPhysXPSMeshParticleSystem classes define the scene graph objects for a PhysX-enabled particle system. The former is derived from NiPSParticleSystem while the latter is derived from NiPSMeshParticleSystem. Thus you can choose to render a PhysX particle system as sprites or meshes. In addition, a PhysX particle system requires a prototypical PhysX actor for each generation of particles. The properties of the actors define, in part, how the particles behave, including their collision behavior. You need the actor even if using sprite based particles.

There are three primary benefits to using a PhysX-based particle system over a standard Gamebryo particle system:

  • All collision detection is performed by PhysX and can make use of its full generality. For example, collision groups or filtering can be used to control which objects the particles interact with, and even whether or not they interact with each other. Particles can collide with any PhysX shape, including general meshes.
  • The PhysX particles can have mass and collision forces on other objects in the environment. For example, you could use a particle system to model a paint-ball gun and have objects be knocked over by the paint pellets, complete with a secondary particle system for the splatter.
  • If PhysX hardware is present, then particle collisions will be hardware accelerated.
