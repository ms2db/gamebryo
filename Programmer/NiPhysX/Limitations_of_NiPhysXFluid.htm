Limitations of NiPhysXFluid

See the PhysX documentation for a complete set of limitations imposed by PhysX, and check the limitations carefully if your fluid simulation is not behaving as expected.

Hardware is Best for Performance

PhysX fluids supported on PhysX hardware when available, or software otherwise. Fluid simulation is likely to be significantly faster on hardware, and different settings may be required depending on the presence of hardware. All fluid simulation on consoles is in software.

Limits to Interaction Between PhysX Object Types

PhysX fluids at this time have different behaviors with different types of rigid body shapes, and cloth. Use the compartment system in PhysX to manage the various interactions between fluids, rigid bodies and cloth.

Drains

Drains in the Gamebryo-PhysX integration must be created in software.

Pre-Filled Fluids

Gamebryo does not support the streaming of fluid systems with non-zero particle counts. If you wish to add support, be aware that all particle creation is driven by the fluid creation data which contains particle IDs that can be used to map Gamebryo particles to PhysX particles.
