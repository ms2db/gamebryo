Tuning logging, asserts, and debugging performance

While logging and assertions are very useful in verifying the system operation and troubleshooting problems, they do have negative impact on the system performance. This section provides information about how you can fine tune the logging and assertion settings to improve the system performance.

Assertions are controlled by the EE_DISABLE_ASSERTS flag. If this flag is true, then all assertions will be disabled. There are different types of assertion macros defined in efd/Asserts.h that allow for some customization and simplification of some assertion scenarios.

Flags to control log levels can be found in efd/ILogger.h. There is a distinction made between an error log message and a normal log message. Error log messages should use the kERR* level, while generic log messages can be set with kLVL* levels. Through the use of these level flags, one can customize the logging from the application to great levels of detail.

Log Level Definition
kERR0 System fatal errors.
kERR1 Serious errors, loss of connection, failure to load critical data, etc.
kERR2 Common and recoverable errors.
kERR3 Trivial errors.
kLVL0 Rare, important events, such as client connects, disconnects, DB connection established.
kLVL1 Significant, but more common events (e.g., avatar moves between locales).
kLVL2 Common events (a message is handled).
kLVL3 Debugging (generally only enabled during development).

These log levels are masked for ease of use when using the ILogger::SetLogLevel function.

Log Mask Description
kLogMask_Err0 kERR0
kLogMask_Err1 kERR1
kLogMask_Err2 kERR2
kLogMask_Err3 kERR3
kLogMask_UptoErr1 kLogMask_Err0 | kLogMask_Err1
kLogMask_UptoErr2 kLogMask_UptoErr1 | kLogMask_Err2
kLogMask_UptoErr3 kLogMask_UptoErr2 | kLogMask_Err3
kLogMask_Lvl0 kLVL0
kLogMask_Lvl1 kLVL1
kLogMask_Lvl2 kLVL2
kLogMask_Lvl3 kLVL3
kLogMask_UptoLvl1 kLogMask_Lvl0 | kLogMask_Lvl1
kLogMask_UptoLvl2 kLogMask_UptoLvl1 | kLogMask_Lvl2
kLogMask_UptoLvl3 kLogMask_UptoLvl2 | kLogMask_Lvl3
kLogMask_All kLogMask_UptoErr3 | kLogMask_UptoLvl3
kLogMask_None Turn off logging
