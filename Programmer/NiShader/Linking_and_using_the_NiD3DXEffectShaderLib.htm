Linking and using the NiD3DXEffectShaderLib

Just as with the other shader libraries, NiD3DXEffectShaderLib can be either statically or dynamically linked to an application. Then, at runtime, the application can call NiD3DShaderFactory::LoadAndRegisterShaderLibrary to create the NiD3DXEffectShaderLibrary object and have it load any shaders it finds in D3D Effect files. Details on this process can be found at Registering your ShaderLibrary with the ShaderFactory. If a renderer exists at this point in the application, it will be used to load the D3D Effect files; otherwise, a temporary DX9 device will be created for the purpose. As long as a valid DX9 renderer exists before the application attempts to use the shaders, the effects will be correctly applied; if no such renderer exists, the shaders will not be applied correctly.
