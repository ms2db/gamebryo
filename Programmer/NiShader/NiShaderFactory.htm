NiShaderFactory

This class defines the factory interface for handling shaders in the Gamebryo engine. NiShaderLibrary instances are registered with the factory during initialization. This step must be completed before attempting to obtain any shaders, including loading any NIF files that contain materials that reference the shaders.

It is important to note that if multiple libraries are registered with the factory, if two (or more) of those libraries implement a shader of the same name, the first library that was registered will be the one whose shader implementation will be returned.

Several callbacks also exist in this class, which can be overridden to provide custom behavior. The first is the NISHADERFACTORY_CLASSCREATIONCALLBACK for initializing each registered NiShaderLibrary. When a library is requested with the LoadAndRegisterShaderLibrary function, this callback will be made, requiring the application to create the appropriate shader library instance and return it. The callback is registered via the RegisterClassCreationCallback function.

The next callback is the NISHADERFACTORY_RUNPARSERCALLBACK. When an application uses the LoadAndRunParserLibrary function to run a parser library, requiring the application to run the appropriate parser function for the requested library.

The final callback is the NISHADERFACTORY_ERRORCALLBACK, which is utilized by the shader system in general to report errors to the application. In the current version of Gamebryo, only the error string is valid. The other parameters, eError and bRecoverable, are provided for future expansion of the error-handling system.

For a complete description of the functions and members contained in this class, please refer to the NiShaderFactory class reference.
