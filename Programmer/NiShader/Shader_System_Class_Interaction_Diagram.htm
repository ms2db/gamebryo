Obtaining an NiShader from an NiShaderLibrary:

Relationships and interactions between NiD3DShaderFactory, NiD3DShaderLibrary, and NiD3DShader

The attachment of a shader to a geometry instance takes the following route through the classes:

  • The application and/or the material system will use the static NiShaderFactory interface to request that a shader of a given name be applied to a given piece of geometry. If the geometry will get the shader from a material, whether applied in the art pipeline or at runtime, then the material system will automatically apply the shader. Otherwise, the application can associate the shader with the geometry using an NiD3DShaderFactory will attach the shader from said array to the object.
  • If the shader is not in the loaded shader array, the NiD3DShaderFactory will query each registered NiD3DShaderLibrary for an instance of the shader. If any of the NiD3DShaderLibrary instances contain the shader, it is returned to the factory. The factory will then map it into its loaded shader array.
  • The factory then returns the shader if it is found
  • If the shader was not found, the factory will return NULL.
