Introduction

The Gamebryo shader system allows total reconfiguration of the art pipeline. With that flexibility certain considerations must be made to ensure that these assets fold seamlessly into an application. Additionally, applications that use shader driven assets require additional application code to load shader libraries and to synchronize the various parameters of the shaders. The following sections outlines valuable ways to ease application integration issues by discussing the need to determine a lighting model for the application, parsing and loading of shader files, use of time controllers to synchronize information, and load balancing considerations.
