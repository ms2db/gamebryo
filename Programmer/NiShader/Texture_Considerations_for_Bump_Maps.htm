Texture Considerations for Bump Maps

Textures for use in NSF shaders can be placed in either the default Gamebryo texture slots or they can be placed as extra data in the art packages. In the example shader, three textures are used. The base and glow maps are placed in the default Gamebryo slots for those maps. The normal map is placed as extra data to avoid any problems with forward differencing.

Normal Mapping is not Bump Mapping

There is a certain amount of confusion regarding the nomenclature of bump mapping. Dot3 Bump Mapping is one of the most common techniques implemented in shader programs today. Dot3 Bump Mapping is not a bump mapping technique, however. It is a normal mapping technique. Bump maps perturb normals while normal maps replace normals. For this reason, it is critical in the art packages that any normal maps used by a shader be placed in a slot other than the bump map slot of a Gamebryo shader. The runtime engine will automatically forward difference any map placed in the bump map slot to create dU and dV offsets. In code, programmers can control this behavior by correctly setting the format preferences of a texture.
