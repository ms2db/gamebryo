Loading Binary Shaders at Runtime

Once NSF files are parsed into NSB files, they need only be loaded in as binary shaders. There are libraries for each renderer to do this. The DX9/Xenon version is named NiBinaryShaderLib.lib, the D3D10 version is named NiD3D10BinaryShaderLib.lib, the D3D11 version is named ecrD3D11BinaryShaderLib.lib, and the PS3 version is named NiPS3BinaryShaderLib.lib. Applications using static libraries must register an additional callback to handle instantiation of the class that would otherwise be handled in the DLL initialization. The NiBinaryShaderLib libraries represent Gamebryo binary shaders that load NSB files at runtime.
