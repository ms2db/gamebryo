Use Time Controllers

In Gamebryo, shaders share common information through global attributes. Gamebryo time controllers provide an excellent way to link scene graph information with global shader attributes. This linkage is critical as it allows shaders to read parameters from scene objects. For example, an artist could set up and animate an arbitrary object in the scene from which a time controller might read position and orientation information. A shader could then use that information to perform some effect, such as transforming an eyeball's texture coordinates so the eyes track that object.

Since time controllers can be attached in the scene graph and execute when Update is called, they can be used by an application to update global attributes on a once per frame basis (assuming that Update is only called once per frame.) A sample NiTimeController subclass is listed in the next section. With such controllers, the application need only attach the controllers at initialization of the application. The controllers will update global attributes automatically.
