Introduction to CGFX And COLLADA FX

Gamebryo's NiShader system is a powerful way for applications to make use of advanced rendering techniques, and the NSF/NSB system makes it easy to write advanced shaders and integrate their development into the art pipeline.

The CGFX and COLLADA FX support extends that ease of use by allowing developers to use the CGFX file format to create Gamebryo shaders just as they would use NSF files directly.

When developing for PS3, CGFX shader files, and materials based on them, can be authored using tools such as FX Composer. The tools then convert and export these assets as COLLADA FX files. COLLADA FX files are XML documents (with a .dae extension) that describe one or more effects in full detail.

Gamebryo provides a script, called BuildShader, that converts COLLADA FX files to the NSF format and compiles the referenced CGFX files. As a result shader authors don't have to deal with COLLADA or NSF files directly. Because the CGFX and COLLADA FX support is based on Gamebryo's NiShader architecture, it would be useful to read the documentation for the system before trying to understand this implementation of the system.

Applications can request shaders, which encapsulates everything that is needed to render an object with a given technique. By making use of the annotations and semantics that the CGFX format provides, the parameters in the shader can be linked to Gamebryo data, just as is possible with NSF/NSB files that are created by hand. In addition, the CGFX and COLLADA FX support provides the same data to the artist's tools that the hand written NSF/NSB files do, allowing effects contained in CGFX files to be integrated into the art pipeline.
