NSF File Format Reference

The Gamebryo Shader System provides a text-based file format for rapid prototyping of shaders. The NSF format allows shader authors to quickly implement and test shaders, as well as allowing for tweaking of shader code without requiring the build-test-edit-repeat cycle. Shaders may be adjusted and all that is required to test the new version is to re-launch the application.

The NSF file support is provided via the NiShaderFactory::LoadAndRunParserLibrary function, and the NSBParserLib parser library. This function will load the parser library, which will in turn process any NSF files found in the given directory, writing out the corresponding NSB file for each one found. The NSB file will be named <SHADER NAME IN NSF FILE>.nsb. The NiBinaryShaderLib can then be used to load the created NSB files to complete the system. Doing these operations in this order will ensure that any NSB files used will get regenerated if the NSF file source was modified in between runs of the application.

It is expected that most developers will not want to ship actual NSF files with an application, but will compile the files to binary format and ship only the NSB files.
