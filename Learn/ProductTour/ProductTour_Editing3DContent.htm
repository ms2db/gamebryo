The Digital Content Creation Tools

Gamebryo LightSpeed supports the three major Digital Content Creation tools: 3ds Max, Maya, and XSI with robust and fully-featured exporters. Let's explore the Maya exporter by taking a look at the graphics for the player character. The source asset for this character installs by default in the following location:

%EMERGENT_PATH%\Samples\GameDemos\MangledMetal\Art Files\Export\Characters\Vandal\Vandal.mb.

If you installed the Maya exporter, you should be able to open this asset in Maya.

The Vandal character in Maya

The exporters support a wide variety of features, including animations, particle systems, materials, pixel and vertex shaders, lighting, and much more. For more information about these exporters, see the following sections:

  • Gamebryo 3ds Max Plug-in
  • Gamebryo Maya Plug-in
  • Gamebryo XSI Plug-in

If you were to export the Vandal character from Maya, it would produce several files:

  • NIF file—This file contains the geometry and bones for the character. It is also possible for this file to contain platform-specific versions of your textures.
  • KF file—This file contains the animation sequences for your character.
  • KFM file—This file contains a database of your animation sequences and how they interact with each other. This file may be originally generated from the DCC tool, but it is primarily edited in the Animation Tool. In fact, if you were to re-export the Vandal character, you would be given an option to overwrite this file and you would choose not to.

A generic version of these assets have already been generated for the Vandal. By default, these files install to the following location:

%EMERGENT_PATH%Samples\GameDemos\MangledMetal\Data\MangledMetal\GamebryoData\Generic\Characters\Vandal\

Note

Rapid iteration of NIF, KF, and KFMs are supported. For example, by modifying and re-exporting out one of the various MangledMetal art assets, you can see the resulting changes show up in the running MangledMetal application and also in World Builder. Note, however, that World Builder uses the "Generic" version of the assets. This means that a targeted export for DX9 may serve only to update the MangledMetal application, but not World Builder. Likewise, a "Generic" targeted export would update Toolbench but might not update the MangledMetal application.

Asset Viewer

As mentioned above, the Vandal character exports out to a NIF file, which contains all of the geometry and bones for character.

In the image below, we've opened up the previously exported NIF file for the Vandal, found at %EMERGENT_PATH%\Samples\GameDemos\MangledMetal\Data\MangledMetal\GamebryoData\Generic\Characters\Vandal\Vandal.nif.

The Vandal character in Asset Viewer

Above, take note to the Scene Graph Tree view on the right-hand side of Asset Viewer. Gamebryo LightSpeed represents 3D graphics objects using a scene graph. Scene graphs represent graphical data using hierarchical parent-child relationships. In the case of the Vandal, the bones that define the Vandal's skeleton are nodes in the graph which are rooted at the pelvis. Other nodes, like "khador_lj_geoShape", represent the 3D geometry of the Vandal.

Asset Viewer is a great tool for understanding how the exporters are converting your DCC content. You can explore all of the parameters for a given scene graph node in the Scene Graph Tree view by right-clicking the desired entry and choosing "Properties" from the context menu.

Animation Tool

As mentioned above, the Vandal character exports out to KF files, which contain all of the animation sequences for the character, like "idle", "walk", and "run". These files can be combined and edited in the Animation Tool.

In the image below, we've opened up the previously generated KFM file for the Vandal, found at %EMERGENT_PATH%\Samples\GameDemos\MangledMetal\Data\MangledMetal\GamebryoData\Generic\Characters\Vandal\Vandal.kfm. KFM files are essentially animation databases where the artist has set up transitions between individual animation sequences so that they may be correctly played back in-game.

The Vandal character in Animation Tool.

The Vandal is playing the "charge_attack" animation in the image above. You can get a preview of any animation sequence by clicking on an animation sequence name in the Project View. The transition table below identifies how the individual animations interact with each other. Click any transition in the table to get a preview of how the two animations will blend.

Individual animations can be activated within Behavior scripts for entities. For more information on this, see the egmAnimation Lua reference documentation and the StandardModelLibrary::Actor Model reference documentation. The Vandal mixes-in this model through the MMActor model.

The AnimationTool also allows you to play with the character in rag-doll mode. See Working With Physical Sequences in Animation Tool for more details. Similar to the animation sequences, rag-doll and other physics actions can be used in entity Behaviors. For more information, see PhysXModelLibrary::PhysXRagdoll and documentation about PhysX System Services. Additional information about creating and setting up rag-dolls with Max and Maya is also available.

For more information on the types of blending supported in Animation Tool and to find out more about the tool, you should consider following the Animation Tool tutorial track.

Note

Rapid iteration of KFM files is supported. For example, you can edit a KFM file using the Animation Tool while you have your MangledMetal application running. When you save the KFM, any changes to the various sequence transitions will be reflected in the running game.

Moving on...

You've now completed the Product Tour, but you've really only hit on a fraction of what Gamebryo LightSpeed is capable of doing. We provide additional tutorials that introduce you to the Core Runtime, Game Framework, and Online subsystems. We recommend that you work your way through them to gain further insight into Gamebryo LightSpeed capabilities.

You can also check out Kickstart, a project from Emergent Labs. Kickstart extends Gamebryo LightSpeed by providing libraries, tools, and game samples that are designed to support your rapid prototyping needs and reduce the time that it takes you to create a first playable. Kickstart demonstrates Gamebryo LightSpeed best practices, provides samples that are easy to follow and update, and includes useful library code in both C++ and scripting languages.

Visit the Kickstart Project at the following location:

https://pulse.emergent.net/wiki/index.php/Kickstart
