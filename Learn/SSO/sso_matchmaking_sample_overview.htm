Matchmaking Sample Overview

Note

The Matchmaking sample is not currently supported on the PC platform.

The Matchmaking sample demonstrates how to create a simple multiplayer network game using a matchmaking host. It also demonstrates how to implement multiplayer gameplay via messaging.

One instance of the sample will act as the host and will attempt to create the game on the platform-specific matchmaking host. It will then wait in the "lobby" for the peer players to join the game. The rest of the sample instances will act as peers and will attempt to join the game created by the host instance. After the configured number of peer players have joined the game, the host instance will start the actual game play.

The game itself is a simple simulation of teamed warships that fire on each other until all of a team's ships are eliminated. There are two teams of warships. One team is owned by the single simulated host player. The other team is owned by all connected simulated peer players. Each connected peer player controls one ship. The host player starts with a number of ships equal to the number of peer ships. That is, there is one host ship for each connected peer.

Source Location

By default, source for the Matchmaking sample installs in the following location:

%EMERGENT_PATH%\Samples\OnlineTechDemos\MatchmakingSample

Running the Sample

For general information about running samples on Win32 and supported consoles, see Running Gamebryo LightSpeed Samples. Depending on the platform that you plan to run the sample on, you might have to perform some additional configuration.

If you are running this sample on the Xbox 360 or PS3, you need to make sure that you are signed in to Xbox Live or Playstation Network before starting the application. Refer to the manufacturer's documentation for instructions on how to sign in to these networks.

For the Matchmaking sample, the two most important parameters for starting a new game are defined in the ConfigCommon.ini file.

Parameter Description Default
Game.PeerPlayersDesired Number of peer players that must join the game before actual game play starts. 1
Game.Host Determines if this instance should host a new game (true) or join an existing game. False

The host instance should be started first. However, before you start the host, you need to override the default setting for the Game.Host parameter via the command line (or via a temporary change to the ConfigCommon.ini file). Set this parameter to TRUE. If you want to run a game with more than two players, you also need to override the Game.PeerPlayersDesired parameter's default value (1). For example, if you want to run a game with four players, set the parameter to 3. (This parameter is not used by the peer instances.)
