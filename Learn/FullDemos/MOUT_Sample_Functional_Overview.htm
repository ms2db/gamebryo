MOUT Sample Functional Overview

The sample uses a large number of the available systems in Gamebryo, including the main rendering engine, the animation system, and the collision detection system, as well as the full tool pipeline from 3ds max. Some specific Gamebryo-based features of the sample include:

  • The application's character system uses the NiActorManager system for animation control.
  • The character system uses the NiCollision collision detection system to prevent intersections with buildings in the world.
  • The character system includes a rendered projected shadow that is drawn as a separate piece of transparent, dynamic geometry.
  • The terrain is rendered using a shader that operates within the standard Gamebryo rendering pipeline. The shader blends between multiple textures to provide seamless ground-covering transitions.
  • The Gamebryo input system, NiInput, is used to implement a typical first-person-style control paradigm, including mouse-look.
  • The Gamebryo NiParticle particle system library is used to implement and display fire and smoke for burning buildings.
  • The camera system uses Gamebryo's 2D screen elements to implement a lens flare effect and 2D map of the MOUT world.
  • The application shows the flexibility of the Gamebryo rendering system when buildings in the scene obscure the view of the character. The render state of these buildings is dynamically modified to put them in an "x-ray" mode to prevent character obstruction.
  • Automatic frame-rate limitation is used to lock the rendering to a 60 frame per second maximum. On machines above the minimum specification, the frame rate should be a constant 60 frames per second. This ensures a smooth, even frame rate.
