Dodgeball Overview

Dodgeball is a light-hearted game that demonstrates rag-doll physics and shadows. A character, named Slim, is trapped in the playground and under player control. Two oversize air guns fire bouncing balls at Slim at irregular intervals. When Slim is hit had by a ball, he transitions from player or demo control to physical control. After flopping around on the ground for a while he transitions back to player control.

The feature of the demo include:

  • Smooth transitions from key-framed animation to physics, and back again, automatically handled by the Gamebryo-PhysX integration into the animation system
  • Application of non-physical forces to add amusement
  • Dynamic shadow maps within the playground area, using Gamebryo's integrated shadow map system
  • The use of animation accumulation using Gamebryo's animation system
  • Manipulation of the transparency of the buildings as the camera moves behind them
  • Camera damping
  • Customized UI elements

Minimum Recommended System Specifications

Dodgeball includes shadows and other real-time shader effects. At startup, the sample will check the capabilities of the graphics hardware being used, and if insufficient will prompt the user with an error message box, after which the application will exit.

The minimum recommended PC system used with this application is as follows:

  • Pentium IV 2.X GHz CPU
  • 512 MB RAM
  • Graphics card capable of shader model 2.0 and multiple render targets, such as nVidia GeForce 6800 or ATi Radeon 9700.
  • Windows XP
  • DirectX 9.0
