Trigger Library

Overview

The TriggersLibrary is an Entity Modeling Tool (EMT) project which defines models to handle Triggers. Triggers are entities which may check for a condition to occur and contain behaviors for how to respond to that condition.

Triggers have two basic components: Sources and Responses. Fully-implemented Triggers are created by mixing in zero or one Trigger Sources and at least one Trigger Response. Since the source and response models are independent, unique triggers can be created by mixing and matching sources and responses without having to implement any additional behaviors.

Trigger Sources

A Trigger Source is just a model that calls an OnTrigger behavior on itself under some condition. What that condition is varies from trigger source to trigger source. The TriggersLibrary defines a set of sources, but you are free to define your own. Make sure to mix AbstractTriggerSource into your trigger source.

Note that the Trigger Source does not define the OnTrigger behavior; it merely refers to it. It is the responsibility of the Trigger Response object to define the behavior that will be called.

The following trigger sources are defined in TriggersLibrary:

Model Name Description
AbstractTriggerSource Base mix-in for all trigger sources.
CollisionTriggerSource Fires when something collides with it.
EntitiesDeadSource Fires when no instances of a given model type exist.
OnCreateTriggerSource Fires immediately when created.
PeriodicTriggerSource Fires repeatedly at a given interval after invoked.
TouchSource Fires when touched by another entity in game.

Trigger Responses

A Trigger Response is a model that implements the OnTrigger behavior to do something. What it does varies from trigger response to trigger response. The TriggersLibrary defines a set of responses, but you are free to define your own. Make sure to mix AbstractTriggerResponse into your trigger response. The following trigger responses are defined in TriggersLibrary:

Model Name Description
AbstractTriggerResponse Base mix-in for all trigger responses.
AnimationResponse Plays an animation on self when invoked.
DelayedSpawnResponse Spawns an entity after some given sleep time.
MultiTargetResponse Calls the OnTrigger behavior of multiple entities when invoked. By using MultiTargetResponse, you can cause a single trigger source to invoke multiple trigger responses.
RemoteBehaviorResponse Call an arbitrary behavior on another entity when invoked.
RemoteBoolPropertyResponse Set a boolean on another entity when invoked.
SpawnResponse Create an entity when invoked.

Complete Triggers

A Complete Trigger is a model which mixes in zero or one Trigger Sources and at least one Trigger Response. It represents a complete in-game entity which can take action when certain conditions are met. The TriggersLibrary defines some triggers for you, but it is easy to mix in Trigger Sources and Trigger Responses to create your own meaningful trigger types which handle different conditions. The following Triggers are defined by the TriggersLibrary:

Model Name Description
CollisionSpawnPoint Spawn an entity when something collides with an object.
CollisionTrigger Invoke multiple Trigger Response objects when something collides with an object.
DoorOpenOnTouch Open a door when it is touched.
OnCreateInvoker Call a behavior on an entity when created.
PeriodicMultiInvoker Invoke multiple Trigger Response objects repeatedly with a fixed delay.
TouchAnimTrigger Play an animation when an object is touched.

It is possible to have a complete trigger that only contains Trigger Responses. In this case, it is expected that the response would be remotely activated by another entity. A common example is when one trigger mixes in MultiTargetResponse and points its target list properties at other entities that only contain responses.
