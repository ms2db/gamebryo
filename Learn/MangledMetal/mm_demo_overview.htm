MangledMetal overview

The MangledMetal demo takes place in a world of giant steam-powered mechanical beings called Warjacks, from Privateer Press's table-top role-playing game, Warmachine®. You control the actions of a mechanical being and are able to interact with the world via physical actions (using NVidia PhysX™) and combat (moderated by Lua scripts). It is a full-featured, robust application demonstrating many of the features and best practices of the Emergent Game Framework.

The Entity Modeling Tool (EMT) and World Builder are used for level editing. Lua scripting is used extensively for game logic. Rapid iteration of models, art files, Lua scripts, and world files can be used to debug and tune gameplay.

The MangledMetal demo includes the following features:

  • Cross-platform—works on Win32 DX9 and DX10, PS3, and Xbox 360
  • Use of Lua scripting and C++ entity behaviors for game logic control
  • Use of custom Lua built-ins written in C++
  • Tight integration with NVidia's PhysX™ allows placement and preview of simulation objects
  • Prebuilt game components, such as a triggering system and a spawn system, which you can reuse in your own applications
  • Examples of basic combat AI and combat system mechanics
  • A sophisticated in-game camera system controlled via EMT entities and C++ code
  • Supports rapid iteration on the game elements at runtime
  • Supports Lua script debugger
  • SpeedTree assets provided as exported meshes by Interactive Data Visualization, Inc. (makers of SpeedTree®)
  • The biggest, baddest steam-powered robots ever featured in an Emergent Game Technologies demo

Note

For rapid iteration to work correctly for applications that use the Asset Runtime Service, the assets used to run the console version of the application must be identical to the assets that are being edited on the host PC. Refer to the Running Gamebryo LightSpeed Samples doc for information on how to update the assets on the various consoles.
