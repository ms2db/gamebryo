MangledMetal blocks detail

Level03.block is the main block file for the demo, and the default file to be loaded on Win32, Xbox360, and PS3. It is a fully populated level with environment props, terrain, enemies, and spawn areas. It may take a minute or so to load.

Level01.block is a smaller block file that loads more quickly and uses fewer resources.

The terrain entity is locked. This is a safeguard that prevents it from being picked by accident when in selection mode in the render window. If you do need to select it, you can do so in the Entity View under the Environment folder. The terrain entity is instanced from the default PhysXTerrain entity model (PhysXModelLibrary).

The Terrain Entity comprises the following:

  • root.terrain (terrain asset)

This is created when the terrain is created in Toolbench. The assets for terrain for each block are kept within the terrain folder, which is located in the following location:

%EMERGENT_PATH%\Samples\GameDemos\MangledMetal\Data\MangledMetal\GamebryoData\Generic\Terrain

The Block file is organized into the following folders:

  • Characters: Contains all player characters
  • Enemies: Contains all enemy characters
  • Environment: Contains all trees and rocks, graves, gates, etc. These are further split into 12 "Regions"; items grouped into a given region are hidden when that region is not visible to the player in order to improve performance.
  • Lighting: Contains the light nodes for the level
  • Nodes: Contains miscellaneous assorted models like waypoints for enemy AI's and the base PhysX scene. (Any block file that uses PhysX needs a PhysXScene Entity. This entity can be instanced from the existing Entity Model in the PhysXModelLibrary.)
  • RegionTriggers: Contains the trigger areas to make the environment regions visible or invisible. As the player passes through trigger regions, the game toggles environmental objects in other parts of the block file visible and invisible.

The level contains three trigger examples:

  • A simple radius trigger, Door03_Gate0. If the player enters the radius of the trigger, the response is to open the gate.
  • The second is a more complicated trigger that requires an item, SoulCage_forGate0, to be destroyed before the gate will open.
  • The final trigger example is the SpawnTemple, found at the end of the level. This entity is set to constantly keep five active Reaper Enemies in the level at any one time. It can be deactivated by destroying the two SoulCages in front of it.

Other items of interest in MangledMetal.block are the Reapers and Deathripper enemies, which are scattered throughout. Most enemies are set to patrol the waypoints that are placed around the traversable parts of the level. There are "Pedestrians" in the level - noncombatant humans who run in fear when you approach. There are also a few PhysX props set in the level. These include the large blocks and walls that can toppled and pushed around the level. These serve as a basic example of simple PhysX working in a .block file.
