Artist Notes

ABV Proxy Geometry

It is much easier to test for a collision between two spheres than it is to test between two complex objects consisting of thousands of triangles. The same notion applies to other simple geometric shapes, such as capsules, boxes, and half-spaces (one side of a plane). Alternate Bounding Volumes (ABVs) enable you to wrap your complex geometry with these simple shapes in order to significantly increase Collision performance.

Gamebryo enables you to create proxy boxes inside your modeling package so that you may carefully define and align these ABVs to your model geometry.

Collidees and Colliders

All of the models in the "Collides.max" file are separated into two categories: Colliders and Collidees. These groupings refer to the dynamic or static nature of the colliding objects. The collision system uses the knowledge that objects are static for optimization. In CollisionTestStatic, the nodal groupings allow this particular application to find and processes the two types of collision objects quickly. The application creates the collision models in the system appropriately based on the group they are found in. Since ABVs are used, the geometry for these objects could be significantly more complex without impacting collision detection performance, since the number of triangles would only affect the speed of the renderer, not the collision detection system. Without ABVs, the bottleneck would likely be the collision system because of the number of calculations required.

Collides.max illustrates how ABVs may be set up for both simple ABV primitives and for biped skinned characters. For a discussion of how ABVs are setup, see Collision Detection and 3ds Max documentation.

Exporting Collides.max

To export Collides.max to Nif, use the process script "CollisionTestStatic v1.0.script" during export from 3ds max. Actually, if you load Collides.max, the accompanying process script "CollisionTestStatic v1.0.script" will be run by default by the Gamebryo 3ds max Plug-in, during export from 3ds max.

Alternatively, you may create a script to do the same processing yourself by performing the following steps:

  1. Choose to edit the process script from the main export options.
  2. Ensure that the following plug-ins have been added (in order) for the current script:
    • Scene Graph Optimization
    • Create ABV
  3. For the Scene Graph Optimization plug-in, ensure all optimizations are checked except:
    • Merge sibling nodes
  4. For the Create ABV plug-in options, ensure all of the following options are checked:
    • Create ABV
    • Create WireFrames representing ABV Proxies
    • Create WireFrames representing ABVs
    • Create WireFrames for Selections
    • Remove Proxy Geometry
  5. Optionally, save the modified script, if desired.
