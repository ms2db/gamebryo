PhysX Particles Demo Overview

The PhysX Particles sample demonstrates how applications can use PhysX in conjunction with Gamebryo's particle systems to produce physically plausible particle-object interactions. The demo simulates bubbles rising through a medium and pushing around floating rigid shapes. The fluid medium is not simulated. Rather, the bubbles are controlled via a particle system with forces that cause them to rise. Gamebryo's PhysX particles then convey the particle motion to PhysX which uses them to interact with the rigid bodies, and vice versa. This demonstrates the principle reason to use PhysX-based particles over regular Gamebryo particles: they can collide with complex shapes and there is two-way interaction between the particles and shapes.

PhysX Particles demo

The code loads two NIF files, or three if it detects a PPU (PhysX hardware accelerator board). One NIF has a standard Gamebryo particle system. You will see that the particles rise straight up and nothing else moves. The second NIF file, accessed by hitting the space key, is using a PhysX-based particle system and the other shapes are physical rigid bodies. This is the system displayed when the application starts. You see the bubbles push the shapes around, and also stream around the bars at the top of the scene. If you have PhysX hardware, then the third scene (hit space again) looks the same but has a higher frame rate.

From a modeling perspective, these scenes differ only in the Gamebryo Tool Plug-in settings used upon export from Max or Maya. See the documentation for your art package, and the PhysX Extractor tool plug-in, for more information on exporting a particle system to use PhysX-based particles.
