LogoApplication Use and Customization

The basics of using LogoApplication are as follows:

  • Add the LogoRenderer files to your project.
  • Include the LogoRenderer.h file.
  • Instantiate a copy of the LogoRenderer class (it is based on NiRefObject, so use NiNew).
  • In your render loop, call bool RenderLogoFrame(float now); until it returns false.
  • When Update returns false, you can either destroy the LogoRenderer or switch the logo image and call LogoRenderer::Reset. This allows you to chain multiple logos together.
  • When done (Update returns false and you have displayed all logos) delete the LogoRenderer instance and begin calling your own render code.

The LogoRenderer class has several customization points:

  • The constructor has an optional duration. This affects the total duration but does not change the fade in/out rates.
  • The constructor also accepts pointers to NiFloatInterpolator objects. You can pass one or both into the constructor. Passing NULL for one indicates for the renderer to use the default. For simplicity, you can also create default versions by calling the DefaultSize and DefaultAlpha functions, modify these instances, then pass them to the constructor. Using this technique, you can change the fade-in/out rates as well as the length of time to display the logo.
  • If you do not want the size (or alpha) to change, then you can call LogoRenderer::ConstInterpolator() and pass in a constant value. This creates an NiFloatInterpolator that always returns the given value.
  • You can change the background color by calling LogoRenderer::SetBackgroundColor().
  • You can change the logo texture by calling LogoRenderer::SetLogoTexture().
  • You can start the entire animation over by calling LogoRenderer::Reset().
