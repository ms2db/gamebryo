InputDemo Overview

Note

If you haven't already done so, we recommend that you look over the NiInput Basics documentation, as well as the Immediate Mode and Action Mapping Mode sections before investigating the InputDemo sample.

The InputDemo sample was developed to demonstrate the capabilities of the Gamebryo Input Library. This includes showing an example of Immediate Mode processing, two different approaches to using Action Mapping Mode, and rumble support.

Note

In the current release of NiInput, rumble is not supported on the Win32 platform except with the Xbox 360 Gamepad.

The sample also shows the cross-platform capabilities of the library by having very little platform-specific code. The only platform-specific code required is the following:

  • The setting of the application media path (where data files are located).
  • The setup of the NiInputSystem::CreateParams prior to creating the input system.
  • The creation of action map instances.
  • The dump of frame rate information at the termination of the application.

Other than those, the application code is identical regardless of the target platform.

The sample is divided up into 'info screens', which in turn are divided into four sub-screens that display information about up to four input devices. Each info screen demonstrates a different aspect of the input system. These include the following:

  1. Immediate Mode—this screen shows immediate mode in action. It demonstrates how to use the polling functions of input devices, displaying each devices types information in real-time.
  2. Rumble—this screen demonstrates the rumble feature of the input system. Note that only game pads support rumble, so there will only be displays for valid game pad devices. [NOTE: In the current release, Win32 does not support rumble except with the Xbox 360 Gamepad.]
  3. Action Map Mode—this screen demonstrates one possible usage of the action mapping mode. It utilizes defined action enumerations and displays strings indicating which actions have been triggered by input devices.
  4. Callback Action Map—this screen demonstrates another possible usage of the action mapping mode. It utilizes a callback method in the actions which display strings indicating that the action has occurred.
  5. Exit Screen—this screen is used to allow the user to exit the application. Due to the sample demonstrating input, the standard exit conditions couldn't be used as they could accidentally be triggered by the user. This screen is the only valid way to exit the application. [NOTE: On Win32, the ALT-F4 combination will exit the application at any time.]
