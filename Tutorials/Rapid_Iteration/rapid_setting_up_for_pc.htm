Rapid Iteration: Secondary PC Setup

Note

This step is not required if you are running the game on the same PC where you are running Toolbench. If this describes your set up, skip this step and move ahead to Resetting the Game.

You can run your game on one PC and run Toolbench on another PC. For example, you might want to optimize game or tool performance during development. Or, you might need to exercise the game on different hardware. Rapid iteration works well in this scenario. It is similar to developing on PC and running the game on a console.

Overview

The recommended approach is to setup a shared directory on the PC where you are running Toolbench. The shared directory would normally point to your <Asset Web Root Directory> and would be reachable via a UNC path from the PC where your game is running. For example: \\my_tool_pc\GameContent.

After you set up the share, you need to set the configuration parameter for the game in Config.ini:

[AssetWeb]
Path = \\my_tool_pc\GameContent
OverrideSystemId=true

Starting Your Rapid Iteration Session

Start your rapid iteration workflow as follows:

  1. Start Toolbench on your development PC.
  2. Open the game solution file.
  3. Start the game.

Note

Normally, the game uses LAN broadcast to determine the IP address of the PC on which Toolbench is running. However, if you are testing a PC whose IP address is on a different subnet, you must edit the ConfigCommon.ini file.

[AssetWeb]
UseDefaultChannelMgr=true

[ChannelManager]
HostName=<IP of Toolbench PC>

When the game and Toolbench are both connected, you will see a green connection icon in your system tray. Hovering your mouse over the icon in the system tray will display a ToolTip similar to "Connections: 2". This indicates that the PC where the game is running and the PC where Toolbench is running are connected. You can now open World Builder or another tool to begin a rapid iteration workflow.
