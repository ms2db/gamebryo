Note

When changing a behavior script, make sure that the update script has no compile errors or your changes will not be reflected on the application.

Rapidly Iterating on Behavior Scripts

  1. In Toolbench, select View > Behaviors from the menu.
  2. Open EseAvatar.lua.
  3. Add the following code to the OnTick behavior, immediately after the line 'local modelname = self:GetModelName()': 
    local currScale = self.Scale
if currScale < 2 then
    currScale = currScale + 0.05
    self.Scale = currScale
end
  4. Save the file.
  5. In the application window, the Avatar starts growing in size until he reaches double his original size.
  6. To set the scale back to normal, simply replace the new code with: 
    self.Scale = 1
  7. Save the file.

By changing the lua script, the Rapid Iteration Service notified the scheduler that a new file has been detected. The scheduler will try to load the new file and if it fails, it will revert to the old copy.
