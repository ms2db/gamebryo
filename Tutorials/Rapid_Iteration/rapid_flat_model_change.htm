Note

Changing a model should also affect all models that mix in the model. If changes are not seen in all affected models, you might have to reset the game.

If you set the scale property in script as described in Script Changes, you should comment out or revert these changes and reset the game first before continuing with the changes described here. Otherwise, you will not see the effect of changing the scale property in EMT.

Rapidly Iterating on Models

  1. Double-click HelloWorld.emtproj in Toolbench to open it.
  2. Navigate to the EMT Project Explorer tab and double-click the Jack model.
  3. In Toolbench's Entity Model Editor (right pane), click the Mix-in Properties tab.
  4. Set the KfmAsset property under Actor to TreasureChest.
  5. Save the change.

    In the running game, the Avatar and the AI Jack entities both become treasure chests.

  6. Set the KfmAsset back to Jack.
  7. Save the change.

    In the running window, the Avatar and the AI Jack entities both return to their original figures.

  8. Now try setting the Scale property of the Jack model (Located under Placeable) to 2.

    In the application window, only the Avatar should scale to 2 times the size of its original size. This is because the scale of the AI Jack is set (overridden) in the world file.

In this scenario, we modified a flat model file in Toolbench and the Rapid Iteration Service picked up the change and notified the game to reload the involved entities.
