Rapid Iteration: NIF or KF/KFM File Changes

You can rapidly iterate on NIF and KF/KFM art assets. In this example, it is assumed that you want to re-export our cube.nif or Jack.kf file and that you want to see the changes reflected in the game without restarting it.

Rapidly Iterating on NIF or KF/KFM Art Assets

  1. Navigate to the Hello World graphics directory:

    %EMERGENT_PATH%\Media\Samples\Graphics\Art Files.

  2. Open the cube.max source art asset.
  3. Make some obvious change, like changing the texture or perturbing a vertex.
  4. Export the file to

    %EMERGENT_PATH%\Media\Samples\Graphics\cube.nif.

  5. In the running game, note that the cubes in the main room reflect the change.

In this example, the Rapid Iteration Service picks up the change and notifies the Scene Graph Service or the Animation Service. Once they are notified, both of these services will take steps to reload the involved entity with the new file.

Note

For details about file types that do not support rapid iteration, see Asset Infrastructure Limitations.
