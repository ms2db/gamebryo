Rapid Iteration: External Texture Changes

You can rapidly iterate on external textures referenced from a NIF. In this example, it is assumed that you want to edit the texture mapped on a TexturedCube model (ExternalTexture.tga mapped onto the cube in TexturedCube.NIF) and that you want to see the changes reflected in the game without restarting it.

Rapidly Iterating When Editing Textures

In the preliminary steps, we add an object that uses external textures to the scene. After we add the object, we edit the texture.

  1. In Toolbench, double-click HelloWorld.block in the Hello World game solution.
  2. In the Palette View (left pane), navigate to the HelloWorld library, select the TexturedCube model, and place one in the scene. The mesh used for this model references an external texture named ExternalTexture.tga.
  3. Save the world, making sure that you overwrite the existing HelloWorld.xblock file under the %EMERGENT_PATH%\Media\Samples\GameFramework\HelloWorld directory.

    A cube with a picture of Jack should appear in the running application.

  4. Open the texture file %EMERGENT_PATH%\Media\Samples\Graphics\ExternalTexture.tga in an image editing application (for example, Photoshop or Paint.NET).
  5. Make an obvious change like drawing bright circles on top of the image, and save the image.
  6. In the running game, note that the texture mapped on the cube has been updated.

In this example, the Rapid Iteration Service picks up the change and notifies the Texture Rapid Iteration Service. After the Texture Rapid Iteration Service is notified, it will take steps to reload the modified texture map.

Notes

  • For details about file types that do not support rapid iteration, see Asset Infrastructure Limitations.
  • Only some texture formats are supported for rapid iteration, specifically TGA, DDS, BMP, and SGI. For information on how to extend this, see the ecr::TextureRapidIterationService reference documentation.
  • A common workflow will be to work on an uncompressed source format, for example, .PSD or .PDN, and then export to DDS or TGA when the changes should show up in-game.
  • Only external textures are supported (when exporting a NIF, make sure "Original Images" is checked, instead of "Include in main NIF").
