Multiplayer Rapid Iteration: Limitations

You should keep the following in mind as you rapidly iterate changes for a multiplayer game.

  • Establishing Game<->Tool communication is the same as for a single-player game. Communication will normally occur automatically if you are running the Asset Controller and your game includes the Rapid Iteration System Service (a non-shipping build configuration).
  • Console setup for multiplayer rapid iteration mirrors the setup for single-player rapid iteration, but with the following notable differences:
    • You will have to devise a way to designate Host and Peer instances of the game. This can be done easily by either including matchmaking or just using configuration variables to designate the roles.
    • The internal networking stacks are separate for rapid iteration communication and the game's online gameplay communication. You can independently establish the Game <-> Tools and Game <-> Game connections.
