Multiplayer Rapid Iteration: Secondary PC Setup

Note

This step is not required if you are running all instances of the game on the same PC where you are running Toolbench. If this describes your set up, skip this step and move ahead to Deploying Assets to Multiple Peers.

You can run Toolbench on a different PC than the PC(s) you use to run instances of your game. For example, you might want to optimize game or tool performance during development. Or, you might need to exercise the game on different hardware. Rapid iteration works well in this scenario. It is similar to developing on PC and running the game on a console.

First Time Configuration

The recommended approach is to set up a shared directory on the PC where you are running Toolbench. The shared directory would normally point to your <Asset Web Root Directory> and would be reachable via a UNC path from the PCs where your game is running. For example: \\my_tool_pc\GameContent.

After you set up the share, you need to modify the Config.ini for all game instances:

[AssetWeb]
Path = \\my_tool_pc\GameContent
OverrideSystemId=true

The ConfigCommon.ini for each peer instance running on a different PC from the host should be modified to include the hostname or IP address of the host machine:

[ChannelManager#]
Hostname=127.0.0.1

If running on multiple PCs, you will also need to modify the ConfigCommon.ini file for peer instances to specify the peer role explicitly:

[Game]
Role=Peer

Otherwise, each PC will launch a game instance in host mode.

Starting Your Rapid Iteration Session

Start your rapid iteration workflow as follows:

  1. Start Toolbench on your development PC.
  2. Open the game solution file.
  3. Start the game host instance.
  4. Start the game peer instance(s).

When the game instances and Toolbench are connected, you will see a green connection icon in your system tray. Hovering your mouse over the icon in the system tray will display a ToolTip similar to "Connections: 3". This indicates that the PC(s) where the game is running and the PC where Toolbench is running are connected. The actual number of connections should be 2 plus the number of peer-configured game instances. You can now open World Builder or another tool to begin a rapid iteration workflow.
