Multiplayer Rapid Iteration: Deploying Assets to Multiple Peers

The PC, PS3, and Xbox 360 set up sections in in this tutorial show you how to prepare the assets for an instance of the game before you start rapid iteration. You need to repeat the same steps for all game instances.

Deploying assets is identical on hosts and peers except in the case of configuration files. As already mentioned in the platform-specific sections: On a peer, the Game.Role parameter within ConfigCommon.ini should be set to "Peer". For the host (where the Channel Manager will be running), you should set the same parameter to "Host", or you can leave it blank. It may also be necessary to specify a hostname in the config files, to be sure peer instances can locate the host game instance.
