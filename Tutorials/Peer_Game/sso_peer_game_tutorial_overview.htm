Peer Game: Tutorial Overview

This tutorial walks you through the steps that you need to take to transform Base Game into a true multiplayer simulation.

We'll start off by using the Entity Modeling Tool to update the Base Game assets to support multiplayer. We will utilize the mixin models from a sample OnlineModelLibrary, which will implement basic Entity Replication support in our BaseGame entities.

The tutorial will go through the changes to the Base Game binary to enable it to be multiplayer-aware. Also, an examination of the code changes needed to add the ability to host a game or join a game in progress will be made. The game host will run a Channel Manager service, which routes messages between peers, ensuring in-order message delivery to all connected parties. The tutorial will also demonstrate how the Replication service is added to allow each process to participate in Entity Replication.

Prerequisites

To build and run this tutorial, you should have set up your environment to include required SDKs and tools. For more information about these requirements, see Build Prerequisites for Gamebryo LightSpeed for Win32.

You should be familiar with the concepts introduced in the Base Game tutorial.

You'll also need to ensure that the PhysX SDK is installed and the PHYSXINSTALLPATH environment variable is set.

Tip

Although this tutorial builds on the Base Game tutorial, it is possible to complete this tutorial without completing Base Game. The following instructions assume that you are starting from scratch, using the pre-built version of Base Game.
