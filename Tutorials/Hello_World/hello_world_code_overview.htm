Hello World Demo Code Overview

By default, files that are specific to the Hello World application install to the following locations:

  • Source files: %EMERGENT_PATH%\Samples\GameFrameworkTechDemos\HelloWorld
  • Entity behavior scripts: %EMERGENT_PATH%\Media\Samples\GameFramework\HelloWorld
  • Configuration files: %EMERGENT_PATH%\Samples\GameFrameworkTechDemos\HelloWorld
File Description
Source files
HelloWorld.cpp The system service that is specific to the Hello World application. Its main responsibilities are:
  • Start necessary system services
  • Start the game's main loop
  • Wait for other system services to complete initialization before the application actually starts the game
  • Asynchronously create the default camera and wait for creation to complete. A camera can be added to the level while creating it in Toolbench, but we create the default camera programmatically to simplify the application.
Main.cpp Contains C++ main functions that are specific to platforms. Implements the HelloWorld class, whose main responsibilities are:
  • Create and start the Hello World service
Entity behavior scripts
EseAIJack.lua This file implements the behaviors of the AIJack (small Jack) model.
EseAvatar.lua This file implements the behaviors of the Avatar (big Jack) model.
EseJack.lua This file implements the behaviors of the Jack model. These behaviors are common to the AIJack and Avatar, the two characters in the Hello World application.
point3.lua This is not a behavior script (no "Ese" prefix). This is a helper Lua file that implements 3D point related functions.
Configuration files
Note: The Hello World application expects the Config.ini file to be located in the directory where the application executable is located. When you deploy the application on your specific platform, copy the platform-specific configuration and ConfigCommon.ini to the directory where the executable is located. Then, rename the configuration file to Config.ini. Gamebryo LightSpeed ships with the solutions that include the post-build steps to actually perform renaming.
Config.ini Configuration file specific to the Windows platform.
ConfigCommon.ini File containing configuration common across all platforms. This file is included by the other platform specific configuration files.
ConfigPS3.ini Configuration file specific to the PS3 platform.
ConfigXbox360.ini Configuration file specific to the XBox360 platform.
